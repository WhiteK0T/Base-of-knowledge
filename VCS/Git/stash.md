# 🌿 git stash — временно спрятать правки

`git stash` убирает незакоммиченные изменения «в карман», возвращая рабочее дерево к чистому состоянию `HEAD`, — чтобы переключиться на другую задачу, сделать [`pull`](pull.md)/[`switch`](switch.md), а потом вернуть правки обратно. Это «черновой буфер», не часть истории веток.

## 🧬 Зачем нужен stash

Частая ситуация: ты в середине правок, но нужно срочно переключить ветку или забрать обновления — а Git не даёт («local changes would be overwritten»). Варианты: закоммитить «недоделку» или **спрятать**:

```bash
git stash            # спрятать правки, дерево станет чистым
git switch hotfix    # заняться срочным
# ... поправил, вернулся ...
git switch feature
git stash pop        # вернуть отложенные правки
```

> [!info] Что попадает в stash
> По умолчанию прячутся **отслеживаемые** изменённые файлы (и индекс, и рабочее дерево). **Неотслеживаемые** (новые) и игнорируемые файлы **не** прячутся — для них нужны флаги `-u`/`-a` (ниже).

## 📝 Базовое использование

```bash
git stash                        # спрятать (= git stash push)
git stash push -m "описание"     # спрятать с пометкой
git stash list                   # список заначек (stash@{0}, stash@{1}, ...)
git stash show -p stash@{0}      # посмотреть содержимое (diff)
git stash pop                    # вернуть последнюю И удалить её из списка
git stash apply                  # вернуть, но ОСТАВИТЬ в списке
git stash drop stash@{0}         # удалить запись
git stash clear                  # удалить ВСЕ заначки
```

## ⚙️ Основные опции и подкоманды

| Команда / опция | Описание |
| :--- | :--- |
| **`git stash`** (`push`) | спрятать изменения |
| **`-m "<текст>"` / `--message`** | подписать заначку |
| **`-u` / `--include-untracked`** | спрятать и **новые** (untracked) файлы |
| **`-a` / `--all`** | спрятать всё, включая **игнорируемые** |
| **`-k` / `--keep-index`** | спрятать, но оставить проиндексированное в дереве (для теста staged) |
| **`-p` / `--patch`** | выбрать **куски** для пряча (как [`git add -p`](add.md)) |
| **`git stash list`** | список заначек |
| **`git stash show [-p] <stash>`** | сводка / полный diff заначки |
| **`git stash pop [<stash>]`** | применить и удалить из списка |
| **`git stash apply [<stash>]`** | применить, не удаляя |
| **`git stash branch <имя> [<stash>]`** | создать ветку из заначки и применить туда |
| **`git stash drop <stash>`** / **`clear`** | удалить одну / все |

## 🗂️ Несколько заначек — стек

Stash работает как **стек**: новые ложатся сверху (`stash@{0}` — самая свежая).

```bash
git stash push -m "правки парсера"
git stash push -m "эксперимент с UI"
git stash list
# stash@{0}: On feature: эксперимент с UI
# stash@{1}: On feature: правки парсера
git stash apply stash@{1}        # применить конкретную (не верхнюю)
```

> [!tip] pop против apply
> `pop` = применить + удалить из стека (обычный случай). `apply` оставляет заначку в списке — удобно, если хочешь применить **те же** правки на несколько веток. После `apply` удали вручную: `git stash drop`.

## 🌿 Заначка → новая ветка

Если правки в stash конфликтуют с ушедшей вперёд веткой, удобно развернуть их в отдельную ветку от того коммита, где прятал:

```bash
git stash branch feature-x stash@{0}
```

Создаёт ветку `feature-x` от исходной базы заначки, применяет правки и удаляет заначку. Конфликтов не будет, т.к. база совпадает.

## ⚔️ Конфликты при возврате

`git stash pop` может конфликтовать, если ветка изменилась:

```text
CONFLICT (content): Merge conflict in file.txt
```

Разрешается как обычный конфликт (маркеры `<<<< ==== >>>>`, правка, [`git add`](add.md)). Важно: при **конфликтном** `pop` заначка **не удаляется** из стека автоматически — после разрешения убери вручную `git stash drop`.

## 🆚 stash против альтернатив

| Хочу… | Команда | Комментарий |
| :--- | :--- | :--- |
| отложить правки и вернуть позже | `git stash` | временный буфер, вне истории |
| сохранить правки как полноценный коммит | [`git commit`](commit.md) (можно `--amend` позже) | попадает в историю ветки |
| автоматически вокруг pull/rebase | `--autostash` ([pull](pull.md)/[rebase](rebase.md)) | Git сам прячет и возвращает |
| выбросить правки совсем | [`git restore`](restore.md) / `git checkout --` | необратимо |

## 🩺 Частые вопросы

- **Новый файл не спрятался** — untracked не входит по умолчанию: `git stash -u`.
- **Где мои заначки** — `git stash list`; содержимое — `git stash show -p stash@{N}`.
- **`pop` дал конфликт и не удалил заначку** — это by design: разреши конфликт, затем `git stash drop`.
- **Случайно `git stash clear`** — заначки не в reflog веток, но есть `git fsck --unreachable | grep commit` для поиска «висячих» stash-коммитов (сложно, не всегда). Лучше не клирить вслепую.
- **Заначка не на той ветке** — stash не привязан к ветке жёстко: можно `apply` на любой; при расхождении — `git stash branch`.

## 💎 Резюме

- Спрятать: `git stash` (`-u` чтобы и новые файлы, `-p` по кускам, `-m` с подписью).
- Список/просмотр: `git stash list`, `git stash show -p`.
- Вернуть: `pop` (и удалить) или `apply` (оставить в стеке).
- Развернуть в ветку: `git stash branch <имя>`.
- Для pull/rebase удобнее `--autostash`, чем ручной stash.
