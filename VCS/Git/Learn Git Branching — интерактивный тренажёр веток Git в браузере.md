# 🌳 Learn Git Branching — интерактивный тренажёр веток Git в браузере

**Learn Git Branching** ([learngitbranching.js.org](https://learngitbranching.js.org/?locale=ru_RU), [pcottle/learnGitBranching](https://github.com/pcottle/learnGitBranching), **MIT**, ~**33.6k★**, JavaScript) — классический браузерный **тренажёр Git** с живым **визуальным графом коммитов**: вводишь команду — и на дереве сразу видно, как двигаются ветки и указатели. Идеален, чтобы «увидеть» то, что в консоли остаётся абстрактным: `branch`, `merge`, `rebase`, `cherry-pick`, перемещение `HEAD`, работа с удалёнкой. Есть **русская локализация** (`?locale=ru_RU` или переключатель языка в интерфейсе). Полностью **бесплатный и опенсорсный**, живёт с 2012 года и до сих пор обновляется.

> [!info] Что внутри
> - **Уровни с заданиями** (от простого к сложному): вступление → ветки и слияния → перемещение работы (`rebase`, `cherry-pick`, интерактивный rebase) → смешанные задачи; отдельный блок про **удалённые репозитории** (push/pull, fetch, origin).
> - **Песочница (sandbox)** — свободный режим: вводи любые команды и смотри на граф без условий задачи.
> - **Визуальное дерево** коммитов в реальном времени + краткие пояснения к каждому уровню.
> - Команда `levels` — список уровней, `help` — справка, можно делать и **свои уровни**.

## 🧪 Факты против хайпа

> [!warning] Это симулятор графа, а не настоящий Git
> Тренажёр работает с **визуализированным «понарошку»-репозиторием** в браузере — он **не трогает твои реальные файлы** и не запускает настоящий `git`. Это его сила (безопасно экспериментировать, мгновенно видно результат) и одновременно ограничение:
> - Отлично учит **топологию веток** и логику `merge`/[`rebase`](rebase.md)/[`cherry-pick`](cherry-pick.md)/`reset` — то, что путает новичков.
> - **Не учит** реальной рутине CLI: индекс/staging ([`git add`](add.md)), рабочую директорию, **ручное разрешение конфликтов слияния**, аутентификацию к удалёнке, `.gitignore`, stash и т.п. Конфликты тут не возникают — `merge`/`rebase` в симуляторе всегда «проходят».
> - **Вывод:** проходи тренажёр для понимания **модели веток**, но настоящие команды закрепляй на живом репозитории (см. серию заметок ниже и [карту команд](Git%20%E2%80%94%20%D0%BA%D0%B0%D1%80%D1%82%D0%B0%20%D0%BA%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%20%28%D1%88%D0%BF%D0%B0%D1%80%D0%B3%D0%B0%D0%BB%D0%BA%D0%B0%29.md)).

> [!tip] Как пройти с пользой
> 1. Сначала **вступительные уровни** — закрепи, что коммит это снимок, а ветка — подвижный указатель.
> 2. Дойди до **rebase vs merge** — именно здесь визуализация даёт больше всего: видно, что rebase *переписывает* историю, а merge *сводит*.
> 3. Пройди блок **Remote** — наглядно про `origin`, отслеживаемые ветки, `fetch` vs `pull`.
> 4. После каждого уровня — повтори ту же операцию руками в тестовом репозитории (`git init` в пустой папке), чтобы перенести понимание в реальный CLI.

## 💻 Как это ложится на твои системы

| Система | Применимость |
| :--- | :--- |
| **Gentoo / Debian / Arch** (десктоп) | Просто **открой сайт в браузере** — ставить нечего, всё крутится на стороне клиента. Работает офлайн после загрузки страницы. |
| **Самостоятельный хостинг** | Это **статический сайт** — можно собрать (`git clone` + `npm install` + `grunt`) и раздавать локально/в LAN с любого из десктопов. |
| **Entware / RT-AX56U** | Для **прохождения** — N/A (на роутере нет браузера). Теоретически роутер мог бы **раздавать** собранную статику другим устройствам в сети, но сборка тяжёлая (Node/grunt) — проще держать на десктопе. |

## 💡 Кому полезно

- **Новичкам в Git** — самый наглядный способ понять ветки/слияния/rebase без страха «сломать» репозиторий.
- **Тем, кто «вроде умеет git, но боится rebase/cherry-pick»** — визуализация снимает страх перед «опасными» командами.
- **Преподавателям/менторам** — готовые уровни и песочница для демонстраций.
- **Дополнение, а не замена** практике на реальном репозитории и серии заметок по командам.
