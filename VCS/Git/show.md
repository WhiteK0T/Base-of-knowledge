# 🌿 git show — показать объект (коммит, тег, файл)

`git show` выводит подробности **одного** Git-объекта: для коммита — сообщение + его diff (изменения против родителя), для тега — аннотацию, для файла на коммите — его содержимое. Это «крупный план» для одной точки истории (в отличие от [`git log`](log.md), который перечисляет много коммитов).

```bash
git show                 # последний коммит (HEAD): сообщение + diff
git show <commit>        # конкретный коммит
```

## 🧬 Что показывает show

Для коммита по умолчанию:

```text
commit a1b2c3d... (HEAD -> master)
Author: WhiteK0T <mail@example.com>
Date:   Sat Jun 14 01:30:00 2026 +0300

    VCS/Git: add git show note

diff --git a/VCS/Git/show.md b/VCS/Git/show.md
...
```

То есть **метаданные + изменения именно этого коммита** (его diff против первого родителя). См. формат diff в заметке [`git diff`](diff.md).

## 📝 Частые формы

```bash
git show HEAD~2                  # коммит «2 назад»
git show <commit> --stat         # только сводка изменённых файлов
git show <commit>:path/to/file   # содержимое файла НА этом коммите
git show <commit> -- path        # diff коммита только по указанному пути
git show v1.0.0                  # аннотированный тег (сообщение + целевой коммит)
git show <branch>                # последний коммит ветки
```

> [!tip] Достать старую версию файла
> `git show <commit>:<путь>` печатает файл, каким он был на том коммите, — удобно сравнить или восстановить:
> ```bash
> git show HEAD~5:config.yml > config.old.yml
> git show main:README.md | less
> ```
> Путь — **от корня репозитория**. Для текущей версии из индекса: `git show :config.yml`.

## ⚙️ Полезные опции

| Опция | Описание |
| :--- | :--- |
| **`--stat`** | сводка файлов и +/- строк вместо полного diff |
| **`--name-only`** | только список изменённых файлов |
| **`--name-status`** | файлы + буква статуса (`M`/`A`/`D`) |
| **`-p`** | показать патч (по умолчанию для коммита) |
| **`--no-patch` / `-s`** | без diff — только сообщение/метаданные |
| **`--oneline`** | компактный заголовок коммита |
| **`-w`** | игнорировать различия в пробелах |
| **`--format=<fmt>`** | свой формат заголовка (как `--pretty` в [log](log.md)) |
| **`-m`** | для merge-коммита — показать diff против **каждого** родителя |

## 🔀 Просмотр merge-коммита

У merge-коммита два родителя, и по умолчанию `git show` показывает «комбинированный» diff (часто почти пустой). Чтобы увидеть изменения против каждой стороны:

```bash
git show -m <merge-commit>       # diff против обоих родителей
git show <merge> --first-parent  # только против основной линии
```

## ⚡ Частые сценарии

```bash
git show                         # что я только что закоммитил?
git show <commit> --stat         # какие файлы затронул коммит
git show <commit>:src/app.py     # как выглядел файл в том коммите
git show <tag>                   # на какой коммит указывает тег и его аннотация
```

## 🩺 Частые вопросы

- **`show` против `log`** — `log` перечисляет **много** коммитов; `show` — детально **один** объект.
- **`show` против `diff`** — `git show <commit>` = изменения коммита (против родителя); [`git diff`](diff.md) сравнивает **произвольные** две точки/состояния.
- **Merge-коммит показывает «пусто»** — комбинированный diff; используй `git show -m` или `--first-parent`.
- **Нужно содержимое файла из прошлого** — `git show <commit>:<путь>` (или восстановить через [`git restore --source`](restore.md)).

## 💎 Резюме

- `git show [<commit>]` — метаданные + diff одного коммита (по умолчанию HEAD).
- Файл на коммите: `git show <commit>:<путь>`; сводка: `--stat`; без diff: `-s`.
- Тег: `git show <tag>`; merge: `git show -m`.
- Много коммитов — [`log`](log.md); произвольное сравнение — [`diff`](diff.md).
