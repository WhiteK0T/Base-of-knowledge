# 🌿 git diff — просмотр изменений

`git diff` показывает **построчную разницу** между состояниями: рабочим деревом, индексом, коммитами, ветками. Если [`git status`](status.md) отвечает на вопрос «**какие** файлы изменены», то `git diff` — «**что именно** в них поменялось».

## 🧬 Три «слоя» сравнения

Ключ к пониманию `git diff` — между **чем и чем** он сравнивает (см. зоны в [`git add`](add.md)):

| Команда | Сравнивает | Отвечает на вопрос |
| :--- | :--- | :--- |
| `git diff` | рабочее дерево **↔** индекс | что я изменил, но **ещё не** добавил (`git add`)? |
| `git diff --staged` | индекс **↔** последний коммит | что **уйдёт** в следующий коммит? |
| `git diff HEAD` | рабочее дерево **↔** последний коммит | все изменения с последнего коммита (staged + unstaged) |

> [!info] `--staged` = `--cached`
> Это синонимы. `git diff --cached` и `git diff --staged` — одно и то же: разница индекса с `HEAD`.

```text
        git diff            git diff --staged
рабочее ───────────► индекс ──────────────────► HEAD (коммит)
 дерево                                  
        └──────────── git diff HEAD ────────────┘
```

## 📝 Базовое использование

```bash
git diff                      # незастейдженные правки (дерево ↔ индекс)
git diff --staged             # застейдженные правки (индекс ↔ HEAD)
git diff HEAD                 # все правки с последнего коммита
git diff <файл>               # только по одному файлу/пути
git diff -- path/to/dir       # ограничить путём (после -- )
```

## 🔀 Сравнение коммитов и веток

```bash
git diff <commitA> <commitB>      # между двумя коммитами
git diff HEAD~3 HEAD              # последние 3 коммита суммарно
git diff main feature            # между ветками (что в feature, чего нет в main — и наоборот)
git diff main...feature          # с момента их РАСХОЖДЕНИЯ (от общего предка)
git diff v1.0 v2.0 -- src/       # изменения каталога между тегами
```

> [!warning] Две точки `..` против трёх `...`
> - **`git diff A..B`** (или просто `A B`) — прямая разница между состояниями A и B.
> - **`git diff A...B`** — разница от **общего предка** A и B до B (что появилось в ветке B с момента ответвления). Удобно для ревью: «что добавила только моя ветка».

## ⚙️ Полезные опции

| Опция | Описание |
| :--- | :--- |
| **`--staged` / `--cached`** | сравнить индекс с `HEAD` (что уйдёт в коммит) |
| **`--stat`** | сводка: файлы + сколько строк добавлено/удалено (гистограмма `+/-`) |
| **`--shortstat`** | только итоговая строка статистики |
| **`--name-only`** | только список изменённых файлов |
| **`--name-status`** | файлы + буква статуса (`M`/`A`/`D`/`R`) |
| **`-w` / `--ignore-all-space`** | игнорировать различия в пробелах |
| **`--word-diff`** | разница **по словам**, а не по строкам (удобно для текста/прозы) |
| **`--color-words`** | пословно, с цветом, без `+/-` маркеров |
| **`-U<n>` / `--unified=<n>`** | сколько строк контекста показывать (по умолчанию 3) |
| **`-M` / `--find-renames`** | распознавать переименования (показать как rename, а не delete+add) |
| **`--check`** | предупредить о «мусоре»: пробелы в конце строк, конфликтные маркеры |
| **`-- <путь>`** | ограничить вывод указанными путями |

## 🔬 Как читать вывод

```diff
diff --git a/file.py b/file.py
index 83db48f..bf3a2c1 100644
--- a/file.py            # «было» (a/) — старая версия
+++ b/file.py            # «стало» (b/) — новая версия
@@ -12,3 +12,4 @@ def main():     # hunk: 3 строки старой / 4 новой, с 12-й
     context line
-    old_line             # удалённая строка
+    new_line             # добавленная строка
+    another_new_line
     context line
```

- **`diff --git a/file.py b/file.py`** — строка-**заголовок** вывода (не команда!): git помечает начало diff'а по файлу. `--git` — это формат git, а не опция утилиты `diff`.
- **`index 83db48f..bf3a2c1 100644`** — хеши объектов-блобов «до..после» и режим файла (`100644` = обычный файл, `100755` = исполняемый).
- **`a/`** — версия «до», **`b/`** — «после».
- **`@@ -12,3 +12,4 @@`** — «кусок» (hunk): со строки 12 показано **3** строки старой версии и **4** новой. Формат: `@@ -старая_строка,кол-во +новая_строка,кол-во @@`. Счётчик = контекст + удалённые (для старой) / контекст + добавленные (для новой); здесь 2 контекста +1 удалённая = 3, и 2 контекста +2 добавленных = 4.
- **`-`** (красное) — удалено, **`+`** (зелёное) — добавлено, строки без префикса — контекст (есть в обеих версиях).
- Текст после второго `@@` (`def main():`) — **подсказка**: ближайший контекст (обычно сигнатура функции), куда попадает изменение.

> [!tip] Сводка вместо «простыни»
> Перед коммитом удобно глянуть масштаб: `git diff --staged --stat` (сколько файлов и строк), а детали смотреть уже по конкретному файлу `git diff --staged <файл>`.

## 🖥️ Внешний просмотрщик — `git difftool`

`git difftool` открывает разницу во внешней программе (meld, vimdiff, kdiff3, VS Code) вместо терминала:

```bash
git difftool                          # по очереди для каждого файла
git config --global diff.tool meld    # выбрать инструмент
git config --global difftool.prompt false  # не спрашивать перед каждым файлом
```

> [!note] Пейджер и цвет
> Длинный diff Git показывает через пейджер (`less`): прокрутка стрелками/`PgUp`/`PgDn`, выход — `q`. Цвет включён по умолчанию (`color.ui auto`). Для постраничного «не-интерактивного» вывода: `git --no-pager diff`.

## 🩺 Частые вопросы

- **`git diff` пустой, хотя файлы менялись** — изменения уже **застейджены**; смотри `git diff --staged` (или сразу всё — `git diff HEAD`).
- **Весь файл показан как изменённый** — обычно перевод строк (CRLF↔LF) или режим прав. Помогает `git diff -w`; для CRLF — настроить `core.autocrlf`/`.gitattributes`.
- **Хочу разницу для удалённой ветки** — сначала `git fetch`, затем `git diff main origin/main`.
- **Нужна разница в коммите** — это уже `git show <commit>` (коммит против его родителя), а не `git diff`.

## 💎 Резюме

- Не добавлено: `git diff`; уйдёт в коммит: `git diff --staged`; всё с последнего коммита: `git diff HEAD`.
- Между ветками для ревью: `git diff main...feature` (от точки расхождения).
- Масштаб: `--stat`; только файлы: `--name-only`/`--name-status`.
- Игнор пробелов: `-w`; пословно: `--word-diff`; внешний инструмент: `git difftool`.
