# 🌿 git switch — переключение веток

`git switch` переключает текущую **ветку** (двигает `HEAD` на другую ветку и обновляет рабочее дерево под неё). Появился в Git **2.23** (2019) вместе с [`git restore`](restore.md), чтобы разделить перегруженный `git checkout`: `switch` отвечает **только за ветки**, `restore` — за файлы.

> [!info] switch работает с ВЕТКАМИ, не с файлами
> `git switch` меняет, **на какой ветке** ты находишься (и подгоняет рабочее дерево). Он не откатывает отдельные файлы — для этого [`git restore`](restore.md), и не двигает указатель ветки на произвольный коммит — для этого [`git reset`](reset.md).

## 📝 Базовое использование

```bash
git switch main              # переключиться на существующую ветку main
git switch -c new-feature    # создать новую ветку и сразу перейти на неё
git switch -                 # вернуться на предыдущую ветку (как `cd -`)
```

При переключении Git обновляет файлы рабочего дерева под целевую ветку. **Незакоммиченные правки**, которые конфликтуют с переключением, Git **не даст потерять** — откажет и попросит закоммитить или спрятать (`git stash`).

## ⚙️ Основные опции

| Опция | Описание | Пример |
| :--- | :--- | :--- |
| **`-c` / `--create <имя>`** | создать новую ветку от текущего `HEAD` и перейти | `git switch -c fix-123` |
| **`-c <имя> <старт>`** | создать ветку от указанной точки (коммит/ветка/тег) | `git switch -c hotfix main` |
| **`-C` / `--force-create`** | создать/пересоздать ветку, даже если такая есть (сброс на старт) | `git switch -C feature` |
| **`-`** | переключиться на **предыдущую** ветку | `git switch -` |
| **`-d` / `--detach <rev>`** | перейти в режим **detached HEAD** на коммит/тег | `git switch -d v1.0` |
| **`--orphan <имя>`** | создать «сиротскую» ветку без истории | `git switch --orphan gh-pages` |
| **`-m` / `--merge`** | перенести локальные правки на новую ветку через слияние | `git switch -m other` |
| **`-f` / `--force` / `--discard-changes`** | переключиться, **отбросив** локальные правки | `git switch -f main` |

## 🌿 Создать ветку из удалённой

Если на сервере есть `origin/feature`, которой нет локально:

```bash
git fetch origin
git switch feature           # авто-создаст локальную feature, отслеживающую origin/feature
```

Git распознаёт одноимённую удалённую ветку и сам заводит локальную с настроенным upstream (эквивалент старого `git checkout --track`). Явно:

```bash
git switch -c feature --track origin/feature
```

## 🪂 Detached HEAD — «отсоединённая голова»

```bash
git switch -d <коммит>       # или git switch --detach <тег>
```

`HEAD` указывает **прямо на коммит**, а не на ветку. Можно смотреть/собирать старое состояние, но **новые коммиты не принадлежат ни одной ветке** и потеряются после переключения.

> [!warning] Коммиты в detached HEAD «бесхозны»
> Если в detached-состоянии сделать коммиты и уйти на другую ветку, они станут недостижимыми (их подберёт сборщик мусора). Чтобы сохранить работу — заранее создай ветку: `git switch -c new-branch`. Найти «потерянные» поможет [`git reflog`](log.md).

## 🆚 switch против checkout и reset

| Задача | Современно (рекомендуется) | Старый способ |
| :--- | :--- | :--- |
| переключить ветку | `git switch <ветка>` | `git checkout <ветка>` |
| создать + перейти | `git switch -c <ветка>` | `git checkout -b <ветка>` |
| перейти на коммит (detached) | `git switch -d <rev>` | `git checkout <rev>` |
| откатить файл | `git restore <файл>` | `git checkout -- <файл>` |
| сдвинуть **указатель ветки** | `git reset <commit>` | — |

> [!tip] Почему switch безопаснее checkout
> Старый `git checkout main` и `git checkout file.txt` выглядят похоже, но делают **совершенно разное** — отсюда опасные опечатки (вместо ветки случайно затереть файл). `switch` (только ветки) и `restore` (только файлы) убирают эту двусмысленность. `checkout` остаётся для совместимости и не исчезает, но в новых командах лучше `switch`/`restore`.

## 🩺 Частые вопросы

- **«error: Your local changes would be overwritten»** — незакоммиченные правки мешают переключению: закоммить, `git stash`, либо `git switch -m` (перенести слиянием) / `-f` (отбросить).
- **switch против checkout** — функционально пересекаются; `switch` — современный и однозначный, `checkout` — легаси-комбайн.
- **Создал ветку не от той точки** — `git switch -C <имя> <правильный-старт>` пересоздаст её от нужного места.
- **Случайно накоммитил в detached HEAD** — `git switch -c <имя>` **до** ухода сохранит коммиты в ветке; если уже ушёл — ищи хеш в `git reflog`.

## 💎 Резюме

- Переключиться: `git switch <ветка>`; создать и перейти: `git switch -c <ветка>`.
- Назад на прошлую ветку: `git switch -`.
- Из удалённой: `git fetch` + `git switch <имя>` (авто-track).
- Посмотреть старое состояние: `git switch -d <rev>` (detached — коммить только создав ветку).
- Ветки → `switch`, файлы → `restore`, указатель/история → `reset`.
