# 🌿 git init — создать репозиторий

`git init` создаёт **новый** Git-репозиторий в текущем (или указанном) каталоге: заводит скрытую папку `.git/`, где будет жить вся история. С этой команды начинается версионирование проекта «с нуля» (в отличие от [`git clone`](clone.md), который копирует уже существующий).

```bash
git init                 # инициализировать репозиторий в текущем каталоге
```

## 🧬 Что создаёт init

Появляется каталог **`.git/`** — «сердце» репозитория:

| Внутри `.git/` | Что хранит |
| :--- | :--- |
| `objects/` | все объекты (коммиты, деревья, блобы) |
| `refs/` | ветки (`refs/heads/`) и теги (`refs/tags/`) |
| `HEAD` | указатель на текущую ветку |
| `config` | локальные настройки репозитория |
| `hooks/` | примеры хуков (`pre-commit` и др.) |

> [!info] init не трогает твои файлы
> `git init` лишь создаёт `.git/` — существующие файлы остаются как есть, но **пока не отслеживаются**. Чтобы начать версионировать: [`git add`](add.md) → [`git commit`](commit.md). Удалить репозиторий = удалить каталог `.git/` (история пропадёт, файлы останутся).

## 📝 Базовое использование

```bash
git init                         # в текущем каталоге
git init my-project              # создать каталог my-project и репозиторий в нём
git init -b main                 # сразу задать имя начальной ветки (Git ≥ 2.28)
```

## ⚙️ Полезные опции

| Опция | Описание |
| :--- | :--- |
| **`-b <имя>` / `--initial-branch`** | имя начальной ветки (вместо дефолтной) |
| **`-q` / `--quiet`** | без лишнего вывода |
| **`--bare`** | «голый» репозиторий без рабочего дерева (для сервера/общего origin) |
| **`--shared[=<режим>]`** | общий доступ группе (права на запись нескольким пользователям) |
| **`--template=<dir>`** | взять стартовое наполнение `.git` из шаблона |
| **`--separate-git-dir=<dir>`** | разместить `.git` вне рабочего каталога (положить файл-ссылку) |

> [!tip] Имя начальной ветки
> Исторически дефолт — `master`; многие проекты перешли на `main`. Задать разово: `git init -b main`. Глобально по умолчанию: `git config --global init.defaultBranch main`.

## ⚡ Типовой старт нового проекта

```bash
git init -b main                 # 1. создать репозиторий
# 2. (опц.) задать игнор и идентичность
printf 'node_modules/\n*.log\n' > .gitignore
git config user.name "WhiteK0T"  # локально для этого репозитория
git config user.email "you@example.com"
# 3. первый коммит
git add -A
git commit -m "Первый коммит"
# 4. (опц.) подключить удалёнку и запушить
git remote add origin git@github.com:User/repo.git
git push -u origin main
```

> [!note] init + remote add = «ручной clone»
> Если репозиторий уже есть на GitHub (создан там пустым), можно `git init` локально, добавить [`git remote add origin`](remote.md) и [`git push -u`](push.md) — это эквивалент клонирования «наоборот»: связать локальный проект с удалённым.

## 🗄️ `--bare` — серверный репозиторий

`git init --bare` создаёт репозиторий **без рабочего дерева** — только содержимое `.git/` (по соглашению каталог называют `repo.git`). В него нельзя коммитить напрямую, он служит **точкой обмена** (тем, куда пушат):

```bash
git init --bare /srv/git/project.git     # на сервере
# на клиенте:
git clone user@server:/srv/git/project.git
```

> [!info] Почему сервер «голый»
> В обычный (не bare) репозиторий нельзя пушить в текущую ветку — это сломало бы чужое рабочее дерево. Поэтому общий `origin` всегда **bare**: у него нет рабочих файлов, только история. См. также [`clone --bare/--mirror`](clone.md).

## 🩺 Частые вопросы

- **`init` против `clone`** — `init` создаёт **пустой** репозиторий с нуля; [`clone`](clone.md) копирует **существующий** с сервера.
- **Случайно сделал `git init` не там** — просто удали `.git/` в этом каталоге (`rm -rf .git`), файлы не пострадают.
- **Повторный `git init`** — безопасен (идемпотентен): не затирает существующий репозиторий, лишь до-инициализирует.
- **Хочу переименовать `master` → `main` после init** — `git branch -m master main` (см. [branch](branch.md)); на будущее — `init.defaultBranch`.
- **Где задать имя/почту** — `git config user.name`/`user.email` (локально для репозитория — без `--global`).

## 💎 Резюме

- `git init [каталог]` — новый репозиторий (создаёт `.git/`), файлы пока не отслеживаются.
- Имя начальной ветки: `-b main` (или глобально `init.defaultBranch`).
- Старт: `init` → `.gitignore` → `add` → `commit` → (`remote add` + `push -u`).
- Серверный/общий origin — `git init --bare` (без рабочего дерева).
- Скопировать существующий вместо создания — [`git clone`](clone.md).
