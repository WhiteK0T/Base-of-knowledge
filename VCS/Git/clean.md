# 🌿 git clean — удалить неотслеживаемые файлы

`git clean` удаляет из рабочего дерева **неотслеживаемые** (untracked) файлы: артефакты сборки, временные файлы, мусор. В отличие от [`git rm`](rm.md) (для отслеживаемых) — это «уборка» того, чего Git ещё не знает.

> [!danger] clean удаляет файлы безвозвратно
> Удалённые `git clean` файлы **не** попадают ни в индекс, ни в историю, ни в [`reflog`](reflog.md) — их **нельзя восстановить** средствами Git. Поэтому из соображений безопасности `git clean` без `-n` или `-f` вообще **отказывается** работать. Всегда сначала прогоняй `-n` (dry-run).

## 📝 Базовое использование

```bash
git clean -n                     # СНАЧАЛА: показать, что будет удалено (ничего не трогая)
git clean -f                     # удалить неотслеживаемые ФАЙЛЫ
git clean -fd                    # + неотслеживаемые каталоги
git clean -fdx                   # + игнорируемые (.gitignore) файлы тоже
git clean -fdi                   # интерактивный режим
```

> [!warning] Привычка: всегда `-n` перед `-f`
> ```bash
> git clean -nd      # посмотреть
> git clean -fd      # выполнить
> ```
> `git clean -n` (или `--dry-run`) печатает список «Would remove …», ничего не удаляя. Только убедившись, что в списке нет нужного, запускай с `-f`.

## ⚙️ Основные опции

| Опция | Описание |
| :--- | :--- |
| **`-n` / `--dry-run`** | показать, что было бы удалено, без удаления |
| **`-f` / `--force`** | реально удалить (без него clean не работает) |
| **`-d`** | удалять и неотслеживаемые **каталоги** |
| **`-x`** | удалять и **игнорируемые** (`.gitignore`) файлы тоже |
| **`-X`** | удалять **только** игнорируемые (оставить прочие untracked) |
| **`-i` / `--interactive`** | интерактивно выбрать, что удалять |
| **`-e <шаблон>` / `--exclude`** | исключить из удаления (не трогать) |

> [!note] `-x` против `-X`
> - **`-x`** — снести **всё** неотслеживаемое, включая игнорируемое (полная очистка, как «чистый клон»).
> - **`-X`** — удалить **только** игнорируемое (артефакты сборки), оставив новые рабочие файлы.

## ⚡ Частые сценарии

### Полностью «обнулить» рабочее дерево до состояния коммита

```bash
git reset --hard HEAD            # откатить отслеживаемые правки
git clean -fdx                   # снести весь неотслеживаемый + игнорируемый мусор
```

Комбинация `reset --hard` + `clean -fdx` приводит каталог к состоянию «как после свежего клона» (см. [`git reset`](reset.md)).

### Убрать только артефакты сборки

```bash
git clean -fdX                   # только то, что под .gitignore (build/, *.o, ...)
```

### Не трогать конкретное

```bash
git clean -fd -e "*.env" -e local-notes/
```

## 🆚 clean против rm / reset

| Хочу… | Команда |
| :--- | :--- |
| удалить **неотслеживаемый** мусор | `git clean -fd` |
| удалить **отслеживаемый** файл | [`git rm`](rm.md) |
| откатить правки **отслеживаемых** файлов | [`git restore`](restore.md) / [`git reset --hard`](reset.md) |
| спрятать правки, а не удалять | [`git stash -u`](stash.md) |

> [!tip] Сомневаешься — `stash`, а не `clean`
> Если не уверен, нужны ли неотслеживаемые файлы, безопаснее [`git stash -u`](stash.md) (спрячет, в т.ч. untracked) — потом сможешь вернуть. `git clean` уничтожает без возврата.

## 🩺 Частые вопросы

- **`git clean` ничего не делает** — нужен `-f` (защита от случайного запуска); сначала `-n`.
- **Каталоги остаются** — добавь `-d`.
- **Игнорируемые файлы не удаляются** — по умолчанию clean их щадит; `-x` (всё) или `-X` (только их).
- **Удалил нужное** — Git не поможет: untracked-файлов нет ни в истории, ни в reflog. Используй файловые undelete-утилиты ОС (часто безуспешно).
- **Хочу подтверждать каждый** — `git clean -di` (интерактивно).

## 💎 Резюме

- `git clean` — удаляет **неотслеживаемые** файлы; **безвозвратно**.
- Всегда сначала `-n` (dry-run), затем `-f`; каталоги — `-d`; игнор — `-x`/`-X`.
- «Чистый лист»: [`git reset --hard`](reset.md) + `git clean -fdx`.
- Не уверен — [`git stash -u`](stash.md) вместо clean.
- Отслеживаемые файлы — это [`git rm`](rm.md), не clean.
