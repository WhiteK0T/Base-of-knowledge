# 🌿 git rm — удалить отслеживаемый файл

`git rm` удаляет файл **и из рабочего дерева, и из индекса** — то есть готовит удаление к коммиту. От обычного `rm` отличается тем, что сразу проводит удаление через Git (его не нужно отдельно [`git add`](add.md)). Работает только с **отслеживаемыми** файлами.

```bash
git rm file.txt          # удалить файл с диска И из индекса
git commit -m "Удалить file.txt"
```

## 🧬 rm против `--cached`

| Команда | Рабочее дерево | Индекс (Git) | Итог |
| :--- | :--- | :--- | :--- |
| `git rm file` | **удалён** | удалён | файла нет нигде — удаление уйдёт в коммит |
| `git rm --cached file` | **остаётся** | удалён | файл на диске цел, но Git **перестаёт** его отслеживать |

> [!tip] `--cached` — «перестать отслеживать, не удаляя»
> Главный сценарий: случайно закоммитил то, что должно игнорироваться (`.env`, `node_modules/`). `git rm --cached` убирает из репозитория, оставляя на диске; дальше добавляешь в [`.gitignore`](gitignore.md):
> ```bash
> git rm --cached secrets.env
> echo "secrets.env" >> .gitignore
> git commit -m "Перестать отслеживать secrets.env"
> ```

## 📝 Базовое использование

```bash
git rm file.txt                  # один файл
git rm -r logs/                  # рекурсивно каталог
git rm --cached file             # только из индекса (файл на диске остаётся)
git rm '*.log'                   # по шаблону (кавычки — чтобы шаблон раскрыл Git, не shell)
git rm -f file                   # принудительно, если есть несохранённые изменения
```

## ⚙️ Основные опции

| Опция | Описание |
| :--- | :--- |
| **`-r`** | рекурсивно (для каталогов) |
| **`--cached`** | удалить только из индекса, файл в рабочем дереве оставить |
| **`-f` / `--force`** | удалить, даже если есть незакоммиченные/неотслеженные различия |
| **`-n` / `--dry-run`** | показать, что было бы удалено, ничего не делая |
| **`--ignore-unmatch`** | не падать с ошибкой, если шаблон ничего не нашёл (для скриптов) |
| **`--`** | отделить опции от путей (если имя начинается с `-`) |

> [!warning] git rm удаляет реальные файлы
> Без `--cached` команда стирает файл с диска. Если он был **изменён** или проиндексирован, Git попросит `-f`. Восстановить **закоммиченный** файл можно из истории ([`git restore --source`](restore.md) / [`git checkout <commit> -- file`](switch.md)), а вот несохранённые правки пропадут.

## 🆚 Соседние команды

| Хочу… | Команда |
| :--- | :--- |
| удалить файл из репозитория (и диска) | `git rm <файл>` |
| перестать отслеживать, но оставить на диске | `git rm --cached <файл>` |
| удалить **неотслеживаемые** (мусорные) файлы | `git clean -fd` |
| переместить/переименовать | `git mv <из> <в>` |

> [!note] rm — для отслеживаемых, clean — для неотслеживаемых
> `git rm` работает только с тем, что Git знает. Чтобы вычистить **untracked** мусор (артефакты сборки, временные файлы), нужен `git clean` — это другая команда.

## 🩺 Частые вопросы

- **Удалил обычным `rm`** — Git увидит удаление как изменение; зафиксируй его `git rm <файл>` или `git add -u` / `git add -A`, затем коммить.
- **«error: the following file has changes staged…»** — есть несохранённые различия: `git rm -f` (осознанно) или сначала разберись с правками.
- **Случайно `git rm`, ещё не коммитил** — восстановить: `git restore --staged --worktree <файл>` (вернёт из индекса/HEAD).
- **Перестать отслеживать каталог** — `git rm -r --cached <каталог>/` + правило в `.gitignore`.
- **Файл остался в истории** — `git rm` убирает из будущих коммитов, но не из прошлых; для полной чистки — `git filter-repo` (секреты считай скомпрометированными).

## 💎 Резюме

- `git rm <файл>` — удалить из дерева и индекса (удаление уйдёт в коммит).
- `git rm --cached <файл>` — перестать отслеживать, оставив на диске (пара к [`.gitignore`](gitignore.md)).
- Каталог: `-r`; принудительно: `-f`; проверка: `-n`.
- Откатить до коммита: [`git restore --staged --worktree`](restore.md).
- Неотслеживаемый мусор — не `rm`, а `git clean`.
