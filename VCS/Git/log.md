# 🌿 git log — история коммитов

`git log` показывает **историю коммитов** текущей ветки — от свежих к старым. Это основной инструмент «посмотреть, что и когда менялось, кто автор и почему». Вывод гибко настраивается: от полного лога до однострочного графа веток.

## 📝 Базовое использование

```bash
git log                  # полный лог (хеш, автор, дата, сообщение)
git log <файл>           # история конкретного файла
git log <ветка>          # история другой ветки
git log origin/master    # история удалённой ветки (после fetch)
```

Полный вывод по коммиту:

```text
commit a1b2c3d4e5f6... (HEAD -> master, origin/master)
Author: WhiteK0T <mail@example.com>
Date:   Sat Jun 13 17:00:00 2026 +0300

    VCS/Git: add git log note
```

Лог открывается в **пейджере** (`less`): прокрутка стрелками/`PgUp`/`PgDn`, поиск `/текст`, выход — `q`.

## ⚙️ Формат вывода

| Опция | Что делает |
| :--- | :--- |
| **`--oneline`** | по одной строке на коммит (короткий хеш + тема) |
| **`--graph`** | ASCII-граф ветвлений/слияний слева |
| **`--decorate`** | показать имена веток/тегов у коммитов (в новых Git включено по умолчанию) |
| **`--all`** | коммиты **всех** веток, а не только текущей |
| **`--stat`** | сводка изменённых файлов и строк по каждому коммиту |
| **`-p` / `--patch`** | показать **diff** каждого коммита |
| **`--name-only`** | только список изменённых файлов |
| **`--abbrev-commit`** | сокращённые хеши |
| **`--reverse`** | от старых к новым |
| **`-<n>`** (напр. `-5`) | только последние `n` коммитов |

> [!tip] Самый полезный однострочник
> ```bash
> git log --oneline --graph --decorate --all
> ```
> Компактный граф всех веток с метками — идеально, чтобы понять, «куда всё расходится». Удобно повесить на алиас (ниже).

## 🎯 Фильтрация коммитов

| Фильтр | Пример |
| :--- | :--- |
| **по числу** | `git log -10` (последние 10) |
| **по дате** | `git log --since="2 weeks ago" --until="yesterday"` |
| **по автору** | `git log --author="WhiteK0T"` |
| **по тексту сообщения** | `git log --grep="CVE"` |
| **по содержимому изменений** (pickaxe) | `git log -S"имя_функции"` — коммиты, где строка добавлена/удалена |
| **по регулярке в изменениях** | `git log -G"regex"` |
| **по файлу/пути** | `git log -- path/to/file` |
| **диапазон коммитов** | `git log A..B` (есть в B, нет в A) |
| **исключить merge-коммиты** | `git log --no-merges` |

> [!info] `--` отделяет пути от ревизий
> Если имя файла совпадает с именем ветки, Git запутается. `git log -- <путь>` явно говорит «дальше идут пути», убирая двусмысленность. То же правило, что в [`git diff`](diff.md)/[`git add`](add.md).

## 🎨 Кастомный формат — `--pretty`

```bash
git log --pretty=format:"%h %ad | %s [%an]" --date=short
```

```text
a1b2c3d 2026-06-13 | VCS/Git: add git log note [WhiteK0T]
```

Частые плейсхолдеры `--pretty=format`:

| Плейсхолдер | Значение |
| :--- | :--- |
| **`%H`** / **`%h`** | полный / короткий хеш |
| **`%an`** / **`%ae`** | имя / e-mail автора |
| **`%ad`** / **`%ar`** | дата автора / относительная («3 days ago») |
| **`%cn`** / **`%cd`** | имя коммиттера / дата коммита |
| **`%s`** | тема (subject) сообщения |
| **`%d`** | декорации (ветки/теги) |
| **`%C(...)`** / **`%Creset`** | цвет / сброс цвета |

Готовые пресеты: `--pretty=oneline`, `--pretty=short`, `--pretty=full`, `--pretty=fuller` (показывает обе даты — author и committer).

## 🔍 Кто и когда менял строки

- **`git log -p <файл>`** — эволюция файла с полными диффами.
- **`git log -L 10,20:file.py`** — история **конкретных строк** 10–20 файла.
- **`git blame <файл>`** — для каждой строки: кто, в каком коммите и когда её ввёл (это отдельная команда, не `log`).
- **`git show <commit>`** — один коммит целиком (сообщение + diff против родителя).

## 🧭 git reflog — «журнал перемещений HEAD»

Отдельная, но родственная команда. `git reflog` показывает **локальную** историю того, куда двигался `HEAD` (коммиты, переключения веток, `reset`, `rebase`, `amend`) — даже на коммиты, которых уже нет в `git log`.

```bash
git reflog                       # список последних позиций HEAD
git reset --hard HEAD@{2}        # вернуться к состоянию «2 шага назад»
```

> [!tip] Спасательный круг
> «Потерял» коммиты после неудачного `reset --hard`/`rebase`? Они почти всегда ещё в `reflog` — найди хеш и восстановись (`git checkout <hash>` / `git reset --hard <hash>`). Reflog локальный и со временем чистится (по умолчанию ~90 дней).

## ⚡ Удобные алиасы

```bash
git config --global alias.lg "log --oneline --graph --decorate --all"
git config --global alias.last "log -1 HEAD --stat"
```

Теперь `git lg` рисует граф, `git last` — последний коммит со статистикой.

## 🩺 Частые вопросы

- **Лог «завис» и не отдаёт терминал** — это пейджер `less`, выйти — клавиша `q`. Для разового вывода без пейджера: `git --no-pager log -5`.
- **Не вижу другие ветки** — добавь `--all` (по умолчанию показывается только текущая ветка).
- **Где мои «пропавшие» коммиты** — `git reflog` (см. выше).
- **Нужен лог для скрипта** — фиксированный формат через `--pretty=format:"..."`, парсинг стабилен.

## 💎 Резюме

- Компактно с графом: `git log --oneline --graph --decorate --all`.
- Фильтры: `--author`, `--grep`, `--since/--until`, `-S` (по содержимому), `-- <путь>`.
- Детали коммита: `git show <commit>`; история строк: `git log -L`; авторство строк: `git blame`.
- Свой формат: `--pretty=format:"%h %ad %s"`; «спасение» потерянного: `git reflog`.
