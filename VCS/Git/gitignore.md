# 🌿 .gitignore — игнорирование файлов

`.gitignore` — текстовый файл с **шаблонами путей**, которые Git не должен отслеживать: артефакты сборки, зависимости, логи, секреты, временные файлы IDE. Игнорируемые файлы не попадают в [`git status`](status.md) как untracked и не добавляются [`git add`](add.md).

```text
# пример .gitignore
node_modules/
*.log
.env
/build
```

## 🧬 Что и как игнорируется

`.gitignore` кладут в корень репозитория (и/или в подкаталоги). Правила применяются к путям **относительно расположения** файла `.gitignore`.

> [!warning] gitignore не действует на УЖЕ отслеживаемые файлы
> `.gitignore` влияет только на **неотслеживаемые** файлы. Если файл уже закоммичен, добавление его в `.gitignore` **не** перестанет его отслеживать — нужно сначала убрать из индекса: `git rm --cached <файл>` (см. ниже).

## 📝 Синтаксис шаблонов

| Шаблон | Значение |
| :--- | :--- |
| `file.log` | файл `file.log` в любом каталоге |
| `*.log` | все файлы с расширением `.log` |
| `build/` | каталог `build` (слеш в конце = только каталог) |
| `/build` | `build` только в **корне** репозитория (ведущий слеш = от корня) |
| `doc/*.txt` | `.txt` прямо в `doc/`, но не в подкаталогах |
| `doc/**/*.txt` | `.txt` в `doc/` на любой глубине (`**` = любое число каталогов) |
| `temp?` | один любой символ (`temp1`, `tempA`) |
| `[abc].o` | один из символов в скобках |
| `# комментарий` | строка-комментарий (игнорируется) |
| `!important.log` | **исключение**: не игнорировать, даже если попадает под правило выше |

### Отрицание `!` — вернуть файл из-под игнора

```text
*.log              # игнорировать все логи
!keep.log          # но keep.log отслеживать
```

> [!note] Нюанс с `!` и каталогами
> Нельзя «разыгнорить» файл, если **родительский каталог** целиком исключён — Git даже не заходит внутрь. Сначала разрешают каталог, потом исключают лишнее:
> ```text
> logs/*
> !logs/important/
> !logs/important/*.log
> ```

## 🗂️ Уровни игнорирования

| Файл | Область | Коммитится? |
| :--- | :--- | :--- |
| **`.gitignore`** (в репозитории) | проектные правила, общие для всех | **да** |
| **`.git/info/exclude`** | локальные правила только для тебя | нет (внутри `.git`) |
| **глобальный** (`core.excludesFile`) | для всех твоих репозиториев | нет (в `~`) |

```bash
# глобальный игнор (напр. файлы ОС/IDE: .DS_Store, .idea/)
git config --global core.excludesFile ~/.gitignore_global
```

> [!tip] Что куда класть
> - **`.gitignore`** — то, что должно игнорироваться у **всех** (артефакты сборки проекта).
> - **`.git/info/exclude`** — личные временные файлы **в этом** репозитории, которые не навязываешь команде.
> - **глобальный** — мусор твоей среды (`.DS_Store`, `*.swp`, каталоги IDE), не относящийся к проектам.

## 🧹 Перестать отслеживать уже закоммиченное

Если файл уже в репозитории, а должен игнорироваться:

```bash
echo "secrets.env" >> .gitignore
git rm --cached secrets.env          # убрать из индекса, файл на диске остаётся
git commit -m "Перестать отслеживать secrets.env"
# для каталога:
git rm -r --cached node_modules/
```

> [!danger] Игнор ≠ удаление из истории
> Если секрет **уже попал в коммит**, `.gitignore` и `git rm --cached` уберут его только из будущих коммитов — в **истории он остаётся**. Для полного удаления нужен `git filter-repo` (или BFG) + ротация секрета. Считай засвеченный секрет скомпрометированным.

## 🔍 Отладка правил

```bash
git status --ignored                 # показать игнорируемые файлы
git check-ignore -v <путь>           # какое правило и в каком файле сработало
git add -f <файл>                    # принудительно добавить вопреки игнору
```

`git check-ignore -v` — главный инструмент, когда «файл игнорируется, а не должен» (или наоборот): покажет конкретную строку конкретного `.gitignore`.

## 🌐 Готовые шаблоны

Не пиши с нуля — бери готовые под язык/стек:

- **github/gitignore** — официальная коллекция ([github.com/github/gitignore](https://github.com/github/gitignore)).
- **gitignore.io** — генератор по технологиям ([toptal.com/developers/gitignore](https://www.toptal.com/developers/gitignore)).
- При `git init`/создании репозитория на GitHub можно выбрать шаблон сразу.

## 🩺 Частые вопросы

- **Добавил в `.gitignore`, а файл всё равно в `status`** — он уже отслеживается: `git rm --cached <файл>`.
- **Правило не срабатывает** — проверь `git check-ignore -v <путь>`; частая причина — ведущий `/` или слеш каталога.
- **Нужно закоммитить пустой каталог** — Git не хранит пустые каталоги; кладут файл-заглушку `.gitkeep`.
- **Не могу разыгнорить файл через `!`** — исключён родительский каталог; разреши его, потом уточняй.
- **Хочу один игнор на все репо** — `core.excludesFile` (глобальный), а не копировать `.gitignore` повсюду.

## 💎 Резюме

- `.gitignore` — шаблоны путей, которые Git не отслеживает; кладётся в репозиторий и коммитится.
- Синтаксис: `*` `**` `?` `[]`, `/` (от корня), `dir/` (каталог), `!` (исключение).
- Действует только на **неотслеживаемые**: уже закоммиченное — `git rm --cached`.
- Уровни: проектный `.gitignore`, локальный `.git/info/exclude`, глобальный `core.excludesFile`.
- Отладка: `git check-ignore -v`; принудительно: `git add -f`; шаблоны: github/gitignore.
