# 🗂️ FileBrowser Quantum — self-hosted веб-менеджер файлов

**FileBrowser Quantum** ([github.com/gtsteffaniak/filebrowser](https://github.com/gtsteffaniak/filebrowser)) — самохостируемый **веб-интерфейс для управления файлами** на твоём сервере: заходишь браузером и работаешь с папками/файлами как в облаке, но у себя. Один лёгкий бинарь на **Go** (или Docker-образ ~180 МБ с ffmpeg), мгновенный поиск через SQLite-индекс, миниатюры, гибкий шаринг, корпоративная аутентификация. Apache-2.0, **~7.5k★**, очень активный, релиз **v1.5.0-stable**.

> [!warning] Это ФОРК, а не «тот самый» filebrowser — не перепутай
> Репозиторий называется `gtsteffaniak/filebrowser`, но это **масштабный форк** оригинального [filebrowser/filebrowser](https://github.com/filebrowser/filebrowser) (~35k★, тоже жив). Отличия важны:
> - Конфиг другой (**`config.yaml`**), миграция с оригинала **не «в один клик»** — это фактически другое приложение.
> - **Удалены shell-команды** (в оригинале был запуск команд — известный источник RCE-риска). Здесь их нет — это **плюс к безопасности**.
> - Форк ведёт **один разработчик** (хоть и активно/популярно). Оригинал — отдельный проект со своей командой.
> Если ищешь «классический filebrowser» из старых гайдов — это **не он**, а его переосмысление.

> [!info] Что реально умеет (по README, подтверждено)
> - **Несколько источников** сразу + правила include/exclude.
> - **Аутентификация**: OIDC, LDAP, JWT, пароль + **2FA**, proxy-auth. SSO — есть (в оригинале не было).
> - **SQLite-индексация** → поиск в реальном времени по мере ввода, фильтры по размеру файлов/папок, live-обновление UI.
> - **Миниатюры**: office, видео, обложки альбомов, **3D-модели**; размеры папок показываются.
> - **Шаринг**: срок жизни ссылки, доступ (включая анонимный), права на просмотр/редактирование/загрузку, темы.
> - **Директорий-уровневый контроль доступа** (на пользователя/группу), **API-токены** + Swagger на `/swagger`.

> [!caution] Чего пост не говорит — ограничения и рамки
> - **Это файловый МЕНЕДЖЕР существующих файлов, а не облако-синхронизация.** Нет клиента синхронизации, **нет S3/FTP-бэкендов** (WebDAV есть), **нет мобильного приложения**. Он показывает **каталог на диске** через веб — не замена Nextcloud/Google Drive по функционалу «облака».
> - **Часть «продвинутых» фич ещё в разработке** (🚧 по их же таблице): квоты, корзина (trash), теги, журнал активности, метрики, кастомные задачи, event-уведомления. «Расширенное управление» из поста местами = планы, а не готовое.
> - **Экспозиция файловой системы в веб = поверхность атаки.** Хорошо, что shell убрали, но всё равно: только за **HTTPS/reverse-proxy**, сильная аутентификация, не выставляй голым в интернет. Директорий-контроль и 2FA помогают.
> - **Не content-aware поиск** (ищет по именам/метаданным, не по содержимому документов).

## 🖥️ Применимость на системах владельца

| Система | Как использовать |
| :--- | :--- |
| **Gentoo / Debian-Ubuntu / Arch** | ✅ идеальный кейс: один Go-бинарь или Docker на домашнем сервере/десктопе; веб-доступ к твоим файлам, поиск, шаринг. Спрятать за [nginx](../Network/WebServers/nginx/nginx%20%E2%80%94%20%D0%B2%D0%B5%D0%B1-%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80%20%D0%B8%20reverse-proxy%20%E2%80%94%20%D1%83%D1%81%D1%82%D0%B0%D0%BD%D0%BE%D0%B2%D0%BA%D0%B0%2C%20%D0%BA%D0%BE%D0%BD%D1%84%D0%B8%D0%B3%2C%20server-location%2C%20%D1%81%D1%82%D0%B0%D1%82%D0%B8%D0%BA%D0%B0%2C%20%D0%BF%D1%80%D0%BE%D0%BA%D1%81%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5%2C%20TLS%2C%20%D0%BA%D1%8D%D1%88%20%28%D0%BF%D0%BE%D0%B4%D1%80%D0%BE%D0%B1%D0%BD%D1%8B%D0%B9%20%D1%80%D0%B0%D0%B7%D0%B1%D0%BE%D1%80%29.md) + TLS |
| **Windows / macOS** | ✅ есть standalone-бинарь и под них |
| **Entware / RT-AX56U** | ⚠️ **на грани**: min-требование **512 МБ RAM** = ровно весь объём роутера (плюс уже занятый ОС) → Quantum будет тесно. Go-бинарь под armv7 собрать можно, но для роутера с USB-диском **реалистичнее лёгкий оригинальный filebrowser** (образ ~31 МБ, 128 МБ RAM) или просто [Samba](../Network/Samba.md). Quantum лучше держать на десктопе/домашнем сервере |

> [!tip] Тебе это в тему (домашний сервер + USB-диск на роутере)
> У тебя ASUS RT-AX56U с USB-диском. Веб-доступ к этим файлам удобно дать именно так — но **Quantum ставь на десктоп/домашний сервер**, а не на сам роутер (тесно по RAM). Для доступа к файлам роутера у тебя уже есть [Samba](../Network/Samba.md); FileBrowser Quantum — про удобный веб-UI с поиском и шарингом поверх файлов на нормальной машине.
