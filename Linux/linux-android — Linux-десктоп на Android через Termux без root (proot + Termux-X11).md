# 📱➡️🐧 linux-android — Linux-десктоп на Android через Termux без root (proot + Termux-X11)

[**linux-android**](https://github.com/mayukh4/linux-android) (mayukh4, **Shell/Bash**, ~1958★, 163 форка) — набор скриптов, который **автоматически разворачивает полноценное Linux-окружение с графикой прямо на Android-смартфоне** — без root и без ПК, силами одного только **Termux**. Идея: дать старому телефону вторую жизнь как **лёгкий Linux-десктоп**, **dev-сервер по SSH** или **сервер умного дома (Home Assistant)**. Вся ценность — в **автоматизации**: руками этот стек (Termux + proot-distro + Termux-X11 + GPU-драйверы) собирать долго и муторно, скрипт делает за один прогон.

## ⚙️ Как это устроено (важно понимать)

- **proot, а не chroot/root.** Дистрибутив запускается в **пользовательском пространстве** через `proot` — он перехватывает системные вызовы и подменяет пути, имитируя «настоящий» Linux **без прав root**. Это и плюс (ничего не надо разблокировать/рутовать), и минус (**накладные расходы** на перехват syscalls → медленнее нативного).
- **Графика — Termux-X11, а не VNC.** GUI рисуется в приложении **Termux-X11** (X-сервер под Android), без возни с VNC-клиентом. Ускорение через **Mesa/Zink** поверх **Vulkan**, на Qualcomm Snapdragon — драйвер **Turnip** (Adreno) для аппаратного GPU.
- **Android-ядро остаётся снизу.** Это **не** замена Android и не «настоящая» установка дистрибутива на железо — Linux работает **поверх** Android. Перезагрузка телефона = выход из окружения.

## ✨ Что ставит и умеет

- **Выбор окружения рабочего стола:** **XFCE4** (по умолчанию), **LXQt**, **MATE** или **KDE Plasma**.
- **Софт из коробки:** Firefox, VLC, Git, Python 3, **OpenSSH-сервер** (заходить на телефон по SSH), PulseAudio (звук), опционально **Wine** для x86-Windows-приложений.
- **Два сценария — два скрипта:**
  - `termux-linux-setup.sh` — **графический Linux-десктоп**.
  - `setup-homeassistant.sh` — **Home Assistant** в контейнере **Ubuntu через proot-distro** (умный дом).
- **Применения:** учебный Linux-десктоп, SSH dev-сервер, Python-станция, медиа-сервер, домашняя автоматизация, сетевой мониторинг.

> [!warning] Требования к железу (arm64)
> - **Только 64-битный ARM (arm64/aarch64)** — старые 32-битные телефоны не подойдут.
> - **ОЗУ:** от **3 ГБ** (для **KDE Plasma — 4 ГБ+**).
> - **Память:** **5–10 ГБ** свободно.
> - Полноценное GPU-ускорение — на **Qualcomm Snapdragon** (Turnip); на Mali/других чипах графика пойдёт, но скорее через программный рендер.

## 📦 Установка (на Android, через Termux)

> [!danger] Ставь Termux из F-Droid, не из Google Play
> Версия Termux в **Google Play устарела и заброшена** — пакеты будут ломаться. Бери Termux из **[F-Droid](https://f-droid.org/packages/com.termux/)** или из релизов GitHub. Это не каприз, а частая причина «у меня не работает».

```bash
# 1) Десктоп — скачать, дать права, запустить
curl -O https://raw.githubusercontent.com/mayukh4/linux-android/main/termux-linux-setup.sh
chmod +x termux-linux-setup.sh
bash termux-linux-setup.sh
# дальше: запуск окружения
bash ~/start-linux.sh        # затем открыть приложение Termux-X11 → увидеть рабочий стол
```

```bash
# 2) Home Assistant (умный дом) — отдельный скрипт
curl -O https://raw.githubusercontent.com/mayukh4/linux-android/main/setup-homeassistant.sh
bash setup-homeassistant.sh
```

> [!caution] Сначала прочитай скрипт, потом запускай
> Это `bash <скачанный_скрипт>` — он ставит пакеты, тянет дистрибутив и правит окружение Termux. Перед запуском **открой `.sh` и просмотри**, что он делает (`less termux-linux-setup.sh`). Запускать чужие установочные скрипты вслепую — плохая привычка; здесь хотя бы скачивание и запуск разнесены (можно проверить).

## 🧪 Факты против хайпа

> [!note] «Старый смартфон → Linux-десктоп» — с оговорками
> - **Это proot-окружение, а не нативный Linux.** Производительность ниже из-за перехвата syscalls; для тяжёлых задач/компиляции телефон будет греться и тормозить.
> - **Wine x86 на arm64** работает через эмуляцию (box64-подобный слой) — медленно и не для всего; не жди игр/тяжёлых .exe.
> - **Не новая технология, а автоматизация.** Под капотом — известный стек **Termux + proot-distro + Termux-X11**; скрипт просто избавляет от ручной настройки.
> - **Лицензии нет.** В репозитории **не указана лицензия** → формально это «все права защищены»: код открыт для просмотра, но условия повторного использования/форка юридически не определены. Для личного эксперимента ок; в проект с чёткой лицензией — с осторожностью.
> - **Телефон как 24/7-сервер:** учитывай **деградацию батареи** от постоянной зарядки и нагрев. Для умного дома лучше телефон с обходом контроллера заряда или вынести в прохладное место.

## 💻 Как это ложится на твои системы

Установка идёт **на Android-телефон**, а не на десктопные дистрибутивы — но твои системы рядом полезны:

| Система | Роль относительно этого проекта |
| :--- | :--- |
| **Gentoo / Debian / Arch** (десктопы) | Заходить на телефон-сервер **по SSH** (скрипт ставит OpenSSH), управлять им как обычным Linux-хостом; для зеркала экрана — `scrcpy` (`emerge`/`apt`/`pacman install scrcpy`) |
| **Entware / RT-AX56U** | Альтернатива для лёгкого **always-on** домашнего сервиса: на роутере уже крутятся службы 24/7 без деградации батареи. Home Assistant на роутер не влезет (нужен полноценный Linux), но для простых задач (DNS, торрент, мониторинг) Entware надёжнее «телефона на зарядке» |

> [!tip] Когда это реально полезно
> Есть **лежащий без дела Android (arm64, 3 ГБ+)** и хочется бесплатный учебный Linux-десктоп, карманный SSH/Python-стенд или пробный Home Assistant — без покупки Raspberry Pi. Для постоянного прод-сервера лучше выделенное железо (Pi/мини-ПК/роутер), а телефон — для экспериментов и обучения.

## 💡 Кому полезно

- **Энтузиастам/студентам:** потрогать Linux-десктоп и SSH-сервер на том, что уже есть в ящике.
- **Умный дом «на попробовать»:** поднять Home Assistant без отдельного железа.
- **Не** замена нативному Linux/серверу и **не** про производительность — это repurpose старого телефона.
