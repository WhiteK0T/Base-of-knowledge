# 🐧📖 Linux-Guide (mikeroyal) — большой README-каталог по Linux

[**Linux-Guide**](https://github.com/mikeroyal/Linux-Guide) (*mikeroyal*, ~**1.2k★**, 96 форков) — объёмный **README-справочник по экосистеме Linux**: дистрибутивы (Ubuntu, Debian, Fedora, Kali…), терминал и основы, пакетные менеджеры, файловые системы, десктоп-окружения и оконные менеджеры, безопасность и харденинг (SELinux, firewalld, iptables, WireGuard), графика (AMD/NVIDIA/Intel ARC, Vulkan), облако/серверы/виртуализация/контейнеры, вендоры железа, игры (Steam/Steam Deck). Пост из «Библиотеки программиста» назвал его «одним из самых полных опенсорс-гайдов по Linux». Полнота охвата действительно большая — но есть важные оговорки ниже.

> [!info] Что это по формату
> Это **не пошаговый учебник**, а огромный **список ссылок с короткими описаниями** (стиль *awesome-list*): по каждой теме — что это + линки на инструменты/официальные доки. Один длинный README (плюс родственные под-гайды автора). Хорош как **карта «что вообще есть»** в мире Linux, а не как «прочитал и научился».

## 🧪 Факты против хайпа

> [!warning] Не обновляется с января 2024 — «актуальным» его назвать нельзя
> Последний коммит — **4 января 2024** (репозиторий создан в 2020). На июль 2026 это **~2.5 года без обновлений**. Для быстро меняющихся тем (релизы дистрибутивов, облачные инструменты, версии пакетов, security-рекомендации) часть ссылок и советов **устарела**. Как отправная точка по «вечнозелёным» понятиям (что такое SELinux, зачем WireGuard) — ок; как источник **свежих** инструкций — сверяйся с официальной документацией и вики дистрибутива.

> [!note] «Rust» и «нет лицензии» — не то, чем кажется
> - GitHub помечает язык репо как **Rust**, но это **артефакт детектора** (в репо есть немного кода/сниппетов) — это **не** Rust-проект, а Markdown-справочник.
> - **Лицензия отсутствует** (`License: None`). Формально это значит **«все права защищены»**: читать и учиться — пожалуйста, но копировать/переиспользовать содержимое как «свободное» нельзя без разрешения автора.

> [!caution] Это каталог, а не «выучить Linux» — и он англо/десктоп-центричен
> - Для **обучения с нуля** голый список ссылок слабоват — лучше структурированные курсы/книги (например [Linux Journey](https://linuxjourney.com/), [The Linux Command Line](https://linuxcommand.org/tlcl.php), [ArchWiki](https://wiki.archlinux.org/) как эталон практических статей, [Gentoo Handbook](https://wiki.gentoo.org/wiki/Handbook)).
> - Уклон — **десктоп и «железо/графика/игры»** (Steam Deck, вендоры GPU), англоязычный. Это **срез автора**, а не нейтральная энциклопедия.

> [!tip] У автора — целое семейство таких «Guide»
> mikeroyal ведёт серию однотипных мега-README: **Windows-Guide**, **macOS-Guide**, **Kali-Linux-Guide**, гайды по Cloud/Security/Photogrammetry и др. Стиль и **проблема с актуальностью** у них общие — удобно для широкого обзора темы, но проверяй даты последних коммитов.

## 💡 Кому полезно

- **Как обзорная карта** экосистемы Linux: быстро увидеть, какие бывают темы/инструменты и куда копать (harden, WireGuard, контейнеры, графика).
- **Не** учебник и **не** актуальный (2024) справочник инструкций: для практики бери официальные вики/доки, для обучения — курсы/книги.
- Тебе под Gentoo/Entware он покроет далеко не всё — держи рядом свои заметки по [пакетным менеджерам](Package-Manager/%D0%A1%D1%80%D0%B0%D0%B2%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5%20%D0%BA%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%20%D0%BC%D0%B5%D0%BD%D0%B5%D0%B4%D0%B6%D0%B5%D1%80%D0%BE%D0%B2%20%D0%BF%D0%B0%D0%BA%D0%B5%D1%82%D0%BE%D0%B2.md) и Gentoo.
