# 🆘 SysRq — как привести в чувство «зависший» Linux

> [!info] О заметке
> Переработанный конспект старой статьи с `it-simple.ru` (исходник уже недоступен). Команды и комбинации сверены с актуальной [документацией ядра](https://www.kernel.org/doc/html/latest/admin-guide/sysrq.html).

## 🧊 Не всё то Windows, что висит

Картинка на экране замерла, окна не двигаются, не помогает даже **Ctrl + Alt + Backspace** — клавиатура будто не работает. Спинной мозг с виндовым опытом подсказывает давить **Reset**.

**Не делайте этого.** Аппаратный сброс рискует повредить файловую систему (данные пишутся не мгновенно, кэш не сброшен на диск). Linux чаще всего только *выглядит* зависшим: само ядро живо и продолжает обслуживать прерывания — намертво встаёт обычно лишь графика или отдельный процесс. Настоящий `kernel panic` — редкость, и обычно это симптом неисправного железа.

## ⌨️ Magic SysRq — что это

**PrtScr / SysRq** — это **одна** физическая клавиша (в Windows-мире — «снимок экрана»). В Linux она «низкоуровневая»: ядро ловит её напрямую, без участия драйверов клавиатуры, X-сервера и пользовательских процессов. Поэтому Magic SysRq работает даже когда всё остальное намертво висит.

Базовая комбинация — **Alt + SysRq + `‹буква›`**. Удерживая `Alt + SysRq`, по очереди нажимайте буквы.

> [!warning] Одна клавиша, не две
> `SysRq` и `PrtScr` — это один и тот же физический клавиш. Писать «Alt + PrtScr + SysRq» некорректно — нажимать нужно **Alt + SysRq + буква**.

## 💊 REISUB — безопасная перезагрузка

Если система не отвечает, аккуратная перезагрузка без потери данных — удерживая **Alt + SysRq**, нажать по очереди:

```
R  E  I  S  U  B
```

| Клавиша | Что делает |
| :---: | :--- |
| **R** | un**R**aw — забирает клавиатуру у графики, возвращает ввод напрямую ядру |
| **E** | SIGT**E**RM — мягко завершает все процессы (кроме `init`/PID 1) |
| **I** | SIGK**I**LL — жёстко убивает оставшиеся процессы |
| **S** | **S**ync — сбрасывает дисковый кэш на носители (защита данных) |
| **U** | **U**mount — перемонтирует ФС в режим «только чтение» |
| **B** | re**B**oot — немедленная перезагрузка |

> [!tip] Как запомнить
> **R**eboot **E**ven **I**f **S**ystem **U**tterly **B**roken (или мнемоника «Raising Elephants Is So Utterly Boring»). Если вместо перезагрузки нужно **выключение** — в конце вместо `B` нажмите `O` (**O** = power **O**ff): REISU**O**.
>
> **Делайте паузу** между нажатиями (особенно `E`→`I` и перед `S`/`U`), чтобы предыдущая операция успела отработать. Из «зависшей» графики — паузы побольше.

## 📋 Полезные комбинации

| Alt + SysRq + | Действие |
| :---: | :--- |
| **R** | un**R**aw — выключить RAW-режим клавиатуры (вернуть ввод вне X) |
| **K** | Secure Access **K**ey — убить все программы текущей виртуальной консоли |
| **E** | SIGT**E**RM всем процессам, кроме `init` |
| **I** | SIGK**I**LL всем процессам, кроме `init` |
| **S** | **S**ync — синхронизировать диски |
| **U** | **U**mount — перемонтировать ФС в read-only |
| **B** | re**B**oot — немедленная перезагрузка |
| **O** | power **O**ff — немедленное выключение |
| **F** | вызвать **OOM-killer** (убить самый прожорливый по памяти процесс) |
| **P** | вывести в консоль регистры и указатели (**P**rocessor) |
| **T** | список всех задач/процессов (**T**asks) |
| **M** | информация о памяти (**M**emory) |
| **L** | backtrace по всем CPU |
| **J** | разморозить замороженные ФС (un**J**am, после `fsfreeze`) |
| **C** | вызвать **C**rash/kexec-дамп (для отладки) |
| **\[0–9\]** | задать уровень логирования ядра в консоль |

## 🌐 То же самое по SSH

Если до машины есть сетевой доступ, всё это можно сделать через файл-триггер `/proc/sysrq-trigger` (запись делается **от root**):

```bash
# мгновенный sync + reboot (от root)
echo s > /proc/sysrq-trigger   # сбросить кэш на диск
echo u > /proc/sysrq-trigger   # перемонтировать в read-only
echo b > /proc/sysrq-trigger   # перезагрузка
```

> [!danger] Это «жёсткий» reboot
> `echo b` перезагружает немедленно, минуя штатное завершение служб. Перед ним обязательно `s` и `u`, иначе рискуете тем же повреждением ФС, что и от кнопки Reset.

## 🔧 Включение SysRq

В большинстве дистрибутивов функция включена частично или выключена. Проверка текущего значения:

```bash
sysctl kernel.sysrq        # или: cat /proc/sys/kernel/sysrq
```

`kernel.sysrq` — это **битовая маска**, а не просто 0/1:

| Значение | Что разрешено |
| :---: | :--- |
| `0` | всё выключено |
| `1` | разрешены **все** функции |
| `2` | управление уровнем логирования |
| `4` | управление клавиатурой (R, K) |
| `8` | отладочные дампы (P, T, M…) |
| `16` | sync (S) |
| `32` | remount read-only (U) |
| `64` | сигналы процессам (E, I, K) |
| `128` | reboot / poweroff (B, O) |

Биты суммируются (например `176 = 16+32+128` — разрешает только S, U, B). Для «всё включено» проще задать `1`.

**Включить прямо сейчас (до перезагрузки):**

```bash
sudo sysctl -w kernel.sysrq=1
# эквивалент через /proc (НЕ через `sudo echo` — редирект выполнит непривилегированный shell):
echo 1 | sudo tee /proc/sys/kernel/sysrq
```

> [!warning] Классическая ошибка
> `sudo echo 1 > /proc/sys/kernel/sysrq` **не сработает**: `sudo` поднимает права только у `echo`, а перенаправление `>` выполняет ваш обычный shell — и упирается в «Permission denied». Используйте `tee` (как выше) или `sysctl -w`.

**Сделать постоянным** (переживает перезагрузку) — создать файл в `/etc/sysctl.d/`:

```bash
echo 'kernel.sysrq = 1' | sudo tee /etc/sysctl.d/99-sysrq.conf
sudo sysctl --system        # применить, не перезагружаясь
```

Способ с `/etc/sysctl.d/*.conf` + `sysctl --system` работает одинаково на всех целевых системах:

| Система | Применение настройки |
| :--- | :--- |
| **Gentoo** (OpenRC) | `/etc/sysctl.d/99-sysrq.conf`; применяет служба `sysctl` (`rc-service sysctl restart`) либо `sysctl --system`. Старый `/etc/sysctl.conf` тоже читается |
| **Debian / Ubuntu** | systemd-сервис `systemd-sysctl` читает `/etc/sysctl.d/*.conf` при загрузке; `sudo sysctl --system` применяет сразу |
| **Arch** | то же, что Debian — `/etc/sysctl.d/*.conf` + `systemd-sysctl` |

> [!note] Поддерживает ли ядро SysRq
> Нужен собранный с `CONFIG_MAGIC_SYSRQ=y` kernel. Проверка:
> ```bash
> # Debian/Ubuntu/Arch — конфиг ядра лежит в /boot:
> grep -i sysrq /boot/config-$(uname -r)
> # Gentoo (если включён CONFIG_IKCONFIG_PROC):
> zcat /proc/config.gz | grep -i sysrq
> ```
> Ответ должен содержать `CONFIG_MAGIC_SYSRQ=y`.
