# 🪟🐧 WinPodX — Windows-приложения как нативные окна Linux (контейнерная Windows-ВМ + RemoteApp)

[**WinPodX**](https://github.com/kernalix7/winpodx) — запускает **приложения Windows как отдельные нативные окна Linux**: кликнул иконку — открылся Word, как будто это родное Linux-приложение (своё окно, иконка, ассоциации файлов, общий буфер обмена/принтер/звук). Под капотом — **настоящая Windows в контейнере** (`dockur/windows` поверх **KVM**), а приложения прокидываются через **FreeRDP RemoteApp (RAIL)**. Python, лицензия **MIT**, ~1.1k★, статус — **бета (v0.7.3,** 20 июня 2026**)**.

> [!warning] Факты против поста: это НЕ «без виртуалки» и НЕ Wine
> Telegram-пост пишет, что приложения «работают в изолированных контейнерах **без установки полноценной ВМ**». Это вводит в заблуждение:
> - Внутри крутится **реальная Windows в KVM-виртуалке** (просто упакованной в контейнер через `dockur/windows`). Это **не** эмуляция системных вызовов как у **Wine** и **не** «лёгкий контейнер».
> - Поэтому нужны **аппаратная виртуализация (VT-x/AMD-V), 8+ ГБ RAM (лучше 12+), ~30 ГБ диска**, `/dev/kvm` и пользователь в группе `kvm`.
> - Первый запуск — **5–10 минут** (скачивание ISO, Sysprep, OEM).
> Плюс по сравнению с Wine — **настоящая совместимость** (это реальная Windows). Минус — куда тяжелее по ресурсам.

> [!info] Чем отличается от соседей
> - **vs Wine** — Wine эмулирует Windows API без самой Windows (легко, но совместимость не 100%). WinPodX гоняет реальную Windows → совместимость выше, ресурсов больше.
> - **vs WinApps** — идея та же (RemoteApp-окна), но WinApps использует **классическую ВМ** (libvirt/QEMU или Docker), а WinPodX — **контейнерную Windows** (`dockur/windows`) + многосессионный RDP (`rdprrap`) и обратные ассоциации (открывать файлы C:-диска Linux-приложениями).

## ✨ Возможности

- **Seamless-окна (RemoteApp/RAIL):** каждое Windows-приложение — отдельное окно Linux с реальной иконкой, закрепляемое в таскбар (не весь рабочий стол).
- **Двусторонние ассоциации файлов:** Windows-приложения в Linux + (через Rust-шимы по slug) Linux-приложения в меню «Открыть с помощью» внутри Windows.
- **Общие ресурсы:** буфер обмена, принтеры, звук, проброс файлов.
- **Многосессионный RDP**, idle-автостоп (гасит ВМ при простое), проброс мультитача.
- **Конфиг:** TOML + Docker Compose; почти **без внешних Python-зависимостей** (stdlib на 3.11+, `tomli` как fallback на 3.9/3.10).

> [!caution] «Bare-metal disguise» (сокрытие ВМ) — по желанию, по умолчанию выкл.
> Есть опция маскировки виртуалки от приложений, которые отказываются работать в ВМ (некоторые античиты/DRM). По умолчанию **выключена**. Учти: обход VM-детекта в играх с античитом может привести к **бану**, а обход DRM — нарушать лицензию. Включай осознанно и только там, где это законно.

## 📦 Установка

Нужны: CPU x86_64/aarch64 с VT-x/AMD-V, модуль **KVM** в ядре, `/dev/kvm`, юзер в группе `kvm`, **Podman ≥4** (или Docker), **FreeRDP**, **Python 3.9+**.

```bash
# быстрый старт (прочитай скрипт перед запуском — он от третьего лица)
curl -fsSL https://raw.githubusercontent.com/kernalix7/winpodx/main/install.sh | bash
# при установке из пакета — инициализация:
winpodx setup
# запуск приложения / полноценного рабочего стола (RDP fullscreen — opt-in)
winpodx app run desktop
```

| Система владельца | Применимость |
| :--- | :--- |
| **Gentoo** (основная) | в официальном списке (openSUSE/Fedora/Debian/Ubuntu/RHEL/Arch/NixOS) Gentoo **нет** → ставь через **AppImage** или `install.sh`; предварительно собери ядро с **KVM** (`CONFIG_KVM`/`CONFIG_KVM_INTEL`/`AMD`), `emerge app-containers/podman` (или docker) и `net-misc/freerdp`, добавь себя в группу `kvm` |
| **Debian / Ubuntu** | поддержано (apt-пакет/скрипт). Поставить `podman`/`docker.io` + `freerdp2-x11`, включить KVM, группа `kvm` |
| **Arch** (план с июня 2026) | поддержано; `pacman -S podman freerdp`, модуль KVM, группа `kvm` |
| **Entware / RT-AX56U** | **N/A** — нет KVM/x86, 256 МБ flash. Категорически не для роутера |

> [!note] Лучше всего работает на хосте с аппаратной виртуализацией
> Если у тебя уже Linux как основная ОС и нужен 1–2 «упрямых» Windows-приложения (Office, специфичный софт) — это удобнее, чем держать целый Windows-десктоп в окне. Для разовых задач и лёгких приложений сначала попробуй **Wine/Proton** — дешевле по ресурсам.

## 💡 Кому полезно

- Сидишь на **Linux**, но нужно 1–2 Windows-приложения с **реальной совместимостью** (где Wine не тянет), и не хочется ловить целый рабочий стол Windows.
- Есть мощный хост (RAM/диск/VT-x) и нормально с тем, что внутри крутится настоящая Windows-ВМ.
- **Не** твой вариант, если железо слабое или нужен лёгкий запуск простого .exe — тогда Wine/Proton.
