# ✂️ sed — потоковый редактор текста (шпаргалка)

**sed** (*stream editor*) — построчно прогоняет текст через набор команд: заменяет, удаляет, вставляет строки **без открытия редактора**. Незаменим для массового рефакторинга, правки конфигов и логов, работает в пайпах с `grep`/`find`. Базовый синтаксис подстановки: `s/что/на_что/флаги`.

## 🔄 Замена (`s`) — главное

| Команда | Что делает |
| :--- | :--- |
| `sed 's/old/new/' file` | заменить **первое** вхождение в каждой строке (вывод в stdout) |
| `sed 's/old/new/g' file` | заменить **все** вхождения (`g` = global) |
| `sed 's/old/new/2' file` | заменить только **2-е** вхождение в строке |
| `sed 's/old/new/2g' file` | со 2-го и до конца строки |
| `sed 's/old/new/gI' file` | глобально, **без учёта регистра** (`I` — расширение GNU) |
| `sed -i 's/old-api/new-api/g' config.yaml` | заменить **прямо в файле** (`-i` = in-place) |
| `sed -i.bak 's/old/new/g' file` | in-place + сохранить **бэкап** `file.bak` |

> [!tip] Меняешь пути/URL — смени разделитель
> Разделитель в `s` не обязан быть `/`. Если в тексте есть слэши (пути, URL) — возьми другой символ, чтобы не экранировать: `sed 's#/old/path#/new/path#g'` или `sed 's|http://a|https://b|g'`.

**Спецсимволы в замене:**

| Символ | Значение |
| :--- | :--- |
| `&` | весь найденный фрагмент (`s/err/[&]/` → `err` станет `[err]`) |
| `\1 \2 …` | обратные ссылки на группы `\(...\)` (в GNU с `-E` — просто `(...)`) |
| `\L \U \l \u` | перевод замены в нижний/верхний регистр (GNU): `s/.*/\U&/` → ВЕРХНИЙ |

```bash
# поменять местами две колонки через запятую (группы + backref)
sed -E 's/^([^,]+),([^,]+)$/\2,\1/' data.csv
```

## 🗑️ Удаление и вставка строк

| Команда | Что делает |
| :--- | :--- |
| `sed -i '/DEBUG/d' app.log` | удалить все строки, **содержащие** `DEBUG` |
| `sed -i '/^$/d' file` | удалить **пустые** строки |
| `sed -i '/^#/d' file` | удалить строки-**комментарии** (с `#`) |
| `sed -i '5d' file` | удалить **5-ю** строку |
| `sed -i '2,10d' file` | удалить строки **со 2-й по 10-ю** |
| `sed -i '/server {/a\    listen 443 ssl;' nginx.conf` | **вставить после** (`a`) совпадения |
| `sed -i '/server {/i\    # tls-блок' nginx.conf` | **вставить до** (`i`) совпадения |
| `sed -i '/old line/c\new line' file` | **заменить целиком** (`c`) строку с совпадением |

> [!note] `a` / `i` / `c` и GNU-сокращение
> Классический POSIX-синтаксис требует `a\` и текст с новой строки. **GNU sed** позволяет однострочно: `sed '/pat/a текст'`. На BSD/macOS чаще нужна форма с `\` и переносом — см. ниже о различиях.

## 🔍 Печать выборочных строк (`-n` + `p`)

`-n` глушит автопечать, `p` печатает явно — получается «как grep, но мощнее»:

| Команда | Что делает |
| :--- | :--- |
| `sed -n '5,10p' file` | напечатать строки **5–10** |
| `sed -n '/start/,/stop/p' file` | блок **от** строки со `start` **до** строки со `stop` |
| `sed -n '$p' file` | только **последняя** строка |
| `sed -n '1~2p' file` | каждая **нечётная** строка (GNU `первая~шаг`) |
| `grep -n foo file` ≈ `sed -n '/foo/{=;p}' file` | номер + строка |

## 🌳 Массовая правка по дереву (с `find`)

```bash
# рекурсивно во всех .yaml — переименовать эндпоинт
find . -name "*.yaml" -exec sed -i 's/old-api/new-api/g' {} +

# то же, но безопаснее для имён с пробелами/спецсимволами
find . -name "*.yaml" -print0 | xargs -0 sed -i 's/old-api/new-api/g'
```

> [!tip] Несколько правок за один проход
> `-e` объединяет команды: `sed -e 's/a/b/g' -e '/DEBUG/d' file`. Или через `;`: `sed 's/a/b/g; /DEBUG/d' file`. Длинные наборы команд удобно вынести в файл: `sed -f script.sed file`.

## ⚠️ Безопасность правок in-place

> [!danger] `-i` перезаписывает файл без подтверждения
> Кривой regex массово испортит файлы молча. **Перед `-i`:**
> 1. Прогони **dry run** — без `-i`, посмотри вывод глазами: `sed 's/old/new/g' file`.
> 2. Проверь, что и где совпадёт: `grep -rn 'old-api' .`.
> 3. **Закоммить** до правки и потом просмотри `git diff` — отловишь лишние замены.
> 4. На ценных файлах — бэкап: `sed -i.bak ...` (или скопируй заранее).

## 🌍 GNU sed vs BSD/macOS — важные различия

> [!warning] `-i` ведёт себя по-разному
> - **GNU sed** (Linux: Gentoo, Debian/Ubuntu, Arch): `sed -i 's/.../.../' file` — суффикс бэкапа **необязателен**.
> - **BSD/macOS sed**: после `-i` **обязателен аргумент суффикса**, для «без бэкапа» — пустая строка: `sed -i '' 's/old/new/g' file`. Без неё — ошибка.
> - Расширенные regex: GNU — `-E` **или** `-r`; BSD — только `-E`. Фишки `\L\U`, `1~2`, `I`-флаг, однострочный `a текст` — это **расширения GNU**, на BSD их может не быть.
> - Совет: для переносимых скриптов используй POSIX-подмножество либо явно ставь GNU sed (на macOS — `gsed` из `brew install gnu-sed`).

## 📦 Доступность по системам

| Система | sed |
| :--- | :--- |
| **Gentoo** (основная) | GNU sed предустановлен (`sys-apps/sed`) |
| **Debian / Ubuntu** | GNU sed предустановлен (пакет `sed`) |
| **Arch** (план с июня 2026) | GNU sed предустановлен (пакет `sed`, в базовой группе) |
| **Entware / RT-AX56U** | по умолчанию **busybox sed** — урезанный (нет `\L\U`, `~`, ограниченный `-E`/`-i`). Полноценный ставится: `opkg install sed` → GNU sed в `/opt/bin/sed` |

## 🧰 Частые рецепты

```bash
# обрезать пробелы по краям строк
sed -i 's/^[[:space:]]*//; s/[[:space:]]*$//' file

# раскомментировать строку с параметром в конфиге
sed -i 's/^#\s*\(PermitRootLogin\)/\1/' /etc/ssh/sshd_config

# заменить значение ключа в YAML/конфиге
sed -i 's/^\(timeout:\).*/\1 30/' config.yaml

# показать только N-ю строку
sed -n '42p' file

# удалить со строки с маркером и до конца файла
sed -i '/# BEGIN BLOCK/,$d' file

# вывести файл без последней строки
sed '$d' file
```
