[**APT (Advanced Packaging Tool)**](https://help.ubuntu.ru/wiki/apt) — это набор утилит и пакетный менеджер, используемый в операционных системах на базе Debian (например, Debian, Ubuntu и их производных). Он предоставляет удобный интерфейс для управления программными пакетами, выступая в роли надстройки над низкоуровневым инструментом [dpkg](DPKG.md).

> [!note] `apt` vs `apt-get` / `apt-cache`
> `apt` — современный человеко-ориентированный фронтенд (прогресс-бар, цветной вывод, объединяет команды установки и поиска). Для **скриптов** рекомендуется использовать `apt-get` и `apt-cache`: у них стабильный, не меняющийся между версиями вывод и набор опций. Сам `apt` при запуске из скрипта предупреждает: *«WARNING: apt does not have a stable CLI interface»*.

#### Преимущества APT
- **Автоматическое управление зависимостями** — пользователю не нужно вручную искать и устанавливать библиотеки.
- **Удобство использования** — простые и понятные команды.
- **Широкая поддержка** — используется во многих популярных дистрибутивах Linux.
- **Надёжность и безопасность** — пакеты поставляются из проверенных репозиториев.
#### Основные команды APT
- `apt update` — обновление списка пакетов.
- `apt upgrade` — обновление всех установленных пакетов.
- `apt install <пакет>` — установка пакета.
- `apt remove <пакет>` — удаление пакета.
- `apt purge <пакет>` — удаление пакета вместе с данными/конфигами.
- `apt autoremove` — удаление неиспользуемых зависимостей.
- `apt dist-upgrade` (он же `apt full-upgrade`) — обновление системы с установкой/удалением пакетов при необходимости.
- `apt search <запрос>` — поиск пакетов.
- `apt depends <пакет>` — зависимости пакета.
- `apt rdepends <пакет>` — обратные зависимости.
- `apt list --installed` — список установленных пакетов.
- `apt list --upgradable` — список обновляемых пакетов.
- `apt show <пакет>` — информация о пакете.

#### Установка и обновление: полезные приёмы
- `apt install -y <пакет>` — флаг `-y` отвечает «да» на все вопросы (для скриптов и неинтерактивного режима).
- `apt install ./package.deb` — установка локального `.deb`-файла **с автоматическим дотягиванием зависимостей** (в отличие от `dpkg -i`, который зависимости не ставит). Путь должен начинаться с `./` или `/`.
- `apt reinstall <пакет>` — переустановка пакета (например, чтобы восстановить случайно удалённые файлы).
- `apt-get install --only-upgrade <пакет>` — обновить пакет, только если он уже установлен (не устанавливать новый).

#### Управление версиями пакетов
- `apt-mark hold <пакет>` — «заморозить» пакет: не обновлять его при `apt upgrade`.
- `apt-mark unhold <пакет>` — снять заморозку.
- `apt-mark showhold` — показать список замороженных пакетов.
- `apt policy <пакет>` — какие версии пакета доступны и из каких репозиториев (полезно для пиннинга и диагностики).

#### Очистка кэша
Скачанные `.deb`-пакеты складываются в `/var/cache/apt/archives` и со временем занимают место.
- `apt clean` — удалить **все** скачанные пакеты из кэша.
- `apt autoclean` — удалить только устаревшие пакеты, которых уже нет в репозиториях.

#### Репозитории и ключи
- `/etc/apt/sources.list` — основной список репозиториев.
- `/etc/apt/sources.list.d/*.list` (или `*.sources` в формате deb822) — дополнительные репозитории, по файлу на источник.
- `/etc/apt/keyrings/` (ранее `/etc/apt/trusted.gpg.d/`) — GPG-ключи для проверки подписи пакетов. Со времён `apt-key` устарел: ключ кладут отдельным файлом и ссылаются на него через `signed-by=` в строке репозитория.
- `apt edit-sources` — безопасно отредактировать списки репозиториев с проверкой синтаксиса.
