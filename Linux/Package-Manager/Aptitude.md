**Aptitude** — это система управления пакетами для Debian/Ubuntu, которая, как и `apt`, работает поверх библиотеки `libapt-pkg`, но предоставляет и обычный интерфейс командной строки, и полноэкранное текстовое меню (TUI). Известна своим мощным алгоритмом разрешения зависимостей (лучше справляется со сложными конфликтами, чем `apt`) и удобным интерактивным меню.

### Установка
В современных версиях Ubuntu он может не стоять по умолчанию:
```bash
sudo apt install aptitude
```

---

### 1. Командная строка (CLI)
Синтаксис: `aptitude [опции] <действие> …`

**Действия (команды):**
*   `install` — Установить/обновить пакеты.
*   `remove` — Удалить пакеты.
*   `purge` — Удалить пакеты и их файлы настройки.
*   `hold` — Зафиксировать пакеты.
*   `unhold` — Отменить команду hold для пакета.
*   `markauto` — Пометить пакет как установленный автоматически.
*   `unmarkauto` — Пометить пакет как установленный вручную.
*   `forbid-version` — Запретить aptitude обновлять до определённой версии пакета.
*   `update` — Загрузить список новых/обновлённых пакетов.
*   `safe-upgrade` — Выполнить безопасное обновление.
*   `full-upgrade` — Выполнить обновление, возможна установка и удаление пакетов.
*   `build-dep` — Установить сборочные зависимости пакета.
*   `forget-new` — Забыть, какие пакеты «новые».
*   `search` — Найти пакет по имени и/или выражению.
*   `show` — Показать подробную информацию о пакете.
*   `showsrc` — Показать подробную информацию о пакете с исходным кодом (обёртка над apt).
*   `versions` — Показать версии указанных пакетов.
*   `clean` — Удалить загруженные файлы пакетов.
*   `autoclean` — Удалить старые загруженные файлы пакетов.
*   `changelog` — Посмотреть журнал изменений пакета.
*   `download` — Загрузить файл .deb для пакета (обёртка над apt).
*   `source` — Загрузить пакет с исходным кодом (обёртка над apt).
*   `reinstall` — Переустановить уже установленный пакет.
*   `why` — Объяснить почему определённый пакет следует установить.
*   `why-not` — Объяснить почему определённый пакет нельзя установить.
*   `add-user-tag` — Добавить пользовательскую метку к пакетам/шаблонам.
*   `remove-user-tag` — Убрать пользовательскую метку у пакетов/шаблонов.

**Опции (параметры/флаги):**
*   `-h` — Эта справка.
*   `--no-gui` — Не использовать графический интерфейс GTK, даже если это возможно.
*   `-s` — Имитировать действия, но не выполнять их на самом деле.
*   `-d` — Только загрузить пакеты, ничего не устанавливать и не удалять.
*   `-P` — Всегда запрашивать подтверждение действий.
*   `-y` — Предполагать ответ «да» на простой вопрос да/нет.
*   `-F format` — Задать формат отображения результатов поиска; смотрите руководство.
*   `-O order` — Задать способ сортировки результатов поиска; смотрите руководство.
*   `-w width` — Задать ширину вывода результатов поиска.
*   `-f` — Принудительно исправлять неработоспособные пакеты.
*   `-V` — Показывать версию устанавливаемых пакетов.
*   `-D` — Показывать зависимости автоматически изменяемых пакетов.
*   `-Z` — Показывать изменение размера установки каждого пакета.
*   `-v` — Отображать дополнительную информацию. (может указываться несколько раз).
*   `-t [release]` — Задать выпуск, из которого должны быть установлены пакеты.
*   `-q` — Не выводить индикаторы хода работы в режиме командной строки.
*   `-o key=val` — Непосредственно задать опцию настройки с именем «key».
*   `--with(out)-recommends` — Указать, будут ли рекомендуемые пакеты трактоваться как строгие зависимости.
*   `-S fname` — Читать информацию о расширенном статусе aptitude из fname.
*   `-u` — Загрузить новые списки пакетов при запуске. (только в текстовом интерфейсе)
*   `-i` — Запустить установку при запуске. (только в текстовом интерфейсе)

> **Пример:** `sudo aptitude -y install git`

### 2. Интерактивный режим (TUI)
Запустите просто `aptitude` (без команд), чтобы открыть текстовое меню.

**Навигация:**
*   `Стрелки` — перемещение по списку.
*   `Enter` — войти в подраздел или выполнить действие.
*   `q` — выход.
*   `/` — поиск по пакетам.

**Управление пакетами (в списке):**
*   `+` — пометить пакет на **установку**.
*   `-` — пометить пакет на **удаление**.
*   `_` — пометить пакет на **удаление с очисткой** (purge).
*   `?` — показать информацию о выделенном пакете.

**Глобальные действия:**
*   `u` — обновить списки пакетов (update, как `apt update`).
*   `U` — пометить все обновляемые пакеты к обновлению.
*   `g` — показать превью изменений и **применить** их (повторное `g` подтверждает).
*   `Ctrl+T` — открыть строку меню (доступ ко всем командам).

### 3. Синтаксис поиска
Команда `aptitude search` выводит все пакеты, соответствующие переданным шаблонам или словам.  
По сравнению с apt/apt-cache, aptitude предоставляет богатую систему поисковых шаблонов, удобную для сложных запросов.

- Базовый синтаксис: `aptitude search <шаблон> [<шаблон2> ...] [опции]`
- Поиск по имени (по умолчанию ищет в имени пакета): `aptitude search nginx`
- Поиск по описанию (короткая форма ~d): `aptitude search '~dnginx'`
- Поиск среди установленных пакетов (короткая форма ~i): `aptitude search '~i nginx'`

#### Поисковые шаблоны (краткая справка)

-  `~n<regex>` — имя пакета (name).    
-  `~d<regex>` — описание пакета (description).    
-  `~i` — установленные пакеты (installed).    
-  `~U` — пакеты, которые можно обновить (upgradable). Отдельной короткой буквы для «неустановленных» нет — используйте отрицание `!~i`.    
-  `~m<maintainer>` — maintainer (мейнтейнер пакета).    
-  `~t<task>` — пакеты из task.
-  `!` (или `~!`) — логическое НЕ (not).    
-  `|` — логическое ИЛИ (or). Внутри одного шаблона пробел (соседство шаблонов) означает И (AND): `'~i ~dapache'` = установлен **и** в описании apache.   
    (Полный список и длинные формы терминов доступны в справочнике aptitude.) 
    [debian aptitude](https://www.debian.org/doc/manuals/aptitude/ch02s04s05.en.html)        

#### Формат вывода и опции форматирования

- По умолчанию каждая строка вывода выглядит так: один символ состояния, пробел, имя пакета и краткая информация о версиях; символ состояния показывает текущее состояние пакета (например, "i" — установлен, "p" — отсутствует в системе и т.д.).
- Опция -F / --display-format позволяет задать пользовательский формат вывода.
- Опция --sort позволяет сортировать вывод. 

#### Практические примеры

- Найти пакет по точному имени: `aptitude search '^nginx$'` — вернёт пакеты с точным именем nginx (регулярное выражение с `^` и `$`; внутри одинарных кавычек `$` экранировать не нужно).
- Найти пакеты, в описании которых встречается слово "editor": `aptitude search '~deditor'`
- Найти установленные пакеты, у которых в описании встречается "apache": `aptitude search '~i ~dapache'` — оба условия должны выполняться.
- Найти пакеты, имя которых начинается с "lib" но не содержит "gtk": `aptitude search '~n^lib !~ngtk'` — оба условия в одном шаблоне (AND). Если передать их двумя отдельными аргументами, aptitude покажет объединение (OR), а не пересечение.
- Найти пакеты, поддерживаемые конкретным мейнтейнером: `aptitude search '~mCanonical'` — регистрозависимость/регулярные выражения применимы.
- Пример OR: найти nginx или apache2 по имени: `aptitude search '~nnginx | ~napache2'`

#### Пояснения по логике поиска

- Без префикса шаблон ищет по имени пакета по умолчанию.
- Шаблоны можно комбинировать; пробел внутри одного шаблона = И (AND); несколько отдельных аргументов команды объединяются по ИЛИ (union). Для предсказуемости берите выражение в кавычки и используйте явные `|` и `!`. 
- Для сложных запросов используйте кавычки вокруг выражения, чтобы оболочка не интерпретировала символы. Например: `aptitude search '~i ~dapache'` — в одинарных или двойных кавычках.

#### Примеры вывода и как его читать

- Строка вывода: `"i nginx - small, powerful, scalable web/proxy server"` — буква i означает, что пакет установлен; далее имя пакета и краткое описание.
- Если отображаются версии и источники, aptitude может показывать доступные версии и pin-приоритеты.

### Когда использовать?
*   **`apt`** — для простых задач (стандарт в Ubuntu).
*   **`aptitude`** — когда `apt` не может разрешить зависимости, нужно массово отметить пакеты к удалению или нравится текстовый интерфейс.
