# 🖥️ sshm (Gu1llaum-3) — TUI-менеджер SSH-хостов на Go

[**sshm**](https://github.com/Gu1llaum-3/sshm) (автор **Gu1llaum-3**, **Go 1.23+**, лицензия **MIT**, ~1.26k★, v1.11.0 от 18.03.2026) — консольный **TUI-менеджер SSH-подключений**: красивый текстовый интерфейс, где твои хосты видно списком, по ним можно искать, группировать тегами и подключаться в один клик. Ключевое, что стоит сразу подчеркнуть (пост это смазал): **sshm не заводит свою базу — он работает прямо с твоим `~/.ssh/config`**, дописывая туда лишь комментарии-теги. То есть это «удобная морда» над штатным конфигом OpenSSH, а не отдельное хранилище.

## ✨ Что умеет

- **TUI-браузер хостов** с vim-навигацией (`j/k`, стрелки) и индикаторами доступности (🟢🟡🔴⚫ — пингует и показывает статус).
- **Поиск/фильтр** по хостам, сортировка по имени или времени последнего входа (история логинов).
- **CRUD через формы:** добавить/изменить/удалить хост интерактивно, с валидацией ввода.
- **Теги** для организации, включая спец-тег `hidden` (спрятать хост из списка).
- **Проброс портов** прямо из интерфейса: локальный `-L`, удалённый `-R`, динамический SOCKS `-D` — с сохранением истории пробросов.
- **ProxyJump / ProxyCommand** — подключение через бастион (jump-host).
- **Прямой CLI:** `sshm hostname` — сразу коннект; `sshm hostname <команда>` — выполнить команду на хосте.
- **Автобэкап** конфига перед каждым изменением; поддержка `Include`-директив (раскладка хостов по нескольким файлам, перенос хоста между ними); свой конфиг через `-c`.
- **Shell-completion** для bash/zsh/fish/PowerShell; автопроверка обновлений.

## ⚠️ Что важно понимать (факты против хайпа)

> [!warning] «Команды на нескольких серверах одновременно» — в посте преувеличение
> В README **нет** параллельного/массового выполнения по флоту серверов. Есть `sshm hostname command` — это запуск команды на **одном** выбранном хосте. Если нужен реальный «прогон по многим серверам разом» — это уже **Ansible / pssh / xargs+ssh / mussh**, а не sshm. sshm — про **удобное управление и коннект**, а не про оркестрацию.

> [!note] «Сохранение учётных данных» — это НЕ хранение паролей
> sshm **не хранит пароли**. Он опирается на стандартную аутентификацию SSH из `~/.ssh/config` — то есть **ссылается на твои ключи/identity-файлы**, а не складывает секреты в собственное хранилище. Это плюс для безопасности: нет ещё одной «базы паролей», которую можно утащить.

> [!tip] Главное достоинство — совместимость с `~/.ssh/config`
> Поскольку всё живёт в штатном конфиге OpenSSH, твои хосты **остаются рабочими и без sshm** (обычный `ssh hostname` продолжает работать), конфиг переносим между машинами, а sshm можно в любой момент убрать без «vendor lock-in». Теги пишутся как комментарии — стандартный `ssh` их просто игнорирует.

> [!caution] Не путать с похожими именами
> Есть неродственные проекты с близкими названиями (напр. другой `sshm` — менеджер SSH-туннелей на Python). Здесь речь про **Gu1llaum-3/sshm** на Go. Ставь по точному репозиторию.

## 🆚 С чем сравнить

Категория «TUI/CLI-менеджер SSH» плотная: **sshs**, **gossh**, **sshto**, классический **~/.ssh/config + fzf**, а из GUI — **Termius** (проприетарный, с облачной синхронизацией и хранением секретов). Фишка sshm — **не изобретать формат**, а дать красивый TUI поверх уже существующего `~/.ssh/config` + удобный проброс портов и jump-host из коробки.

## 🖥️ По системам

Один Go-бинарь без зависимостей.

| Система | Установка |
| :--- | :--- |
| **Gentoo** (основная) | `go install github.com/Gu1llaum-3/sshm@latest` или бинарь из релизов (Linux AMD64/ARM64); скрипт-инсталлятор one-liner |
| **Debian / Ubuntu** | Бинарь из релизов или `go install`; one-liner-скрипт с сайта проекта |
| **Arch** (план с июня 2026) | `go install` / бинарь; при наличии — AUR-пакет |
| **macOS** | **Homebrew:** `brew install Gu1llaum-3/sshm/sshm` (Intel и Apple Silicon) |
| **Windows** | PowerShell-инсталлятор или бинарь (AMD64/ARM64) |
| **Entware / RT-AX56U** | ⚠️ Готового бинаря под **armv7** (32-бит) нет — есть только ARM64/AMD64. Нужен **кросс-компил** `GOOS=linux GOARCH=arm GOARM=7 go build` на десктопе → закинуть бинарь на роутер. Работать будет (тулза лёгкая, TUI), но обычно SSH-хостами удобнее рулить **с рабочей машины**, а не с роутера |
