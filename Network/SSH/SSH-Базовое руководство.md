# SSH — Базовое руководство

Основы работы с `SSH` (Secure Shell): подключение, разбор командной строки, файл конфигурации клиента, копирование файлов и повседневные приёмы удалённого администрирования. Про генерацию и типы ключей см. отдельную заметку [SSH-Ключи](SSH-Ключи.md). Продвинутые сценарии (туннели, проброс портов, VPN, прыжки по хостам) — в [SSH — Продвинутое руководство](SSH-Продвинутое%20руководство.md).

## Разбор командной строки SSH

Типичное подключение к удалённому серверу со часто используемыми параметрами:

```bash
localhost:~$ ssh -v -p 22 -C neo@remoteserver
```

- `-v` — вывод отладочной информации, особенно полезен при анализе проблем аутентификации. Можно повторять (`-vv`, `-vvv`) для большей детализации.
- `-p 22` — **порт** удалённого SSH-сервера. 22 — значение по умолчанию, можно не указывать; для нестандартного порта (`Port 2222` в `sshd_config`) укажите его явно.
- `-C` — сжатие соединения. Ускоряет работу на медленном канале или при большом объёме текста.
- `neo@` — имя пользователя для аутентификации. Без него используется текущий локальный пользователь (`whoami`). Альтернатива — параметр `-l`.
- `remoteserver` — имя хоста: FQDN, IP-адрес или запись из локального `hosts`. Для хостов с IPv4 и IPv6 уточняйте резолвинг параметром `-4` или `-6`.

Обязателен только `remoteserver`, остальные параметры — опциональны.

## Файл конфигурации клиента (~/.ssh/config)

Помимо серверного `sshd_config`, у клиента `ssh` есть свой файл настроек — по умолчанию `~/.ssh/config` (можно задать через `-F`). Он избавляет от длинных команд: настройки применяются автоматически при подключении к конкретному хосту.

```
Host *
     Port 2222

Host remoteserver
     HostName remoteserver.thematrix.io
     User neo
     Port 2112
     IdentityFile /home/.ssh/remoteserver.private_key
```

Первая запись (`Host *`) применяется ко всем хостам, вторая переопределяет имя пользователя, порт, FQDN и ключ для хоста `remoteserver`.

## Копирование файлов: scp и sftp

SSH-клиент несёт два инструмента для копирования по **зашифрованному соединению**. Многие параметры `ssh` работают и здесь.

```bash
localhost:~$ scp mypic.png neo@remoteserver:/media/data/mypic_2.png
```

Файл `mypic.png` копируется на `remoteserver` в `/media/data` и переименовывается в `mypic_2.png`.

> [!warning] Внимание на порт
> В `scp` параметр порта — заглавная `-P`, а **не** `-p`, как в `ssh`. На этом спотыкаются почти все.

Для знакомых с консольным `ftp` команды `sftp` похожи (`put`, `get`, `ls`):

```bash
sftp neo@remoteserver
```

Подробнее о копировании — в заметке [SCP](../../Linux/Commands/SCP.md).

## Удалённое выполнение команд

Чтобы выполнить одну команду, не нужно открывать интерактивную сессию — добавьте команду последним аргументом:

```bash
$ ssh user@host df -h          # заполненность ФС удалённой машины
$ ssh user@host sudo reboot    # перезагрузка
```

Список команд можно держать в файле и скормить его SSH целиком:

```bash
$ ssh user@host "`cat file.txt`"
```

> [!note] Где выполняется команда в конвейере
> ```bash
> $ ssh user@host "cat /var/log/nginx/access.log" | grep badstuff.php
> ```
> Здесь `grep` отрабатывает **локально**, после скачивания лога по каналу. Чтобы grep шёл на удалённой стороне (важно для больших файлов), заключите обе команды в одни кавычки: `ssh user@host "cat ... | grep ..."`.

## Копирование публичного ключа ssh-copy-id

Утилита сама добавит ваш публичный ключ в `~/.ssh/authorized_keys` на сервере:

```bash
$ ssh-copy-id user@host
```

Флагом `-i` можно указать конкретный ключ:

```bash
$ ssh-copy-id -i ~/my_key.pub user@host
```

Эквивалент вручную:

```bash
$ cat ~/.ssh/id_rsa.pub | ssh remoteserver 'cat >> .ssh/authorized_keys'
```

## Повседневные приёмы

### Редактирование удалённого файла локальным редактором

Файл на сервере можно открыть в локальном редакторе (метод создаёт временную копию в `/tmp` и копирует обратно при сохранении):

```bash
$ gvim scp://user@host//путь/к/файлу
$ vim  scp://user@remoteserver//etc/hosts
```

> Двойной `//` после хоста — абсолютный путь; один `/` — путь относительно домашней папки пользователя.

### Содержимое удалённого файла в буфер / diff

```bash
$ ssh user@host cat /путь/к/файлу | xclip            # в буфер обмена
$ ssh user@host uname -a | xclip                      # вывод команды в буфер
$ ssh user@host cat /удал/файл | diff /лок/файл -     # сравнить без копирования
```

### Монтирование удалённой папки (sshfs)

Любой каталог удалённой машины подключается как сетевой диск:

```bash
$ sudo apt install sshfs
$ mkdir ~/remote_files
$ sshfs user@host:/home/user ~/remote_files/
```

Теперь файлы `/home/user` доступны в `~/remote_files/` как обычные.

### Резервное копирование каталога (rsync over SSH)

`rsync` удобнее `scp` для периодических бэкапов: докачивает после сбоя и копирует только изменённые файлы. Здесь сжатие (`-z`) и режим архивирования (`-a`, включает рекурсию):

```bash
$ rsync -az /home/testuser/data remoteserver:backup/
```
