# 🔌 Portpal — GUI-дэшборд портов с «убить в один клик»

**Portpal** ([github.com/wisher567/Portpal](https://github.com/wisher567/Portpal)) — десктоп-приложение (**Tauri 2**, Win/macOS/Linux), которое показывает, **какие приложения слушают какие порты**, и даёт **прибить процесс одним кликом**. Заточено под разработчиков: детект фреймворков, «kill/restart», D3-карта топологии, спарклайны трафика, иконка в трее. Решает боль `EADDRINUSE: address already in use`. MIT, TypeScript, ~301★.

> [!caution] Главное: проект по сути ЗАБРОШЕН и БЕЗ готовых сборок
> Красиво выглядит, но трезво:
> - **Один день жизни**: репозиторий создан **11 апреля 2026**, последний коммит — **12 апреля 2026**. С тех пор (на момент заметки, конец июля) — **~3.5 месяца тишины**. Фактически проект одного уикенда.
> - **Опубликованных релизов НЕТ**: есть git-тег `v0.2.0`, но **ни одного GitHub Release с бинарями** (0 ассетов). Кнопки «Download .msi / .dmg / AppImage» в README ведут в **пустой** `releases/latest`. То есть **готовых установщиков нет — только сборка из исходников** (Tauri: Rust + Node + webkit2gtk). Пост подаёт как готовый инструмент — по факту это исходники брошенного прототипа.

> [!warning] Оно дублирует то, что в Linux есть из коробки
> На твоих системах «кто на порту» и «убить» решается нативно:
> ```bash
> ss -tlnp            # какие процессы слушают TCP-порты (+PID)
> lsof -i :3000       # кто держит порт 3000
> fuser -k 3000/tcp   # убить процесс на порту 3000
> ```
> Ценность Portpal — **GUI/топология/детект фреймворков** (React/Vite/Django/Node) для дев-workflow с кучей локальных серверов, а не новая функциональность. Для CLI-человека выигрыш небольшой.

> [!caution] Kill по порту — это риск
> «Убить в один клик» и особенно **«Kill All» panic-кнопка** требуют привилегий и **могут прибить системный процесс** → сломать что-то. Смотри, что именно гасишь (PID/имя), а не жми массово.

> [!info] Что заявлено (v0.2, по README)
> Дэшборд слушающих портов (имя процесса, PID, соединения, детект фреймворка, имя проекта по `package.json`/`Cargo.toml`/`go.mod`), one-click kill/restart, **D3.js-карта** взаимодействия сервисов (drag/zoom), спарклайны трафика, лог событий, фильтр «Dev vs System», трей со «светофором».

## 🖥️ Применимость на системах владельца

| Система | Как |
| :--- | :--- |
| **Gentoo (основная) / Debian-Ubuntu / Arch** | ⚠️ готовых бинарей нет → **собирать из исходников** (Tauri: Rust + Node + `webkit2gtk`). Для «кто на порту» практичнее нативные `ss`/`lsof`/`fuser`. Portpal — только если очень хочется GUI/топологию и не смущает заброшенность |
| **Windows** | ⚠️ то же — релизов нет, только сборка. На Windows задачу закрывает System Informer (там и порты, и процессы) |
| **Entware / RT-AX56U** | ➖ GUI-десктоп, для роутера неактуально (на нём — `netstat`/`ss` из BusyBox/Entware) |

> [!tip] Практичный вывод
> Идея приятная (особенно D3-топология и детект дев-серверов), но как **готовый продукт это не рассчитывай**: нет релизов, нет поддержки. Если нужен GUI по портам «здесь и сейчас» — на Windows бери System Informer, на Linux хватает `ss -tlnp`/`lsof`. Portpal — скорее «идея на посмотреть/форкнуть».
