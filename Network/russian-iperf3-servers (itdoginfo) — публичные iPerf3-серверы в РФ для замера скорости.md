# 📡📊 russian-iperf3-servers (itdoginfo) — публичные iPerf3-серверы в РФ

[**russian-iperf3-servers**](https://github.com/itdoginfo/russian-iperf3-servers) (*itdoginfo*, **Shell/Python**, ~**134★**, с июля 2025, обновляется **ежедневно**) — поддерживаемый список **публичных iPerf3-серверов в России** (~50 штук по городам) с **автоматической проверкой доступности раз в сутки**. Список серверов лежит в `list.yml`, GitHub Actions каждый день гоняет тест и обновляет таблицу статусов в README. Есть готовый скрипт `speedtest.sh`, который сам находит рабочий сервер и меряет скорость.

**iPerf3** — стандартный инструмент замера **максимальной пропускной способности** канала (TCP/UDP throughput) между двумя хостами: нужен сервер и клиент. Этот репозиторий избавляет от необходимости поднимать свой сервер — даёт список чужих публичных.

> [!info] Что внутри
> - **~50 серверов** по городам РФ: в основном **Ertelecom** (Дом.ru, порты `5201–5209`), плюс **МТС** (`3333`), **ТТК**, **Билайн**, **Hostkey** (Москва).
> - **Fallback:** на город есть основной и резервный сервер; тест перебирает порты `5201–5209`, и только если все упали — уходит на резерв (в таблице помечается `City (F)`).
> - **Скрипт замера** (сам подберёт живой сервер):
>   ```bash
>   bash <(wget -qO- https://github.com/itdoginfo/russian-iperf3-servers/raw/main/speedtest.sh)       # обычный
>   bash <(wget -qO- https://github.com/itdoginfo/russian-iperf3-servers/raw/main/speedtest.sh) -f    # fast mode
>   ```
>   Зависимости: `iperf3`, `jq`, `ping`, `awk`.

## 🧪 Факты против хайпа

> [!warning] iPerf3 меряет скорость ДО КОНКРЕТНОГО сервера, а не «скорость интернета вообще»
> Результат зависит от **пути до выбранного сервера**: пиринг, расстояние, загрузка сервера. Публичные серверы **общие** → могут быть перегружены, и низкое число — не обязательно твоя вина. Выбирай **географически/сетево близкий** сервер (свой город/регион), тогда цифра ближе к реальной пропускной способности канала.

> [!caution] Это серверы операторов — показывают «внутрисетевую» скорость, а не международную
> Ertelecom/МТС/ТТК/Билайн — это **speedtest-серверы самих провайдеров**. Замер до них отражает скорость **внутри РФ / до сети оператора**, которая часто **выше** реального доступа к зарубежным ресурсам (оператор к своему speedtest пускает «на всех парах»). Полезно для проверки **своего линка/LAN→ISP**, но **не** показатель скорости до забугорных сервисов/через туннель.

> [!note] Тонкости самого iPerf3
> - iPerf3 **однопоточный** по умолчанию; на быстрых каналах (гигабит+) один поток может не насытить линию — добавляй параллельные потоки: `iperf3 -c <сервер> -P 8`.
> - По умолчанию TCP; для проверки потерь/джиттера — UDP: `iperf3 -c <сервер> -u -b 100M`.
> - **Этикет:** публичные серверы чужие — гоняй **короткие** тесты (`-t 10`), не держи нагрузку долго.

## 💻 Как это ложится на твои системы

Нужен только **клиент iPerf3** (+ `jq` для скрипта). Ставится везде.

| Система | Установка клиента |
| :--- | :--- |
| **Gentoo** (основная) | `emerge net-misc/iperf` (бинарь `iperf3`) + `app-misc/jq` |
| **Debian / Ubuntu** | `sudo apt install iperf3 jq` |
| **Arch** | `sudo pacman -S iperf3 jq` |
| **Entware / RT-AX56U** | ✅ `opkg install iperf3 jq` — можно мерить **прямо с роутера** реальную скорость WAN-линка (bash/awk/ping там есть; `speedtest.sh` заведётся). Удобно, когда надо проверить именно канал, а не Wi-Fi/устройство |
| **Windows 10/11** | Официальный бинарь iPerf3 (`iperf3.exe -c <сервер>`); `speedtest.sh` — только под WSL/Git Bash, но ручной запуск работает и в cmd/PowerShell |

## 💡 Кому полезно

- **Быстро проверить пропускную способность канала** до близкого RU-города без своего сервера (диагностика после «интернет тормозит», приёмка тарифа, тест роутера/кабеля).
- **Тест самого роутера (RT-AX56U):** запусти `iperf3` на роутере → увидишь честную скорость WAN, отдельно от Wi-Fi и клиентских устройств.
- **Не** для замера скорости до зарубежных сервисов или **через туннель** — тут цифры операторских серверов вводят в заблуждение (для «до забугра» меряй до забугорного iPerf3-сервера).
