# 🛰️ WatchYourLAN — лёгкий сканер устройств в локальной сети с веб-GUI

**WatchYourLAN** ([github.com/aceberg/WatchYourLAN](https://github.com/aceberg/WatchYourLAN)) — лёгкий **сетевой IP-сканер с веб-интерфейсом** на **Go**: следит за устройствами в твоей локалке, **уведомляет о появлении нового хоста**, ведёт **историю online/offline**, хранит список всех устройств (MAC, вендор, IP) и умеет отдавать данные в **Grafana** через InfluxDB2 или Prometheus. MIT, **~7.1k★**, Docker/бинари. Развёртывание: Docker, `.deb`/`.rpm`/`.apk`/`.tar.gz`.

> [!info] Пост описывает точно — это инвентаризация/мониторинг, а не «сканер уязвимостей»
> Уточню категорию, чтобы не было завышенных ожиданий:
> - Это **дашборд-инвентарь устройств LAN** и их присутствия (кто онлайн/офлайн, когда появился новый), **не** nmap-порт-сканер и **не** сканер уязвимостей.
> - Полезная **security-грань**: заметить **чужое устройство** в своей Wi-Fi/сети (новый неизвестный MAC → алерт). Для домашней сети это дёшево и по делу.
> - Уведомления идут через **Shoutrrr** — умеет **Telegram, Discord, Ntfy, Gotify, Matrix, Pushover, e-mail, вебхук** и др.

> [!warning] Как реально работает и что пост не сказал — важные требования
> - **Сканирует по ARP** (зависимость — `arp-scan`) → **видит только устройства своего L2-сегмента** (тот же subnet). За роутером/в других VLAN-ах — только с доп. настройкой (см. их `docs/VLAN_ARP_SCAN.md`). Это не «просканирует весь интернет/подсети за NAT».
> - В Docker нужен режим сети **`host`** и указание интерфейса (`IFACES`) — иначе ARP не увидит соседей. Для ARP нужны привилегии (raw-сокеты/root).
> - **Нет встроенной аутентификации!** README прямо предупреждает: при `host`-режиме порт **8840 открыт** → **обязательно ограничь доступ фаерволом** ([IPTables](IPTables.md)) или прикрой SSO (Authelia / ForAuth). Голым в сеть не выставляй.
> - Бэкенд: **SQLite** (по умолч.) или **PostgreSQL**; история чистится по `TRIM_HIST` (по умолч. 48 ч).

> [!caution] Актуальность
> Проект зрелый (с 2022), но **последний релиз — v2.1.4, сентябрь 2025** (~10 месяцев без обновлений на момент заметки). Стабилен, но активного развития сейчас мало — учитывай при выборе на годы вперёд.

## ⚙️ Быстрый старт (Docker)

```sh
docker run --name wyl \
  -e "IFACES=$YOURIFACE" \
  -e "TZ=$YOURTIMEZONE" \
  --network="host" \
  -v $DATA/wyl:/data/WatchYourLAN \
  aceberg/watchyourlan
# веб-GUI: http://localhost:8840
```
Ключевые переменные: `IFACES` (интерфейс(ы) через пробел), `TIMEOUT` (период скана, по умолч. 120 с), `SHOUTRRR_URL` (уведомления), `USE_DB` (`sqlite`/`postgres`), `THEME`/`COLOR` (светлая/тёмная).

## 🖥️ Применимость на системах владельца

| Система | Как использовать |
| :--- | :--- |
| **Gentoo (основная) / Debian-Ubuntu / Arch** | ✅ поставить бинарь/Docker на всегда-онлайн машину **в той же подсети**, что и сканируемые устройства; зависимость `arp-scan` есть в репозиториях всех трёх. Спрятать за фаервол |
| **Entware / RT-AX56U** | 🟡 **реально интересный кейс**: есть бинарь под **arm_v7** и `.tar.gz`, а роутер — всегда онлайн и **сидит прямо на LAN** (идеальная точка обзора). Нужны `arp-scan` под Entware и raw-сокеты; ОЗУ 512 МБ — Go-бинарь лёгкий, должно влезть. Если `arp-scan` в opkg нет — запускай на десктопе в той же сети |
| **Windows** | ➖ ориентирован на Linux/Docker + `arp-scan`; нативно не целевая платформа |

> [!tip] Тебе по профилю — домашняя сеть под присмотром
> У тебя ASUS RT-AX56U и домашняя сеть. WatchYourLAN даст **веб-инвентарь всех устройств + алерт в Telegram**, когда подключится новый (незнакомый телефон/гаджет — повод проверить). Ставь на всегда-онлайн Linux-хост в LAN (или, если заведётся `arp-scan`, прямо на роутер). Обязательно закрой порт [фаерволом](IPTables.md) — своей auth у него нет.
