# 🗑️ Open Trashmail — самохостируемая одноразовая почта со своим мейл-сервером

**Open Trashmail** ([github.com/HaschekSolutions/opentrashmail](https://github.com/HaschekSolutions/opentrashmail)) — **самохостируемое решение для одноразовой (trashmail) почты**, которое **везёт с собой собственный почтовый сервер**. Принимает письма и отдаёт их через **веб-интерфейс, JSON API, RSS-ленту и вебхуки**. **Catch-all**: любой валидный адрес на твоём домене работает сразу, ничего заранее создавать не надо; файловое хранение, без БД. Автор — [HaschekSolutions](https://github.com/HaschekSolutions) (известный self-host-разработчик). Apache-2.0, **~912★**, зрелый (с 2019).

> [!info] Мелкая правка к посту: сервер — на Python, не «PHP»
> Пост пишет «Lang: PHP». На деле **веб-UI на PHP (8.1+) + htmx**, а сам **SMTP-сервер — на Python (3.11)**. Не принципиально, но «почтовый сервер» здесь питоновский.

> [!tip] Где оно в ряду твоих заметок про почту — это «золотая середина»
> - [MailWave (публичная temp-mail)](../../Apps/MailWave%20%E2%80%94%20%D0%B1%D0%B5%D1%81%D0%BF%D0%BB%D0%B0%D1%82%D0%BD%D0%B0%D1%8F%20%D0%BE%D0%B4%D0%BD%D0%BE%D1%80%D0%B0%D0%B7%D0%BE%D0%B2%D0%B0%D1%8F%20%28temp-mail%29%20%D0%BF%D0%BE%D1%87%D1%82%D0%B0%20%D0%B1%D0%B5%D0%B7%20%D1%80%D0%B5%D0%B3%D0%B8%D1%81%D1%82%D1%80%D0%B0%D1%86%D0%B8%D0%B8%20%28%D0%B7%D0%B0%D1%89%D0%B8%D1%82%D0%B0%20%D0%BE%D1%82%20%D1%81%D0%BF%D0%B0%D0%BC%D0%B0%29%2C%20%D0%B3%D0%B4%D0%B5%20%D0%BF%D0%BE%D0%BB%D0%B5%D0%B7%D0%BD%D0%BE%20%D0%B8%20%D0%B3%D0%B4%D0%B5%20%D0%BE%D0%BF%D0%B0%D1%81%D0%BD%D0%BE.md) — **публичная** temp-mail (чужой сервис, общий ящик, ноль настройки).
> - **Open Trashmail** — **своя** temp-mail: ты владеешь доменом и сервером, ящик не расшарен с чужими людьми (при правильной защите) → годится для **автоматизации и тестов**.
> - [docker-mailserver (полноценный почтовый сервер)](docker-mailserver%20%E2%80%94%20production-ready%20%D0%BF%D0%BE%D1%87%D1%82%D0%BE%D0%B2%D1%8B%D0%B9%20%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80%20%D0%B2%20%D0%BA%D0%BE%D0%BD%D1%82%D0%B5%D0%B9%D0%BD%D0%B5%D1%80%D0%B5%20%28Postfix-Dovecot-Rspamd-ClamAV%29%2C%20%D1%87%D1%82%D0%BE%20%D0%B2%D0%BD%D1%83%D1%82%D1%80%D0%B8%20%D0%B8%20%D0%BF%D0%BE%D0%B4%D0%B2%D0%BE%D0%B4%D0%BD%D1%8B%D0%B5%20%D0%BA%D0%B0%D0%BC%D0%BD%D0%B8%20self-hosting.md) — **полноценный** почтовый сервер (приём **и** отправка, реальные ящики). Trashmail только **принимает** (catch-all-сток), отправлять не умеет.

> [!info] Реальная киллер-фича — не «личная почта», а АВТОМАТИЗАЦИЯ
> Сильная сторона — **JSON API + вебхуки + RSS**: можно **программно вытаскивать письма и коды 2FA**, тестировать регистрацию/письма в CI, ловить события. Явно заявленные кейсы: **email-honeypot** и **автоматическое чтение 2FA-писем** для своих проектов. Есть скачивание вложений, plaintext/STARTTLS/TLS-on-connect.

> [!caution] Приватность и безопасность: это всё ещё «мусорка», плюс catch-all-риски
> - **Не для важных аккаунтов**: одноразовая почта, письма удаляются, восстановить доступ нечем. Пароли/банки/госуслуги — мимо.
> - **Catch-all по умолчанию слабо приватен**: кто знает адрес на твоём домене — может его запросить. **Обязательно закрой доступ**: `PASSWORD` (гейт на сайт+API), `ALLOWED_IPS` (CIDR-allowlist), `ADMIN_PASSWORD` для админ-меню, `DISCARD_UNKNOWN=true`. Иначе открытый веб-UI/API = публичный ящик.
> - **Этика**: автовыгрузка 2FA для **своих** тестовых аккаунтов/своего сервиса — норм; массовое штампование одноразовых аккаунтов ради обхода верификации чужих сервисов — это abuse их правил (та же оговорка, что и по MailWave).

> [!warning] Требования как у любого приёмника почты (пост умалчивает)
> Чтобы реально **получать** письма извне, нужны **свой домен с MX-записью на этот сервер** и **открытый входящий порт 25**. На домашнем провайдере/динамическом IP входящий 25 часто закрыт/в блок-листах → на практике это **VPS**. Для локальной разработки (приложение шлёт письма на твой же сток) домен и 25-й наружу не нужны.

## ⚙️ Ключевые настройки (config.ini / Docker env)

| Параметр | Зачем |
| :--- | :--- |
| `DOMAINS` | домены, на которых принимаем (и для генерации случайных адресов) |
| `MAILPORT` / `MAILPORT_TLS` | порт SMTP (по умолч. 25) / TLS-on-connect (465, нужны cert+key) |
| `PASSWORD`, `ALLOWED_IPS` | защита сайта/API: пароль и allowlist по IP |
| `ADMIN`, `ADMIN_ENABLED`, `ADMIN_PASSWORD` | admin-«catch-all» просмотр всех ящиков (по умолч. выключен) |
| `WEBHOOK_URL` | глобальный вебхук: POST с JSON письма на каждое входящее |
| `DISCARD_UNKNOWN` | выбрасывать почту на несконфигурированные домены |

## 🖥️ Применимость на системах владельца

| Система | Как использовать |
| :--- | :--- |
| **Gentoo / Debian-Ubuntu / Arch** | ✅ Docker (рекомендуется) или PHP 8.1+ и Python 3.11 на хосте. **Локально** — отличный приёмник для тестов (приложение шлёт письма на него). Для приёма из интернета — **VPS с доменом, MX и открытым 25** |
| **Windows** | ✅ через Docker |
| **Entware / RT-AX56U** | ❌ PHP+Python-мейл-сервер на armv7/512 МБ непрактичен, а входящий 25 на домашнем роутере/ISP проблемный. Лучше на десктопе/VPS |
