# 📮 docker-mailserver (DMS) — полноценный почтовый сервер в контейнере

**docker-mailserver** ([github.com/docker-mailserver/docker-mailserver](https://github.com/docker-mailserver/docker-mailserver)) — «production-ready, но простой» **самохостируемый почтовый сервер**, собранный в один Docker-контейнер: приём/отправка почты (SMTP), доступ по IMAP/POP3, антиспам и антивирус, DKIM/DMARC, Let's Encrypt. Ключевая идея — **только конфиг-файлы, никакой SQL-базы**: всё версионируется в git, легко разворачивать и обновлять. Зрелый и популярный проект: MIT, Shell, **~18.6k★**, живёт **с 2015 года** (с января 2021 — на волонтёрах), актуальный релиз **v15.1.0**.

> [!info] Пост описывает проект точно — это редкий случай без хайпа
> Всё из поста подтверждается README: Postfix (SMTP), Dovecot (IMAP/POP3), Rspamd (спам-фильтр), ClamAV (антивирус), LDAP-аутентификация, DKIM/DMARC, интеграция с Let's Encrypt, конфигурация файлами без SQL. Так что «разоблачать» тут нечего — вместо этого ниже честно про то, **о чём пост умалчивает**: свой почтовый сервер сложен не установкой, а **доставляемостью и эксплуатацией**.

## 📦 Что внутри (стек)

| Компонент | Роль |
| :--- | :--- |
| **Postfix** | MTA — приём/отправка SMTP, SMTP/LDAP-аутентификация, OAuth2 (XOAUTH2/OAUTHBEARER) |
| **Dovecot** | IMAP/POP3, SASL, базовый Sieve (фильтры), квоты |
| **Rspamd** + **Amavis** + **SpamAssassin** | антиспам-фильтрация |
| **ClamAV** | антивирус с автообновлением баз |
| **OpenDKIM / OpenDMARC** | подпись и проверка почты (репутация домена) |
| **Fail2ban**, **Postscreen**, **Postgrey** | защита от брутфорса и спам-ботов (greylisting) |
| **Fetchmail / Getmail6** | забор почты с внешних ящиков |
| **Let's Encrypt** | автосертификаты (плюс ручные/self-signed) |

## 🛠️ Как устроено использование

- Всё поднимается через **docker-compose** + один env-файл (`mailserver.env`) и вспомогательный **`setup.sh`** (создать аккаунты, алиасы, DKIM-ключи, квоты).
- Аккаунты/алиасы — в текстовых файлах (или LDAP), **никакой БД**. Конфиг кладёшь в git → воспроизводимо и апгрейдится сменой тега образа.
- Официальная **документация** очень подробная: [docker-mailserver.github.io](https://docker-mailserver.github.io/docker-mailserver/latest/) — читать **под свою версию** обязательно.

> [!warning] Чего в комплекте НЕТ (частое разочарование)
> - **Нет вебмейла** (Roundcube/SnappyMail) и **нет админ-GUI** — DMS это только backend (SMTP/IMAP). Веб-морду и панель управления ставишь **отдельно**.
> - **Нет DNS-магии**: MX, SPF, DKIM, DMARC, **PTR (reverse DNS)** настраиваешь у своего DNS/хостера руками. Без корректного PTR и SPF/DKIM почта будет улетать в спам.
> - Это **не «почта в один клик»**: связка мощная, но требует понимания, как работает email.

> [!caution] Главная боль self-hosting почты — доставляемость и порты, а не установка
> - **Порт 25 часто заблокирован**: большинство домашних провайдеров и облаков (AWS, GCP, Oracle и др.) режут исходящий **25/tcp** → письма не уходят. Нужен провайдер/VPS, где порт открыт, **или relay** (SMTP-релей вроде провайдера-отправителя).
> - **Нужен статический IP + PTR-запись** (reverse DNS на твой домен). На динамическом домашнем IP нормальной отправки не будет — такие диапазоны в чёрных списках.
> - **Репутация IP**: свежий IP гугл/майкрософт встречают недоверчиво; прогрев, отсутствие в блок-листах (Spamhaus и пр.) — твоя забота.
> - **Ты сам себе админ**: бэкапы ящиков, обновления безопасности, мониторинг очередей, защита от взлома (иначе твой сервер станут использовать как спам-релей).
> - **РФ-нюанс**: у ряда хостеров/провайдеров 25 порт и почтовый self-host — отдельная история с блок-листами; проверяй условия VPS до разворачивания.

## 🖥️ Применимость на системах владельца

| Система | Как использовать |
| :--- | :--- |
| **Gentoo / Debian-Ubuntu / Arch** | ✅ на десктопе/домашнем сервере или VPS есть Docker + docker-compose. **Рекомендуется ≥ 2 ГБ RAM** (ClamAV + Rspamd/Amavis прожорливы). Для реальной отправки — VPS с открытым портом 25 и PTR |
| **Entware / RT-AX56U** | ❌ роутер (armv7, **512 МБ RAM**, урезанный opkg) весь стек Docker + ClamAV/Rspamd не потянет. Максимум — держать VPS, а роутер ни при чём |

> [!tip] Когда это стоит того, а когда нет
> **Стоит**, если хочешь свой домен-почту с полным контролем и готов вести PTR/SPF/DKIM/бэкапы (учебно и «для себя» — отлично, тема близка твоему self-host-профилю). **Не стоит**, если нужна просто рабочая почта без возни — тогда managed-провайдер (Migadu, Mailbox.org, ProtonMail-домены и т.п.) дешевле по нервам. Для «мусорных» разовых адресов — это вообще не сюда (см. temp-mail ниже).
