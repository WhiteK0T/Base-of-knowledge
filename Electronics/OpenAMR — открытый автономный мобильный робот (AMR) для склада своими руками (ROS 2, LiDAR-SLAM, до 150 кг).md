# 🤖 OpenAMR — открытый автономный мобильный робот (AMR) для склада своими руками

[**OpenAMR**](https://github.com/openAMRobot/openamr) (организация **openAMRobot**, лицензия **MIT**, ~86★, актив­но пилится) — открытый **референс-проект и гайд по сборке** недорогого **автономного мобильного робота (AMR)** для внутрискладской логистики (intralogistics) малых и средних предприятий. По сути — «сделай сам промышленную тележку-робота», который сам возит грузы по складу. Идейный «старший брат» уже имеющегося в базе [oomwoo](oomwoo%20%E2%80%94%20%D0%BE%D1%82%D0%BA%D1%80%D1%8B%D1%82%D1%8B%D0%B9%20%D1%80%D0%BE%D0%B1%D0%BE%D1%82-%D0%BF%D1%8B%D0%BB%D0%B5%D1%81%D0%BE%D1%81%20%D1%81%D0%B2%D0%BE%D0%B8%D0%BC%D0%B8%20%D1%80%D1%83%D0%BA%D0%B0%D0%BC%D0%B8%20%28RPi%205%2C%20%D0%BB%D0%B8%D0%B4%D0%B0%D1%80-SLAM%2C%20ROS%202%2C%20Home%20Assistant%29.md) (робот-пылесос на том же стеке RPi 5 + LiDAR-SLAM + ROS 2), только промышленного класса.

## 📦 Что в репозитории

Полноценный DIY-пакет, а не просто код:

- **CAD и чертежи** (STEP, DXF) — механика корпуса/шасси.
- **BOM (спецификация)** — список компонентов.
- **Инструкции по сборке** и последовательность производства.
- **Прошивка** для встроенной электроники.
- **ROS 2-стек** на базе **Linorobot** (навигация).
- **Web-UI** (Flask) — операторский интерфейс.
- **Документация по архитектуре** железа.

## ⚙️ Характеристики (референс-дизайн)

| Параметр | Значение |
| :--- | :--- |
| **Грузоподъёмность** | до **150 кг** |
| **Скорость** | 1.5–2.0 м/с без груза, 1.2–1.5 м/с с грузом |
| **Точность позиционирования** | ±20 мм |
| **Батарея** | ~8 ч (системы 24/48 В) |
| **Габарит/вес** | 600×800 мм, ~60 кг |
| **«Мозг»** | **Raspberry Pi 5** |
| **Сенсоры** | LiDAR, камеры, ультразвук/ИК/бампер |
| **Навигация** | **LiDAR-SLAM** + Nav2, **Open-RMF** для управления флотом |

Производство заточено под **дешёвые процессы** (лазерная резка, гибка, 3D-печать — без сложной механообработки), отсюда и «доступность».

## ⚠️ Что важно понимать (факты против хайпа)

> [!warning] Это гайд/референс, а не готовый продукт «из коробки»
> OpenAMR — **проект для сборки**, требующий реальных навыков: механика, электроника, пайка, **ROS 2**, отладка. Плюс **деньги на BOM** (двигатели, драйверы, LiDAR, RPi 5, батареи 24/48 В, стальное/алюминиевое шасси). «Доступный» — это **относительно** коммерческих AMR (те стоят десятки тысяч $), а не «дёшево на выходных».

> [!caution] 150 кг автономно едущей массы — это про безопасность
> Робот, который **сам возит 150 кг** со скоростью до 2 м/с — не игрушка: при сбое навигации он может **травмировать человека или что-то повредить**. Промышленные AMR используют **сертифицированные safety-сканеры и аварийный стоп (e-stop)**. В DIY-варианте **обязательно** продумай аварийную остановку, ограничители скорости и зоны, тесты **вдали от людей**. Ответственность за безопасность — на сборщике.

> [!note] Репозиторий переезжает на модульную архитектуру
> Разработка **мигрирует** из монолитного репо в **отдельные репозитории** (hardware / software / firmware). Этот репо «остаётся доступным и полезен для справки», но за актуальным брать из новых репозиториев организации **openAMRobot** — проверяй, что берёшь свежую версию.

## 🖥️ По системам (что нужно для работы над ним)

Это не софт «под установку», а **робот**. Для разработки/симуляции:

- **«Мозг» робота** — **ROS 2** на **Raspberry Pi 5** (обычно Ubuntu для RPi).
- **Десктоп для разработки/симуляции** — ROS 2 + **Gazebo** удобнее всего на **Debian/Ubuntu** (наиболее поддерживаемая для ROS 2 платформа). На **Gentoo/Arch** ROS 2 ставится, но возни больше (сборка из исходников/AUR/`ros2` overlay) — под робототехнику проще держать Ubuntu-окружение (хоть в ВМ/контейнере).
- **Entware / RT-AX56U** — **N/A** (роутеру тут нечего делать).
