# 📼 Jcorp Nomad — карманный офлайн медиасервер на ESP32-S3

**Jcorp Nomad** ([github.com/Jstudner/jcorp-nomad](https://github.com/Jstudner/jcorp-nomad)) — **портативный офлайн медиасервер** в форм-факторе флешки на **ESP32-S3**. Поднимает **свою Wi-Fi-точку** с captive-порталом и раздаёт фильмы, музыку, книги, «шоу» и (в Mk4) **офлайн-Wikipedia** через браузер — **без интернета**. Открытый firmware + веб-UI. ~1141★, релиз **Mk4**, статус **beta**. Идея — медиатека для путешествий, кемпинга, классов, «на всякий случай».

> [!warning] Главное про «медиасервер» на микроконтроллере — БЕЗ транскодинга
> ESP32-S3 — это **микроконтроллер**, а не Plex-бокс. Поэтому:
> - **Он не перекодирует видео** — просто **отдаёт файлы** по своей Wi-Fi, а **декодирует их твой телефон/браузер** (плеер Plyr). Значит медиа должно быть в **браузеро-совместимых форматах** (H.264 MP4, MP3, EPUB, PDF…); экзотические кодеки/часть MKV могут не проиграться.
> - **Пропускная способность и число зрителей ограничены** слабым Wi-Fi ESP32: несколько одновременных потоков — ок, HD на толпу — нет.
> - Хранилище — **microSD** (в Mk4 тестировали Wikipedia от 0.8 ГБ до **140 ГБ** с картинками).

> [!caution] Нюансы, которые пост опустил
> - **Лицензия — CC BY-NC-SA 4.0** (GitHub не распарсил → показывает `NOASSERTION`). Ключевое: **NC = НЕкоммерческая** — собирать/модифицировать/делиться можно, **продавать — нет**; при этом автор сам продаёт готовые устройства (это его канал монетизации).
> - **Заточен под конкретную плату**: **Waveshare ESP32-S3 Dev Board (1.47" LCD)**. На других ESP32 «как повезёт» (обещан **Nomad Lite** с более широкой совместимостью, но без части фич). Пост говорит просто «ESP32-S3» — на деле нужна именно эта плата.
> - **Beta / semi-stable** — Mk4 недавно чинил перезагрузки на файлах >2 ГБ, heap-краши при индексации и т.п. Core работает, но края ещё шлифуются.
> - «Телепередачи» из поста — это **предзагруженные видеофайлы (shows)**, а не приём эфирного ТВ.
> - «Lang: HTML» — это firmware (C/C++ под ESP32) + HTML/JS-фронт; GitHub помечает по вебу.

> [!info] Что реально умеет (Mk4, по README)
> - **Офлайн-Wikipedia/ZIM** (+ Gutenberg, TED) с быстрым поиском — индекс заранее строит ПК-утилита [Nomad Tools](https://github.com/Jstudner/Nomad-Tools), чтобы устройство не перелопачивало многогигабайтный файл.
> - **Фильмы/шоу** (Plyr, папки сезонов), **музыка** (плейлисты/очередь), **книги** (EPUB/PDF/ридер комиксов), галерея, файлообмен.
> - **Resume** (продолжить с места), **admin-панель** с логином, **28 тем**, mobile-first UI, настройки переживают перезагрузку.

## 🛠️ Как заполучить

- **Собрать самому (рекомендуется автором)**: дешёвые доступные детали, сборка «меньше часа», прошивка по USB. Есть **3D-печатный корпус** (ремикс кейса с MakerWorld). Нужна плата Waveshare ESP32-S3 (1.47" LCD).
- **Купить готовый** — на [nomad.jcorptech.net](https://nomad.jcorptech.net) (это же и есть коммерческая часть). Прошивка на обоих одинаковая, обновляется самостоятельно.

## 🖥️ Применимость на системах владельца

Это **автономное устройство**; твои ОС нужны разве что для подготовки контента/прошивки:

| Что | Как |
| :--- | :--- |
| **Gentoo / Debian-Ubuntu / Arch / Windows** | ✅ прошить ESP32-S3 (Arduino/esptool), накидать медиа на microSD, а для офлайн-Wikipedia — прогнать **Nomad Tools** (строит индекс на ПК). Клиент — любой браузер телефона/ноута по Wi-Fi устройства |
| **Entware / RT-AX56U** | ➖ роутер ни при чём: Nomad — отдельная железка со своим Wi-Fi |

> [!tip] Тебе по профилю (Electronics/3D-печать/DIY)
> Отличный выходной проект: ESP32-S3 + microSD + печатный корпус. Практический смысл — **офлайн-медиатека и Wikipedia «в кармане»** (поездки, дача, отсутствие сети). Держи в голове: это «файлы-по-хотспоту с красивым UI», а не Plex — держи медиа в веб-форматах и не жди транскодинга. Для полноценного домашнего сервера — это другое (см. self-host-заметки).
