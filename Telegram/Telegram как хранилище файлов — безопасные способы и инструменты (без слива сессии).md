# 🔐 Telegram как хранилище файлов — безопасные способы и инструменты (без слива сессии)

Использовать Telegram как «бесконечный диск» — рабочая идея, но сервисы вроде [Unlim Cloud](Unlim%20Cloud%20%E2%80%94%20%C2%AB%D0%B1%D0%B5%D0%B7%D0%BB%D0%B8%D0%BC%D0%B8%D1%82%D0%BD%D0%BE%D0%B5%C2%BB%20%D0%BE%D0%B1%D0%BB%D0%B0%D0%BA%D0%BE%20%D0%B8%D0%B7%20Telegram%20%28%D0%BA%D0%B0%D0%BA%20%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%B0%D0%B5%D1%82%2C%20%D1%80%D0%B8%D1%81%D0%BA%D0%B8%2C%20%D0%B0%D0%BD%D0%B0%D0%BB%D0%BE%D0%B3%D0%B8%29.md) заставляют вводить **полный вход в аккаунт** (номер + код + 2FA) в **чужой веб-клиент**, код которого отдаёт сторонний сервер. Это разовый риск **угона всего аккаунта**. Эта заметка — про то, как получить то же самое **безопасно**.

> [!tip] Главный принцип
> **Никогда не вводи номер/код/2FA основного аккаунта в сторонний веб-клиент, код которого хостит кто-то другой.** Безопасность тем выше, чем меньше посредников между твоими данными и Telegram. Правильный порядок доверия: **свой официальный клиент → локальный open-source инструмент (код у тебя) → self-hosted веб-морда (сервер твой) → бот-токен (нет доступа к перепискам)**. Всё, что отдаёт исполняемый код с чужого сервера и требует твою пользовательскую сессию — небезопасно by design.

## 📊 Лимиты, о которых надо помнить

- **Размер одного файла:** **2 ГБ** (обычный аккаунт) / **4 ГБ** (Telegram Premium). Файлы больше — только с разбивкой на части (чанки); собирать обратно нужно тем же инструментом.
- **Bot API** (для бот-инструментов): скачивание — **до 20 МБ**, отправка — **до 50 МБ**. Поднять до 2 ГБ можно только со **своим локальным Bot API-сервером** (`telegram-bot-api`).
- **Fair-use.** Telegram — мессенджер, а не CDN. За массовые заливки/раздачу аккаунт и канал **банят**, а файлы удаляют. Для личного бэкапа — ок; «бесконечный сид на халяву» — путь к бану.
- **Шифрование.** «Сохранённые сообщения» — **не** секретный чат, для Telegram (и любого из инструментов ниже) файлы видны в открытом виде. Нужна приватность → **шифруй до загрузки** (`gpg`, `7z -p`, `rclone crypt`).

## 1️⃣ Официальные «Сохранённые сообщения» — самый безопасный

Просто перетащи файлы в **Saved Messages** в официальном клиенте. Нулевой посредник: официальный код, твоя сессия никуда не уходит. Минусы — нет «облачного» интерфейса (папки/галерея), ручная организация. Для бэкапа конфигов/архивов этого часто достаточно.

## 2️⃣ Локальные CLI — код и сессия остаются у тебя

Open-source утилиты, которые **ты запускаешь сам** на своей машине. Да, они логинятся твоим аккаунтом (MTProto), но код открыт, его **не подменят на лету**, а сессия лежит **локально**.

- **[tdl](https://github.com/iyear/tdl)** (`iyear/tdl`, **Go**, AGPL-3.0, ~7.7k★, живой) — мощный тулкит: заливка/скачивание файлов (в т.ч. из закрытых чатов), экспорт. Логин через QR или импорт сессии официального Telegram Desktop. Статический бинарь без зависимостей.
- **[telegram-upload](https://github.com/Nekmo/telegram-upload)** (`Nekmo/telegram-upload`, **Python**, MIT, ~1.4k★) — «залить/скачать файлы до 4 ГБ своим аккаунтом» из командной строки. Удобно для скриптов/бэкапов.

> [!tip] Замена самопального bash-скрипта
> Если раньше слал бэкапы ботом на 50 МБ (см. [Шаринг файлов с серверов в Телеграм](%D0%A8%D0%B0%D1%80%D0%B8%D0%BD%D0%B3%20%D1%84%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2-%D0%BF%D0%B0%D0%BF%D0%BE%D0%BA%20%D1%81%20%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80%D0%BE%D0%B2%20%D0%BF%D1%80%D1%8F%D0%BC%D0%BE%20%D0%BA%20%D1%81%D0%B5%D0%B1%D0%B5%20%D0%B2%20%D0%A2%D0%B5%D0%BB%D0%B5%D0%B3%D1%80%D0%B0%D0%BC.md)) — `tdl`/`telegram-upload` поднимают потолок до 2–4 ГБ и работают через твой аккаунт, а не Bot API.

## 3️⃣ Self-hosted веб-морда «как Google Drive»

То же удобство, что у Unlim Cloud (папки, галерея, веб-интерфейс), **но сервер поднимаешь ты** — и код, и сессия под твоим контролем, можно держать в локалке/за VPN.

- **[teldrive](https://github.com/tgdrive/teldrive)** (`tgdrive/teldrive`, **Go**, MIT, ~3k★, живой) — «обёртка над твоим Telegram-аккаунтом» с веб-UI; **совместима с rclone** (можно монтировать как диск). Требует PostgreSQL. Сам README прямо предупреждает: **не злоупотребляй Telegram API — забанят мгновенно**.
- **[TeleDrive](https://github.com/mgilangjanuar/teledrive)** (`mgilangjanuar/teledrive`, GPL-3.0, ~2k★) — аналог Google Drive/OneDrive на Telegram API. ⚠️ Заброшен (последний коммит 2023) — только если готов сам поддерживать.

> [!note] Чем это принципиально лучше Unlim Cloud
> В Unlim Cloud исполняемый код отдаёт **чужой сервер** — владелец в любой момент может подсунуть версию, ворующую сессию. В self-hosted варианте JS/бинарь **у тебя**; обновляешься тогда, когда сам решишь, и после проверки. Тот же UI — но контроль над кодом твой.

## 4️⃣ Бот-инструменты (bot token) — сессия не при делах

Бот **не имеет доступа к твоей переписке и контактам**, а токен **отзывается** одной командой у [@BotFather](https://t.me/BotFather). Компрометация пользовательской сессии в принципе исключена — самый «дуракоустойчивый» вариант по приватности.

Расплата — лимиты Bot API (**20 МБ** скачивание / **50 МБ** отправка). Для файлов до 2 ГБ поднимай **собственный локальный Bot API-сервер** (`telegram-bot-api` от Telegram). Простейший пример своими руками — [Шаринг файлов с серверов в Телеграм](%D0%A8%D0%B0%D1%80%D0%B8%D0%BD%D0%B3%20%D1%84%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2-%D0%BF%D0%B0%D0%BF%D0%BE%D0%BA%20%D1%81%20%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80%D0%BE%D0%B2%20%D0%BF%D1%80%D1%8F%D0%BC%D0%BE%20%D0%BA%20%D1%81%D0%B5%D0%B1%D0%B5%20%D0%B2%20%D0%A2%D0%B5%D0%BB%D0%B5%D0%B3%D1%80%D0%B0%D0%BC.md) (bash + `curl` + `sendDocument`).

## 🖥️ По платформам

| Система | Что подойдёт |
| :--- | :--- |
| **Gentoo** (основная) | `tdl` (готовый бинарь из релизов или `go install`); `telegram-upload` (`emerge dev-lang/python` + `pip`/venv); `teldrive` + PostgreSQL для веб-морды; `rclone` (`net-misc/rclone`) поверх teldrive |
| **Debian / Ubuntu** | То же: бинарь `tdl`, `pipx install telegram-upload`, `teldrive` (Docker) + Postgres, `apt install rclone` |
| **Arch** (план с июня 2026) | `tdl`/`telegram-upload` из AUR или бинарём; `teldrive` через Docker; `pacman -S rclone` |
| **Entware / RT-AX56U** (armv7) | Веб-морды (teldrive+Postgres) **не тянет**. Реально: `rclone` (`opkg install rclone`) и, **если есть linux/arm-сборка**, лёгкий `tdl` — тогда бэкапы с USB-диска роутера можно слать своим аккаунтом (до 2–4 ГБ) вместо 50-МБ бота. Проверь наличие arm-бинаря перед установкой |
