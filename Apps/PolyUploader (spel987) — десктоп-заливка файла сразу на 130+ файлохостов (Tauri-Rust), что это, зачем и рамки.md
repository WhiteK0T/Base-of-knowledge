# ⬆️ PolyUploader — залить файл сразу на 130+ файлохостов

**PolyUploader** ([github.com/spel987/PolyUploader](https://github.com/spel987/PolyUploader)) — **десктоп-приложение**, которое одним действием заливает файл (с диска или по URL) **сразу на 133 файлохостинга**. Ведёт историю загрузок (со сроком годности и кнопкой удаления), профили для частых задач, умеет собрать **одну общую ссылку-агрегатор** из кучи ссылок и «защитить» ссылки через PrivateBin/SafeLinking/Filecrypt. Без регистрации, GPL-3.0, ~363★, активный. По сути — открытый локальный аналог **MultiUp / Mirrored.to / Mirrorace**.

> [!info] Правки к посту: это Tauri-десктоп (Rust+JS), а не веб-сервис
> - Пост пишет «Lang: JavaScript» и «Rust backend» — на деле это **Tauri-приложение**: **Rust-бэкенд + JS/HTML/Tailwind-фронт**, ставится как обычная программа (Windows/Linux/macOS). Сайт `p-u.vercel.app` — это доки/редиректы, а не «через них грузишь».
> - **Плюс для приватности**: заливка идёт **прямо с твоей машины на хосты**, файл **не проходит через сервер разработчика** (в отличие от веб-мультизагрузчиков-посредников).

> [!caution] Что такое эти «130+ хостов» — трезво про экосистему
> Это **анонимные/бесплатные файлохостинги** (тот самый мир warez-зеркал, откуда растут MultiUp/Mirrored.to). Отсюда:
> - **Файлы становятся полу-публичными**: кто получил ссылку (или краулит хост) — скачает. **Никакого шифрования содержимого нет** — «link protection» прячет/обфусцирует *ссылки*, но не шифрует *файл*. **Не заливай туда личное/чувствительное.**
> - **Хосты недолговечны и разношёрстны**: у многих короткий срок хранения, лимиты размера, реклама, часть — мутные. Отсюда и смысл «залить на 130 сразу» — **избыточность/зеркалирование**: один хост умер — живут остальные.
> - **Легальная рамка**: мульти-аплоад-зеркала прочно ассоциируются с пиратством. Сам инструмент нейтрален; **законное применение** — раздать **свои** большие файлы с запасом по зеркалам. Заливка чужого копирайта — нарушение (и ToS хостов, и закона). Уважай [terms](https://p-u.vercel.app/terms), как просит автор.

> [!info] Полезные фишки (по README)
> - Заливка **с диска или по URL**; подключение **своих API-ключей** для совместимых хостов (лимиты/приоритет).
> - **История** с индикатором «жив/протух» и удалением, **профили** загрузки для автоматизации.
> - **Bundle-ссылка**: одна ссылка, разворачивающаяся в набор зеркал.
> - **Link protection**: PrivateBin, SafeLinking, Filecrypt — пачкой или по одной.
> - В README есть раздел **Telemetry** — если важно, загляни и отключи/проверь, что собирается.

## 🖥️ Применимость на системах владельца

| Система | Как использовать |
| :--- | :--- |
| **Gentoo / Debian-Ubuntu / Arch** | ✅ есть релизы **AppImage / `.deb` / `.rpm`**, либо собрать самому: Rust + `cargo install tauri-cli` → `cargo tauri build`. Под твой Linux — нормально |
| **Windows / macOS** | ✅ инсталлятор / `.dmg` (macOS-сборка **не подписана** → первый запуск через «Открыть» в обход Gatekeeper) |
| **Entware / RT-AX56U** | ➖ неактуально: это GUI-десктоп-приложение (Tauri/WebView), не для роутера |

> [!tip] Когда это тебе пригодится
> Разово **раздать свой большой файл** (бэкап, релиз, дистрибутив) с **избыточностью по зеркалам**, чтобы ссылка жила, даже если пара хостов отвалится. Для **приватного** обмена — не сюда (файл уходит на публичные хосты без шифрования); там лучше свой FileBrowser Quantum или зашифрованный контейнер.
