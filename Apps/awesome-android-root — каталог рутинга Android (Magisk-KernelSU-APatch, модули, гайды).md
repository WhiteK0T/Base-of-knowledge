# 🤖 awesome-android-root — каталог рутинга Android (модули, гайды, инструменты)

[**awesome-android-root**](https://github.com/awesome-android-root/awesome-android-root) (MIT, ~**3.6k★**) — большой регулярно обновляемый **awesome-каталог** по рутингу Android: **400+** инструментов, приложений и модулей + пошаговые руководства. Покрывает разблокировку загрузчика, кастомные recovery и выбор метода root (**Magisk / KernelSU / APatch**), а также подборки модулей и приложений (блокировка рекламы, тюнинг производительности, автоматизация, глубокая кастомизация системы). Есть удобный сайт-витрина: [awesome-android-root.pages.dev](https://awesome-android-root.pages.dev).

> [!info] Это каталог-документация, а не программа
> В посте указано «**Lang: Python**» — это лишь язык, который GitHub насчитал по вспомогательным скриптам. По сути репозиторий — **awesome-list + сайт на VitePress** (ссылки, таблицы, гайды), а не приложение для рутинга. Сам он ничего не «рутит»: даёт навигацию по чужим инструментам и инструкциям, которые ты применяешь **на свой страх и риск**.

| **Гайды** | пошаговые инструкции для пользователей и разработчиков модулей |

## 🧪 Факты против хайпа

> [!warning] «400+ инструментов» — это ссылки, качество и безопасность не гарантированы
> Awesome-список — это **агрегатор**. Полезен как карта местности, но:
> - **device-specific**: неправильный recovery/kernel/`boot.img` под твою модель → **кирпич**. Всегда сверяй с темой именно по своему устройству (XDA/4PDA), а не «по аналогии».
> - модули и root-приложения — **сторонний код**, и после root он работает с **максимальными привилегиями**. Ставь только из доверенных источников, проверяй репутацию (это **supply-chain-риск**: вредоносный модуль = полный контроль над телефоном).

> [!caution] Что ломает root (трезво о последствиях)
> - **Разблокировка загрузчика стирает все данные** (factory reset) — бэкап обязателен.
> - **Гарантия**: разблокировка bootloader у многих вендоров снимает гарантию.
> - **Play Integrity / SafetyNet**: банковские приложения, платежи (Google Pay), часть игр и стриминга (DRM Widevine L1) могут **перестать работать** или требовать обхода — это вечная «гонка», не гарантия.
> - **OTA-обновления** обычно ломаются — обновляться придётся вручную.
> - **Безопасность**: root ослабляет песочницу Android; одно вредоносное приложение с выданным root = компрометация всего устройства.

> [!note] Право (РФ и не только)
> Рутинг **собственного** устройства сам по себе законен — это твоя вещь. Серые зоны начинаются дальше: **обход DRM/лицензионных защит**, пиратство платных приложений, нарушение пользовательских соглашений сервисов. Каталог — про кастомизацию своего телефона; всё, что касается чужого контента/чужих устройств, — уже вне «просто root».

## 💻 Как это ложится на твои системы

Рутинг происходит **на Android-устройстве**, но флешат его обычно с десктопа через `adb`/`fastboot`:

| Система | Роль |
| :--- | :--- |
| **Gentoo** | `emerge dev-util/android-tools` (adb/fastboot) для разблокировки/прошивки с ПК |
| **Debian / Ubuntu** | `sudo apt install android-tools-adb android-tools-fastboot` |
| **Arch** | `sudo pacman -S android-tools` |
| **Entware / RT-AX56U** | **N/A** — рутинг телефона роутером не делают; `android-tools` под этот ARM в репозитории нет смысла искать |

> [!tip] Без ПК — тоже можно
> Magisk-патч `boot.img` и KernelSU/APatch часто ставятся **прямо с телефона** через их приложения (после разблокировки загрузчика). Но первый раз безопаснее с десктопом и `fastboot` под рукой — если что-то пойдёт не так, проще восстановить.

## 💡 Кому полезно

- **Энтузиастам кастомизации Android**, кто осознанно идёт на root ради модулей, тюнинга и автоматизации.
- **Разработчикам модулей** — гайды и примеры под Magisk/KernelSU/APatch.
- **Не** новичкам «по приколу»: цена ошибки — кирпич/потеря данных/неработающий банк-клиент. Сначала почитай про **своё** устройство.
