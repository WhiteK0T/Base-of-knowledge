# 🖥️ ScreenConnect — удалённый доступ и поддержка

**ScreenConnect** (ранее *ConnectWise Control*, разработчик — **ConnectWise**) — коммерческое ПО для удалённого доступа, удалённой поддержки и управления устройствами. Популярно у MSP и IT-отделов: техник подключается к экрану пользователя, управляет машиной, передаёт файлы и выполняет команды.

## 🔑 Ключевые возможности

- **Удалённое управление** экраном с поддержкой нескольких мониторов.
- **Attended (с участием пользователя)** и **unattended (без участия)** сессии — постоянный доступ к серверам/рабочим станциям.
- **Передача файлов**, выполнение **фоновых команд**, аннотирование экрана без прерывания пользователя.
- **White-label** — полный брендинг интерфейса под себя.
- Кроссплатформенность (Windows, macOS, Linux), доступ из браузера.

## 🚀 Варианты развёртывания

| Модель | Особенности |
| :--- | :--- |
| **Self-hosted** | Relay-сервер на своей инфраструктуре, все данные подключений под твоим контролем (основное преимущество продукта) |
| **Cloud** | Хостинг на стороне ConnectWise, без своего сервера |

## 💲 Цены (2026)

Лицензирование — **за конкурентного техника** (concurrent), оплата годовая. Планы примерно $30–$55/мес: One ($30), Remote Access ($33), Standard ($45), Premium ($55). Линейка **Unattended Access** тарифицируется **по устройствам** (от ~$30/мес за 25 устройств).

## 🛠️ Установка self-hosted

Закрытый продукт официального open-source репозитория не имеет. Дистрибутив (инсталлятор/контейнер) скачивается с **https://control.connectwise.com** — нужна регистрация даже для on-premises.

**Системные требования:**
- **Windows:** любой современный Windows Server (или Windows для теста) — запустить установщик `.msi`.
- **Linux:** архитектура **x86_64**, **glibc ≥ 2.17**; установка через официальный installer-скрипт.
- **Порты:** `8040` — веб-интерфейс, `8041` — relay (через них идут клиенты); открыть на firewall.

**Быстрый старт в Docker:**

```bash
# официальный образ
docker run -d --name screenconnect -p 8040:8040 -p 8041:8041 connectwise/control
```

После старта открыть `http://<server-ip>:8040` и пройти первичную настройку (админ-аккаунт, лицензия).

**Community-репозитории (Docker):**
- [kenayagi/docker-screenconnect](https://github.com/kenayagi/docker-screenconnect) — дженерик-образ ScreenConnect в Docker.
- [grolston/screenconnect](https://github.com/grolston/screenconnect) — ещё один Docker-образ ConnectWise Control.
- Организация на GitHub: [ConnectWise-ScreenConnect](https://github.com/ConnectWise-ScreenConnect) (расширения/интеграции, не сам сервер).

> [!tip] Лицензия
> Self-hosted-сервер ставится бесплатно для теста, но для реальной работы требует лицензионного ключа (триал → платная подписка). Без активной лицензии число одновременных техников/сессий ограничено.

> [!warning] Безопасность
> ScreenConnect — фактически легитимный RAT, поэтому **активно злоупотребляется мошенниками** (techsupport-скам) и попадает в инцидентах как инструмент удалённого доступа атакующих. Известна критическая уязвимость **CVE-2024-1709** (обход аутентификации в self-hosted). Рекомендации: своевременно обновлять self-hosted сервер, включать **MFA**, ограничивать доступ по IP, и с подозрением относиться к просьбам «установить ScreenConnect для помощи» от незнакомцев.
