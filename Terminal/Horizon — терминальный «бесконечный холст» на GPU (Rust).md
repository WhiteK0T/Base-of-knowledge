# 🪟 Horizon — терминальный «бесконечный холст» на GPU (Rust)

**Horizon** ([github.com/peters/horizon](https://github.com/peters/horizon), автор `peters`, MIT) — GPU-ускоренный «терминальный борд»: вместо вкладок и тайлов даёт **бесконечный 2D-холст**, где каждая сессия — это **панель**, которую можно ставить, двигать, ресайзить и группировать, как стикеры на whiteboard. Написан на **Rust**, рендер через **wgpu**. Молодой проект (~654★, v0.2.6, май 2026), собран автором «за 3 дня» с помощью [Claude Code](../AI/Claude%20Code%20%E2%80%94%20%D0%B3%D0%B0%D0%B9%D0%B4.md)/Codex.

> [!warning] Уточнение к посту — это СВОЙ терминал, а не «интеграция с популярными эмуляторами»
> Пост пишет, что Horizon «интегрируется с популярными эмуляторами». Точнее: это **самостоятельный терминал-эмулятор**, который внутри использует **движок `alacritty_terminal`** (библиотеку от [Alacritty](Alacritty%20%E2%80%94%20GPU-%D1%82%D0%B5%D1%80%D0%BC%D0%B8%D0%BD%D0%B0%D0%BB%20%D0%BD%D0%B0%20OpenGL%20%28%D0%BC%D0%B8%D0%BD%D0%B8%D0%BC%D0%B0%D0%BB%D0%B8%D0%B7%D0%BC%29.md)) — он **не плагин** к kitty/Alacritty. А вот что он реально интегрирует — это **AI-CLI-агентов** (Claude Code, OpenCode, Gemini CLI, KiloCode).

> [!info] wgpu и движок Alacritty
> Рендер — через **wgpu** (Vulkan/Metal/DX12/OpenGL), UI — immediate-mode **egui/eframe**, терминальная эмуляция — **alacritty_terminal**, палитра — Catppuccin. По связке «GPU-рендер + переиспользование кода Alacritty» идейно близок к [Rio](Rio%20%E2%80%94%20%D0%B1%D1%8B%D1%81%D1%82%D1%80%D1%8B%D0%B9%20%D1%82%D0%B5%D1%80%D0%BC%D0%B8%D0%BD%D0%B0%D0%BB%20%D0%BD%D0%B0%20WebGPU%20%28Rust%29.md), но другой парадигмы интерфейса (холст вместо вкладок).

## ✨ Возможности

- **Бесконечный холст:** пан/зум по безграничной поверхности, панели где угодно, **миникарта** в углу для ориентации.
- **Воркспейсы:** группировка панелей в цвето-кодированные пространства, авто-раскладки **rows / columns / grid** или свободное перетаскивание; Quick Nav для прыжков.
- **Полная терминальная эмуляция:** 24-битный цвет, поддержка мыши.
- **AI-агенты из коробки:** Claude Code, OpenCode, Gemini CLI, KiloCode.
- **Git-панель** со статусом и inline-диффами.
- **SSH:** обнаружение хостов и управление подключениями.
- **Персистентность сессий** между перезапусками.
- **Live-редактор настроек** (YAML с мгновенным превью) и **Markdown-редактор** со сплит-вью.

> [!note] Чем это отличается от tmux
> [tmux](tmux%20%E2%80%94%20%D1%82%D0%B5%D1%80%D0%BC%D0%B8%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9%20%D0%BC%D1%83%D0%BB%D1%8C%D1%82%D0%B8%D0%BF%D0%BB%D0%B5%D0%BA%D1%81%D0%BE%D1%80%20%28%D1%81%D0%B5%D1%81%D1%81%D0%B8%D0%B8%2C%20%D0%BE%D0%BA%D0%BD%D0%B0%2C%20%D0%BF%D0%B0%D0%BD%D0%B5%D0%BB%D0%B8%29.md) — это **текстовый** мультиплексор, который живёт **на сервере** и переживает обрыв SSH (сессии остаются на удалённой машине). Horizon — **локальное GUI-приложение** с GPU: его «персистентность» про **локальную** раскладку панелей, а не про серверные сессии. Для удалённой персистентности по-прежнему нужен tmux/screen на хосте; Horizon — про **визуальную организацию** множества терминалов на своей машине.

## 📦 Установка

Целевые десктоп-системы владельца (Gentoo, Debian/Ubuntu, Arch). Готовые бинарники: **Linux x64, macOS (arm64/x64), Windows x64**.

```bash
# Debian/Ubuntu — Snap (проверь имя: snap find horizon) либо бинарь из релизов GitHub
sudo snap install horizon

# Arch — готовый бинарь из релизов, AUR (если появится) или сборка через cargo (ниже)

# Gentoo — пакета в дереве нет: собрать из исходников
#   нужны Rust 1.88+, Git LFS и системные dev-заголовки для GPU-рендера
git lfs install
cargo install --git https://github.com/peters/horizon
#   или: git clone … && cd horizon && cargo run --release

# macOS / Linux — Homebrew
brew install horizon          # точное имя/тап уточни на странице релизов
```

> [!danger] Не для роутера/Entware
> Horizon — **графическое GPU-приложение** (нужны видеокарта с поддержкой wgpu, дисплей, на Linux — dev-заголовки для рендера). На ASUS RT-AX56U (Entware, без графики) **неприменимо**. К роутеру подключайся по SSH уже из терминала на десктопе.

## ⚖️ Трезвая оценка

> [!caution] Очень молодой проект
> v0.2.x, ~654★, написан «за 3 дня» с AI-ассистентом — это **ранняя альфа-стадия**. Ждать стабильности уровня Alacritty/kitty/tmux не стоит: возможны баги, ломающие изменения, прожорливость GPU/RAM на многих панелях. Любопытная идея «терминалы на бесконечном холсте» — да; ежедневный надёжный инструмент — пока под вопросом. Бэкапь конфиг, не клади критичные long-running процессы только в него (для этого — tmux на сервере).

## 💡 Кому интересно

- **Любишь визуальную организацию** и работаешь сразу с кучей терминалов/агентов — холст + воркспейсы + миникарта могут зайти.
- **Гоняешь AI-CLI** (Claude Code, Gemini CLI, OpenCode) и хочешь их в одном борде с git-панелью и SSH.
