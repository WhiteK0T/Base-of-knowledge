# 📡 WifiForge (Black Hills InfoSec) — виртуальная лаборатория для обучения Wi-Fi-пентесту

[**WifiForge**](https://github.com/blackhillsinfosec/WifiForge) (**Black Hills InfoSec**, **Python**, лицензия **Apache-2.0**, ~1.05k★, v3.4.1 сент. 2025) — учебная **виртуальная лаборатория** для отработки атак на Wi-Fi. Главное, что стоит подчеркнуть (пост это смазал): **это песочница**, а не инструмент против реальных сетей — всё крутится **виртуально на базе [mininet-wifi](https://github.com/intrig-unicamp/mininet-wifi)**, без радио-железа и без чужих точек доступа.

> [!info] Ключевое: полностью виртуально, «safe and legal»
> Авторы прямо пишут: *«WifiForge provides a safe and legal environment for learning WiFi hacking»*. Проект **автоматически поднимает виртуальные сети и инструменты** для набора лабораторок по эксплуатации Wi-Fi — **не нужно ни адаптеров/антенн, ни реальных целей**. Ты атакуешь **симулированные** точки доступа и клиентов внутри своей машины.

## 🧪 Что даёт

- **Готовые лабы** по типовым векторам: перехват **WPA/WPA2-handshake**, **деаутентификация**, **PMKID**, **evil twin** (злой двойник).
- **Интеграция с боевыми инструментами** (в виртуалке): `aircrack-ng`, `wifiphisher`, `john-the-ripper`, `arpspoof`, экосистема `airgeddon`.
- **Автосетап** окружения — не надо вручную городить mininet-wifi и раскладывать тулзы.
- Цель — **тренировка пентестеров и обучение** беспроводной безопасности «как это работает» на понятных сценариях.

## ✅ Почему это «чистый» способ учиться

> [!note] Никаких реальных сетей — значит, без правовых рисков от самого тренажёра
> Поскольку атаки идут по **виртуальным** узлам на твоём же компьютере, **никакого неправомерного доступа к чужим сетям нет** — тренажёр законен by design. Это выгодно отличает WifiForge от «поставил Kali + внешний адаптер и пошёл по эфиру».

> [!caution] Но навыки применяются к реальному эфиру — там уже закон
> Приёмы, отработанные в лабе, на **реальных чужих** Wi-Fi — это уже:
> - **перехват/дешифровка чужого трафика** → тайна связи, **ст. 138 УК РФ**;
> - **неправомерный доступ** к сети/данным → **ст. 272 УК РФ**;
> - использование спец-техсредств для негласного получения информации → **ст. 138.1 УК РФ**.
> Легально «в поле» — только **своя** сеть/оборудование или сеть с **письменного разрешения** владельца (пентест по договору). Лаборатория — чтобы учиться, а не чтобы потом «проверить соседский роутер».

## ⚙️ Требования и запуск

- **Linux + Docker + mininet-wifi**, нужен **root/sudo**.
- Проект помечен как **нестабильный (dev-стадия)** — авторы советуют запускать **только внутри ВМ** (заодно и изоляция).
- Работает на Python; ставится из репозитория, поднимает контейнеры/топологию сам.

## 🖥️ По системам

| Система | Применимость |
| :--- | :--- |
| **Gentoo** (основная) | ✅ Хост под ВМ или напрямую: Docker + mininet-wifi + Python; удобнее в отдельной **Linux-ВМ** (совет авторов) |
| **Debian / Ubuntu** | ✅ Наиболее «родная» среда для mininet-wifi (обычно тестируют на Ubuntu) — проще всего развернуть в ВМ |
| **Arch** (план с июня 2026) | ✅ Docker + mininet-wifi из AUR/исходников; тоже лучше в ВМ |
| **Entware / RT-AX56U** | **N/A** — нужен Linux-десктоп/сервер с Docker и mininet-wifi; на роутере не запустить. (Твой RT-AX56U — это **реальная** Wi-Fi-цель для практики «в своей сети», но это уже вне песочницы WifiForge) |
