# 🔍 Web Check — OSINT-досье на сайт (DNS/SSL/заголовки/стек/порты)

**Web Check** (`Lissy93/web-check`, автор **Alicia Sykes**, MIT, ~34k ★) — «all-in-one OSINT-инструмент для анализа сайта»: по домену за секунды собирает **40+ проверок** — инфраструктура, DNS/SSL, заголовки, cookies, трекеры, технологический стек, производительность, открытые порты и пр. Есть публичный инстанс [web-check.xyz](https://web-check.xyz/) и self-host. Пост [@proglibrary](https://t.me/proglibrary/11653) назвал его #musthave для «быстро разобраться в чужом проекте или проверить свой».

> [!warning] Что это на самом деле — и его границы
> - Это в основном **пассивный OSINT**: тянет **публично доступные** данные (DNS-записи, SSL-цепочка, HTTP-заголовки, whois, sitemap, определённый по фронту стек, трекеры). Никакого «взлома» — то же, что видно любому из интернета.
> - **Сканирование открытых портов** — единственная **активная** часть: это внешнее зондирование. На своих ресурсах — без проблем; по чужим — будь аккуратен (см. правовую оговорку).
> - **«Досье на ЛЮБОЙ сайт за пару секунд»** — с оговорками: часть продвинутых проверок требует **API-ключей** (Shodan, BuiltWith, URLScan, SecurityTrails, Google Cloud), у публичного инстанса есть **таймаут ~25с и рейт-лимиты**, а «технологии» определяются эвристикой по фронтенду (может ошибаться).

## 🧰 Что проверяет (40+ модулей, по категориям)

| Категория | Примеры проверок |
| :--- | :--- |
| **Инфраструктура / сеть** | IP-инфо, геолокация сервера, **открытые порты**, traceroute, связанные хосты, статус/инфо сервера |
| **DNS / почта** | DNS-записи, **DNSSEC**, DMARC/SPF/DKIM/BIMI, security.txt |
| **SSL / TLS** | цепочка сертификатов, наборы шифров, конфиг TLS, симуляция handshake, **HSTS** |
| **HTTP / контент** | заголовки, **security-заголовки**, cookies, robots.txt, sitemap, ссылки, соц-теги |
| **Стек / трекеры** | определение **tech stack**, сторонние сервисы/трекеры, фичи сайта |
| **Производительность** | метрики качества, скорость страницы, скриншот, «углеродный след» |
| **Домен / репутация** | whois, история (archive), глобальный рейтинг, цепочка редиректов |
| **Угрозы** | детект malware/фишинга, блок-листы (privacy/malware DNS) |

## 🔒 Приватность: публичный инстанс vs self-host

> [!caution] Публичный web-check.xyz видит, что ты проверяешь
> Когда вбиваешь домен на [web-check.xyz](https://web-check.xyz/), **цель уходит на сторонний хостинг** (и через его API-интеграции дальше). Для разведки по публичным сайтам это ок, но для **внутренних/чувствительных** хостов или приватных проектов — **подними свой инстанс**, чтобы запросы не утекали третьей стороне.

### Self-host (рекомендуется для приватности)

Нужен Docker **или** Node.js ≥ 18.16 + Yarn (опц. Chromium/traceroute/dns-utils для части модулей). Самый простой путь — **Docker**, он одинаков на десктоп-системах владельца:

| Система | Запуск |
| :--- | :--- |
| **Gentoo** | `emerge app-containers/docker` → `docker run -p 3000:3000 lissy93/web-check` |
| **Debian / Ubuntu** | `sudo apt install docker.io` → `docker run -p 3000:3000 lissy93/web-check` |
| **Arch** | `sudo pacman -S docker` → `docker run -p 3000:3000 lissy93/web-check` |
| **Entware / RT-AX56U** | **N/A** — нужен Node 18+/Chromium, тяжело для 256 МБ flash; гоняй на десктопе/сервере |

Альтернативы: one-click деплой на **Netlify/Vercel**, либо из исходников (`yarn install && yarn build && yarn serve`). Продвинутые проверки включаются **API-ключами** через переменные окружения (Shodan и т.д.).

## ⚖️ Правовая/этическая оговорка

> [!note] OSINT по публичным данным — норм; зондирование портов — с головой
> Сбор **публичной** информации (DNS, SSL, заголовки, whois) законен. **Скан открытых портов** — это уже активное взаимодействие с чужим сервером: на **своих** ресурсах делай свободно; по чужим инфраструктурам массово/агрессивно не гоняй (web-check шлёт лёгкие пробы, но ответственность на тебе). Идеальный сценарий — **аудит собственных сайтов** (SSL, security-заголовки, утечки в DNS/почте, лишние трекеры).

## 💡 Кому полезно

- **Себе — аудит:** быстро проверить свой сайт на SSL/HSTS/security-заголовки, корректность SPF/DMARC, лишние трекеры, открытые порты.
- **Recon/OSINT:** быстрый профиль чужого публичного сайта (стек, инфраструктура, редиректы) — разведка перед более глубоким анализом.
- **Связка:** для приватности своих проверок — self-host; принцип «не сливай цель третьей стороне» тот же, что у локального запуска LLM.
