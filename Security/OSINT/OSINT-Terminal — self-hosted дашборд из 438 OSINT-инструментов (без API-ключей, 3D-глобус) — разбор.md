# 🛰️🌐 OSINT-Terminal — self-hosted дашборд OSINT-инструментов (разбор)

[**OSINT-Terminal**](https://github.com/RojanSapkota/osint-terminal) (*Rojan Sapkota*, **Python**, лицензия **MIT**) — **self-hosted веб-дашборд**, собирающий в одном интерфейсе **438 «инструментов» разведки** (сеть, домены, email, телефон, крипта, геоданные, threat-intel), интерактивный **3D-глобус** с живыми слоями (землетрясения, самолёты, C2-инфраструктура, МКС) и группировку результатов по **кейсам/проектам**. Запускается локально (`python server.py` → `http://localhost:8080`), есть Docker. Заявка — «всё в одном месте и без API-ключей». Идея удобная, но у неё есть важные оговорки, которые пост опускает.

> [!warning] Проекту 1 день и 20 звёзд — это не проверенный инструмент
> Репозиторий **создан 2 июля 2026** (буквально накануне поста), ~**20★**, **один разработчик**, **без аудита безопасности**. Это ранний релиз-«проба», а не зрелый OSINT-фреймворк уровня SpiderFoot/Maltego/recon-ng. Оценивай соответственно: гонять в проде/на рабочей машине непроверенный сервер, который **сам ходит в интернет**, — на свой риск. Запускай изолированно (VM/контейнер), читай код.

## 🧪 Факты против хайпа

> [!danger] «Всё локально — данные не покидают вашу машину» — это неверно
> Главная поправка. **438 «инструментов» — это функции-обёртки, которые дёргают внешние бесплатные публичные сервисы** (USGS, OpenSky, abuse.ch / Feodo Tracker, crt.sh и т.п.). Локально хранится только **история и кэш** (JSON в `data/`, TTL по умолчанию 3600 c). А **сами запросы разведки — цели: домены, email, телефоны, IP, крипто-адреса — уходят на сторонние сервисы**. То есть «данные не покидают машину» верно лишь для *журнала действий*, но **не** для того, что ты ищешь. Для OSINT это принципиально: твои цели видны третьим сторонам.

> [!note] «Без API-ключей» — правда, но у этого есть цена
> Ключи не нужны, потому что дашборд стучится **только в публичные эндпоинты** без авторизации. Обратная сторона: **rate-limit и блокировки** при активном использовании (сервисы режут анонимный трафик), **хрупкость** (эндпоинт сменился — «инструмент» отвалился) и **read-only** характер данных. Это не 438 полноценных утилит, а **438 обращений к чужим API/страницам** через единый UI.

> [!caution] 3D-глобус красив, но это внешние API + CDN, а не «offline»
> Глобус рисуется через **globe.gl, подгружаемый с CDN**, а слои (землетрясения USGS, самолёты OpenSky, C2 Feodo Tracker, позиция МКС) — **живые внешние API**. Значит, для картинки нужен интернет и внешний CDN; полностью «оффлайн/приватным» дашборд не является. Это скорее эффектная визуализация threat-intel, чем инструмент расследования.

> [!tip] Про «запускайте через VPN» — да, но это не делает его приватным
> Совет из поста разумен как OPSEC: VPN **скрывает твой IP** от опрашиваемых сервисов. Но он **не** устраняет главную проблему — **цели твоих запросов всё равно уходят третьим сторонам** (и часть lookup'ов может быть заметна самому объекту, например обращения к его инфраструктуре). VPN — это один слой; полноценная гигиена: изолированная ВМ/контейнер, понимание, *какой* инструмент *куда* ходит, и осознанный выбор, что искать через посредника.

> [!caution] Право и этика (защитный контекст)
> В самом README есть оговорка «**только для авторизованных исследований**». OSINT по публичным данным сам по себе легален, но **профилирование людей** легко переходит в нарушение приватности/сталкинг, а разведка **чужой инфраструктуры** без разрешения — в неправомерный доступ. В РФ учитывай **152-ФЗ** (персональные данные). Используй для **своих**/санкционированных целей (свой периметр, разрешённый пентест, журналистика по открытым источникам), не для слежки за людьми. Наступательных how-to здесь нет.

## 💻 Как это ложится на твои системы

Бэкенд лёгкий: **stdlib `http.server`** + `requests` (опционально `Pillow`, `phonenumbers`). Ставится где есть Python 3. UI открываешь в браузере.

| Система | Установка / применимость |
| :--- | :--- |
| **Gentoo** (основная) | ✅ `git clone` + venv/pipx (PEP 668): `python -m venv venv && ./venv/bin/pip install -r requirements.txt && ./venv/bin/python server.py`. Лучше в контейнере/ВМ — сервер ходит в сеть |
| **Debian / Ubuntu** | ✅ `python3-venv` + venv, или Docker-версия для быстрого старта и изоляции |
| **Arch** | ✅ venv/pipx или Docker; браузер на `localhost:8080` |
| **Entware / RT-AX56U** | ⚠️ Технически возможно и уместно: лёгкий stdlib-сервер потянет `opkg install python3 python3-pip` + `pip3 install requests` → дашборд крутится **на всегда-включённом роутере**, открываешь с любого устройства в LAN. Но: 3D-глобус тянет CDN + браузер у тебя на ПК; запросы разведки **всё равно уходят наружу** (та же приватностная оговорка), а роутер — не место для «грязного» трафика расследований. Ниша, но реальная |

## 💡 Кому полезно

- **OSINT-аналитикам/пентестерам (в рамках авторизации), журналистам:** удобный единый UI-агрегатор публичных lookup'ов вместо 20 вкладок — для быстрой первичной разведки.
- **Не** замена зрелым фреймворкам (SpiderFoot, recon-ng, Maltego) и **не** «приватный локальный» инструмент: помни, что запросы уходят третьим сторонам.
- **Не** для слежки за людьми: только санкционированные/собственные цели.
