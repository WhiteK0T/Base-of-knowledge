# 🔎 OpenSERP — self-hosted SERP-API и CLI (Google/Yandex/Baidu и др.)

[**OpenSERP**](https://github.com/karust/openserp) ([karust](https://github.com/karust/openserp), **Go**, лицензия **MIT**, ~1.05k★, v0.8.6 июнь 2026) — открытый **self-hosted API и CLI**, который отдаёт результаты поиска из **Google, Yandex, Baidu, Bing, DuckDuckGo и Ecosia** в машинно-читаемом виде. По сути — бесплатная self-hosted замена платным SerpAPI/serper для **OSINT-разведки, «заземления» LLM (grounding/RAG), SEO-мониторинга и автоматизации**.

## 🧬 Что умеет

- **Много движков, единый формат.** У каждого движка свой эндпоинт + `/mega/search` — параллельный запрос по нескольким сразу с **кросс-движковой дедупликацией** (кластеризация по нормализованному URL).
- **Извлечение страниц.** Может вернуть не только ссылки, но и **чистый markdown/текст целевой страницы за один вызов** — удобно для подачи в LLM.
- **SERP-фичи:** AI-резюме, «блоки ответов», «люди также спрашивают» (People Also Ask).
- **Поиск изображений** и **фильтры**: язык, диапазон дат, тип файла, ограничение по сайту (`site:`).
- **Форматы запуска:** бинарь/из исходников, **Docker** (`karust/openserp`), либо **managed-облако** (openserp.org/cloud), если не хочется хостить.

> [!info] Как это работает под капотом
> OpenSERP **не использует официальные API** поисковиков — он их **скрапит через браузерный рендеринг** (headless). Отсюда и плюсы (доступ к «живой» выдаче, RU/CN-движки), и минусы (см. ниже): **капчи, баны по IP, поломки** при изменениях на стороне поисковика. Поддерживает **HTTP/SOCKS5-прокси** и «resilient mode» для устойчивости; при капче отдаёт `503 captcha detected`.

## 💡 Чем полезно именно тебе

- **Покрытие Yandex и Baidu** — редкость: большинство SERP-API только про Google. Для RU/CN-контента это реальный плюс.
- **Заземление локальных LLM.** Дать [Ollama](../../AI/Local-LLM/Ollama%20%E2%80%94%20%D0%BC%D0%B5%D0%BD%D0%B5%D0%B4%D0%B6%D0%B5%D1%80%20%D0%B8%20%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80%20%D0%BB%D0%BE%D0%BA%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D1%85%20LLM.md)/агенту **веб-поиск** без платного SerpAPI и без утечки запросов стороннему вендору — всё на своём сервере.
- **Приватность:** self-host = поисковые запросы не уходят облачному SERP-провайдеру.

## ⚖️ Правовые и практические нюансы

> [!caution] В README нет ни слова про право — а оно тут важно
> OpenSERP — это **скрапинг поисковиков**, и это надо учитывать:
> - **Нарушение ToS.** Автоматический парсинг Google/Bing/Yandex почти наверняка **против их условий использования** — отсюда капчи и баны IP. Твой IP/прокси могут заблокировать.
> - **152-ФЗ / персональные данные.** Массовый сбор выдачи ради **агрегации персональных данных** людей — это уже обработка ПДн со всеми обязанностями; для «пробива» людей — незаконно. Используй для **тематического ресёрча, мониторинга своих сайтов, подачи в LLM**, а не для профилирования людей.
> - **Нагрузка и этика.** Не «долби» движки на скорости — это и бан быстрее, и нагрузка на чужие сервисы. Разумный rate + прокси, а не «выкачать интернет».

> [!note] «Бесплатно», но не «бесплатно и стабильно»
> Скрапинг хрупок: поисковики меняют вёрстку/антибот — парсер периодически ломается и требует обновлений. Для продакшена нужны прокси-пул и мониторинг. Managed-облако снимает эти хлопоты, но это уже не self-host.

## 🖥️ По системам

| Система | Запуск |
| :--- | :--- |
| **Gentoo** (основная) | Go-бинарь из релизов или `go build`; для browser-rendered режима нужен **headless Chromium** (`emerge www-client/chromium` или chromium-headless-shell). Проще — **Docker** |
| **Debian / Ubuntu** | Бинарь/`docker run karust/openserp`; Chromium для рендеринга ставится как зависимость |
| **Arch** (план с июня 2026) | Бинарь или `docker`; `pacman -S chromium` для рендеринга |
| **Entware / RT-AX56U** | ⚠️ Go-бинарь под arm теоретически соберётся, но **headless Chromium на роутере (armv7, 512 МБ) не потянуть** → browser-rendered скрапинг **не поедет**. Держи OpenSERP на десктопе/сервере, а роутер пусть только ходит к его API |
