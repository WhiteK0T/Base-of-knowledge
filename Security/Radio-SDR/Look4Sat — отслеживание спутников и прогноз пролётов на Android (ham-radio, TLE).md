# 🛰️ Look4Sat — отслеживание спутников и прогноз пролётов на Android (ham-radio, TLE)

[**Look4Sat**](https://github.com/rt-bishop/Look4Sat) (rt-bishop, **Kotlin**, лицензия **GPL-3.0**, ~1168★, v4.4.2) — открытое **Android-приложение для отслеживания спутников и прогноза их пролётов**. Вдохновлено настольным **Gpredict**: показывает, когда и где над тобой пройдёт спутник, его полярную траекторию, зону покрытия (footprint) и **информацию о передатчиках** (частоты). Ориентировано на **радиолюбителей (ham radio), SDR и спейс-энтузиастов**.

## ✨ Возможности

- **Прогноз пролётов до 10 дней** вперёд (в посте — «до недели»; по README — **до 10 суток**).
- Список **активных и предстоящих пролётов**, прогресс прохода, **полярная траектория**.
- Позиция спутника, **footprint** и наземный трек на карте.
- **9000+ активных спутников**: данные **TLE** из **Celestrak** и **SatNOGS**; импорт своих TLE из `.txt`.
- Поиск по **номеру NORAD** или имени; информация о **передатчиках/частотах** (полезно, чтобы поймать сигнал).
- **Offline-first:** все расчёты — локально, интернет нужен **только для обновления TLE** (рекомендуется раз в неделю).
- Современный стек: Kotlin, Coroutines, Jetpack Compose.

> [!info] Что такое TLE и почему их надо обновлять
> **TLE (Two-Line Element set)** — стандартный формат орбитальных параметров спутника. Орбиты медленно «уходят» из-за атмосферного торможения и манёвров, поэтому точность прогноза падает со временем — **обновляй TLE раз в ~неделю** для актуальных пролётов.

> [!note] Это трекер/предсказатель, а не «пульт»
> Look4Sat говорит, **когда и на какой частоте** искать спутник. Это не управление приёмником/антенной и не сам приём сигнала — для реального приёма нужен SDR-приёмник (RTL-SDR и т.п.) и антенна; Look4Sat — планировщик «когда слушать».

## 📦 Установка (Android)

| Способ | Как |
| :--- | :--- |
| **F-Droid** | рекомендую для open-source (чистая GPL-сборка без трекеров) |
| **Google Play** | [страница приложения](https://play.google.com/store/apps/details?id=com.rtbishop.look4sat) |
| **APK из релизов** | [GitHub Releases](https://github.com/rt-bishop/Look4Sat/releases) (v4.4.2) |
| **Из исходников** | Kotlin/Gradle-проект: `git clone … && ./gradlew assembleRelease` (Android SDK + JDK 17) |

> [!tip] Доступы
> Нужен **доступ к геолокации** (для расчёта пролётов относительно твоей точки). Интернет — только на обновление TLE; дальше работает офлайн.

## 🖥️ На десктопе владельца — аналог Gpredict

Самого Look4Sat на ПК нет (это Android-приложение), но его «прародитель» **Gpredict** ставится на все системы владельца:

| Система | Установка Gpredict |
| :--- | :--- |
| **Gentoo** (основная) | `emerge sci-geosciences/gpredict` |
| **Debian / Ubuntu** | `sudo apt install gpredict` |
| **Arch** (план с июня 2026) | `sudo pacman -S gpredict` (или AUR) |
| **Entware / RT-AX56U** | **N/A** — GUI-приложение, на роутер не ставится; для голого расчёта пролётов на embedded есть библиотеки (`predict`/`pyephem`), но это уже не то |

## 💡 Кому полезно

- **Радиолюбителям/SDR:** поймать любительские спутники (AMSAT), метео (NOAA/Meteor), ISS — знать окно пролёта и частоту.
- **Спейс-энтузиастам:** увидеть пролёт МКС/Starlink над своей точкой.
- **Не** для управления железом и **не** сам приёмник — это планировщик пролётов.
