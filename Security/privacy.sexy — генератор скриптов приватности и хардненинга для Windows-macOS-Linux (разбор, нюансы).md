# 🕶️ privacy.sexy — генератор скриптов приватности и хардненинга (Windows/macOS/Linux)

[**privacy.sexy**](https://privacy.sexy/) ([undergroundwires](https://github.com/undergroundwires/privacy.sexy), **TypeScript/Vue/Electron**, лицензия **AGPL-3.0**, ~5.8k★, слоган *«privacy is sexy»*) — открытый инструмент, который **собирает набор твиков** приватности и безопасности под твою ОС: отключение телеметрии, чистка bloatware, ужесточение настроек браузеров/приложений, файрвол и т.д. Есть **веб-версия** ([privacy.sexy](https://privacy.sexy/)) и **десктоп** (Electron, оффлайн) под Windows, macOS и Linux.

> [!note] Как это на самом деле работает — не «одна кнопка всё выключила»
> privacy.sexy **не выполняет твики автоматически**. Ты выбираешь нужные пункты (видя, **что именно** делает каждый), а инструмент **генерирует скрипт** (`.ps1`/`.bat` для Windows, `.sh` для macOS/Linux), который **ты сам запускаешь** с правами администратора/`sudo`. Это плюс к безопасности: ничего не срабатывает втихую, всё под твоим контролем. «1 клик» из поста = выбрал → сгенерировал → применил.

## ✨ Что умеет

- **Телеметрия и сбор данных ОС** — отключение известных каналов слежки Windows/macOS/Linux.
- **Приватность приложений/браузеров** — ужесточение настроек конкретных программ и сервисов.
- **Безопасность/хардненинг** — отключение ненужных служб, улучшение правил файрвола.
- **Чистка** — удаление трекеров и лишних компонентов (debloat).
- **100+ скриптов** под каждую ОС, сгруппированных и с описанием; каждый **независим** (нет перекрёстных зависимостей — можно брать по одному).
- **Обратимость.** Для многих твиков есть **revert-скрипт** — «что-то сломалось → откатил».

## ✅ Почему это относительно безопасный подход

> [!info] Прозрачность + оффлайн + обратимость
> - **Прозрачно:** видно код каждого твика перед применением — не «чёрный ящик».
> - **Оффлайн:** десктоп-версия работает без сети; сам инструмент **не собирает телеметрию** (иначе было бы иронично). Веб-версия просто **генерирует скрипт в браузере** — ничего не исполняет.
> - **Обратимо:** есть revert; но всё равно **сделай точку отката/снапшот** до применения.

## ⚠️ Нюансы и подводные камни

> [!caution] «Отключить всё» ≠ «стало лучше»
> Часть твиков **агрессивна**: отключение служб/компонентов может сломать нужную функциональность (обновления, синхронизацию, магазин приложений, часть сети). Правила:
> - **Читай описание** каждого твика — включай осознанно, а не «выделить всё».
> - **Точка отката/снапшот** до запуска (Windows — restore point; Linux — снапшот [Btrfs](../Linux/Filesystems/Btrfs%20%E2%80%94%20%D1%81%D0%BE%D0%B2%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D0%B0%D1%8F%20CoW-%D0%A4%D0%A1%20%28%D0%BF%D0%BE%D0%B4%D1%82%D0%BE%D0%BC%D0%B0%2C%20%D1%81%D0%BD%D0%B0%D0%BF%D1%88%D0%BE%D1%82%D1%8B%2C%20RAID%29%20%E2%80%94%20%D0%BF%D1%80%D0%B0%D0%BA%D1%82%D0%B8%D0%BA%D0%B0%20%D0%B8%20%D1%83%D1%81%D1%82%D1%80%D0%BE%D0%B9%D1%81%D1%82%D0%B2%D0%BE.md)/ВМ).
> - **Применяй повторно** после крупных апдейтов ОС (обновления часто возвращают телеметрию) — авторы прямо это рекомендуют.

## 🖥️ По системам (важно для владельца)

| Система | Применимость |
| :--- | :--- |
| **Windows** | ✅ Основной и самый проработанный сценарий (телеметрия, bloatware, службы). Пересекается с [GTweak](../Windows/GTweak%20%E2%80%94%20%D0%BF%D0%BE%D1%80%D1%82%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D1%8B%D0%B9%20%D1%82%D0%B2%D0%B8%D0%BA%D0%B5%D1%80-%D0%B4%D0%B5%D0%B1%D0%BB%D0%BE%D0%B0%D1%82%D0%B5%D1%80%20Windows%20%28%D0%BF%D1%80%D0%B8%D0%B2%D0%B0%D1%82%D0%BD%D0%BE%D1%81%D1%82%D1%8C%2C%20%D1%81%D0%BB%D1%83%D0%B6%D0%B1%D1%8B%2C%20%D0%B0%D0%BA%D1%82%D0%B8%D0%B2%D0%B0%D1%86%D0%B8%D1%8F%20HWID-KMS%29.md) — можно сверять |
| **macOS** | ✅ Набор твиков есть |
| **Debian / Ubuntu** | ✅ Linux-скрипты в основном рассчитаны на **systemd** и `apt` — тут заходят «как есть» |
| **Gentoo** (основная) | ⚠️ **Осторожно:** многие Linux-твики предполагают **systemd**/Ubuntu-пути, а у тебя **OpenRC**. `systemctl disable …`, apt-специфика и часть путей **не сработают или неприменимы** — бери твики **точечно** и адаптируй под OpenRC/Portage вручную |
| **Arch** (план с июня 2026) | ⚠️ systemd есть, но `apt`-специфика и Ubuntu-пакеты — нет; фильтруй под pacman |
| **Entware / RT-AX56U** | **N/A** — это десктопный/ОС-хардненинг; для роутера свои средства (см. [Pi-hole](../Apps/Pi-hole%20%E2%80%94%20%D1%81%D0%B5%D1%82%D0%B5%D0%B2%D0%BE%D0%B9%20%D0%B1%D0%BB%D0%BE%D0%BA%D0%B8%D1%80%D0%BE%D0%B2%D1%89%D0%B8%D0%BA%20%D1%80%D0%B5%D0%BA%D0%BB%D0%B0%D0%BC%D1%8B%20%D0%B8%20%D1%82%D0%B5%D0%BB%D0%B5%D0%BC%D0%B5%D1%82%D1%80%D0%B8%D0%B8%20%28DNS-sinkhole%29.md) на уровне DNS) |

> [!tip] Совет по Linux
> На Gentoo используй privacy.sexy скорее как **чек-лист идей** (что вообще стоит отключить/зачистить), а конкретные команды перекладывай на свою систему: `rc-update del …`, USE-флаги, `/etc/portage`. Для десктопного браузера/приложений твики применимее, чем для системных служб.
