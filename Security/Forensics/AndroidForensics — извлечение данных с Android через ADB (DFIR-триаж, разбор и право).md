# 🔎📱 AndroidForensics — извлечение данных с Android через ADB (DFIR-триаж, разбор и право)

[**AndroidForensics**](https://github.com/DouglasFreshHabian/AndroidForensics) (DouglasFreshHabian, **Shell/Bash**, лицензия **MIT**, ~382★) — набор скриптов для **автоматизированного форензик-сбора данных с Android-устройства через ADB**. Подключаешь телефон по USB с включённой отладкой — скрипты прогоняют десятки `adb`/`dumpsys`-команд и складывают результат в отчёт. Позиционируется для **аудитов безопасности и расследования инцидентов**.

> [!danger] Только с авторизацией — извлечение чужих данных незаконно
> Сам README предупреждает: *«только для авторизованного форензик-анализа»*. Извлечение контактов/SMS/звонков с **чужого** устройства без согласия владельца/законного основания в РФ — это **ст. 272 УК РФ** (неправомерный доступ) и **ст. 138** (тайна переписки/переговоров). Запускай **только** на своих устройствах, при письменном согласии владельца или в рамках законного расследования (ордер/договор/scope). Заметка **справочно-образовательная** (DFIR/blue-team).

## 🧰 Что собирает

**Системное (без root):**
- инфо об устройстве (модель, производитель, версия Android, серийник), батарея/питание, сеть, аптайм/счётчик загрузок;
- установленные приложения и запущенные сервисы, **logcat**, снапшот **bugreport**;
- диагностика через **`dumpsys`**: память, аудио, сенсоры, аккаунты, fingerprint, сетевая статистика, питание, геолокация, уведомления, телефония, Wi-Fi и ещё 10+ сервисов.

**Пользовательское (может требовать root на Android 11+):**
- контакты и номера, **журнал звонков**, **SMS**, email/аккаунты, привязанные к приложениям.

> [!warning] Android 11+ ограничивает доступ к контактам/звонкам/SMS
> README честно отмечает: на **Android 11 и выше** доступ к контактам, журналу звонков и SMS через `adb shell content` **ограничен** — без **root** или специальной forensic-сборки эти данные обычно не вытащить. Базовое системное инфо собирается и без root.

> [!caution] Это логический триаж, а не «судебно-чистый» образ
> Инструмент делает **live logical extraction**: меняет состояние устройства (включена отладка, выполняются команды), **нет write-blocker, нет хэширования/цепочки доказательств (chain of custody)** по умолчанию. Для быстрого **триажа** — отлично; для **доказательной** форензики в суде нужны верифицируемое имаджирование, хэши и протокол. Перед работой по делу — фиксируй хэши отчётов отдельно и веди журнал действий.

## 📦 Установка и запуск

Нужен **ADB** на хосте + **включённая отладка по USB** на устройстве + Bash (Linux/macOS).

```bash
git clone https://github.com/DouglasFreshHabian/AndroidForensics.git
cd AndroidForensics
chmod +x extract.sh dumpsys.sh
./extract.sh    # → ADB_Report_[ДАТА]_[ВРЕМЯ]/ (по .txt на команду)
./dumpsys.sh    # → DumpSysReport_[ДАТА]_[ВРЕМЯ]/
```

| Система владельца | Как поставить ADB |
| :--- | :--- |
| **Gentoo** (основная) | `emerge dev-util/android-tools` (даёт `adb`), затем `git clone` и запуск скриптов |
| **Debian / Ubuntu** | `sudo apt install adb` (или `android-sdk-platform-tools`) |
| **Arch** (план с июня 2026) | `sudo pacman -S android-tools` (содержит `adb`) |
| **Entware / RT-AX56U** | **N/A практически** — роутер не форензик-станция, `adb`-хост-режим тут не нужен; гоняй скрипты с десктопа |

> [!note] Перед запуском
> На устройстве: **Настройки → Для разработчиков → Отладка по USB**, подтвердить отпечаток RSA при коннекте. Проверь связь: `adb devices` (должно показать устройство как `device`, а не `unauthorized`).

## 💡 Кому полезно

- **Blue-team / IR:** быстрый триаж Android при разборе инцидента — что за устройство, приложения, сервисы, логи, сетевые артефакты.
- **Самопроверка/бэкап метаданных** своего телефона.
- **Не** инструмент слежки за чужими людьми и **не** замена доказательной форензике (нужны имидж + хэши + протокол).
