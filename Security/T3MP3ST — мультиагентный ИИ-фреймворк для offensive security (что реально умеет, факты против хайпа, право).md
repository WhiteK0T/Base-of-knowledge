# 🛡️ T3MP3ST — мультиагентный ИИ-фреймворк для offensive security

Канал «Не баг, а фича» подал это громко: «делаем хакеров из ИИ-агентов — найдут ЛЮБУЮ уязвимость и пробьют любую защиту». Речь про [**T3MP3ST**](https://github.com/elder-plinius/T3MP3ST) — фреймворк от **Pliny** (*elder-plinius*, известный исследователь LLM-джейлбрейков, автор L1B3RT4S). Инструмент реальный и интересный, но пост — маркетинг: **сам автор в README гораздо честнее** и прямо пишет, что «пробивает» пока далеко не всё. Разберём, что там на самом деле.

> [!warning] Дисклеймер по назначению
> T3MP3ST — **наступательный** (offensive) инструмент для **авторизованного** тестирования, исследований и обучения. Направлять его можно **только** на системы, которыми владеешь ты, или на которые есть **явное письменное разрешение**. Неправомерный доступ к чужим системам — **уголовное преступление** (см. правовой блок ниже). Эта заметка — обзорно-защитная: понять, что за инструмент, чего он реально стоит и как не сесть.

## 🧬 Что это такое

- **Мультиагентный «meta-harness» для red team**: превращает уже подключённый у тебя ИИ-кодинг-агент в «охотника за уязвимостями». Архитектура — **8 операторов** по kill chain: `Recon` (разведка), `Scanner`, `Exploiter`, `Infiltrator` (боковое перемещение/повышение прав), `Exfiltrator`, `Ghost` (persistence/скрытность), `Coordinator` (оркестрация), `Analyst` (отчёты).
- **LLM-агностик и «keyless».** Работает через OpenRouter / Venice / Anthropic / OpenAI **или** полностью **локально без API-ключей** — через уже запущенный агент (Claude Code, Codex) или локальную модель (**Ollama, LM Studio, vLLM**). «Твой агент — мозг, T3MP3ST — обвязка вокруг».
- **Арсенал**: **35 инструментов** по умолчанию, **83** — с опциональным `T3MP3ST_FULL_ARSENAL`. Опасные post-exploitation драйверы (**metasploit, hydra**) — **за гейтом ручного подтверждения человеком**.
- **Стек**: TypeScript/JavaScript (+чуть Python), **AGPL-3.0**, ~4.3k★. Репозиторий **очень новый** (создан 2 июля 2026), активно пилится.

## 🧪 Факты против хайпа

> [!note] «Найдёт любую уязвимость и пробьёт любую защиту» — нет
> Автор сам расставляет статусы: **стабилен только движок разведки (Recon)** и одиночный агент. Цитата: *«engine is real; the swarm is the part still earning its stripes»*. Операторы `Exploiter/Infiltrator/Exfiltrator/Ghost` крутят настоящий tool-backed ReAct-цикл, **но как скоординированный "рой" не проверены и пока ненадёжны** (*«live swarm exploitation still unreliable»*). То есть «навёл — и оно само всё взломало» — это ещё **не** про сегодняшний T3MP3ST.

Что стоит за цифрой «104 задачи, 90% с первой попытки»:

- **90.1% pass@1** — это на **XBEN** (набор XBOW из 104 задач, **black-box, web-app**), причём **одним агентом** на модели gpt-5.5, а **не** роем. Это выше self-report XBOW (85%), число перепроверяется командой `npm run verify-claims` — плюс к доверию. Но это **бенчмарк-полигон**, а не «любой боевой сервис».
- **Cybench** (академический CTF-бенч, без подсказок) — уже скромнее: **23/40 (58%)**.
- **CVE-Zero** — 8/10 точных попаданий (file/line/CWE) на 10 «свежих» CVE после cutoff модели. Честно помечено: **n=10, направленно**, не доказательство «рой лучше одиночки».
- **White-box анализ кода** — Python-only, помечен как ⚗️ экспериментальный; облако/мобилки/бинарники — «строительные леса», live-эксплуатация **не бенчмаркалась**.

> [!tip] Почему это не «кнопка взлома всего»
> **Egress-scope containment включён по умолчанию.** Как только задана цель миссии, встроенные сетевые инструменты **отказываются** бить по хостам вне scope (`SCOPE DENIED`) — не по чужим публичным адресам, не по loopback/private. Плюс дисциплина «disclosure pipeline»: черновики отчётов готовит ИИ, **отправляет человек**. Это ответственный дизайн, а не «наведи на соседа».

## ⚖️ Право (РФ) — читать до запуска

> [!caution] Применение по чужим системам без разрешения — статья УК
> «Идеально для изучения кибербезопасности» — да, **но только на своих/лабораторных/авторизованных целях**. По чужим системам без письменного согласия это:
> - **ст. 272 УК РФ** — неправомерный доступ к компьютерной информации;
> - **ст. 273 УК РФ** — создание/использование/распространение вредоносных программ (эксплойт-обвязка может подпасть);
> - **ст. 274 УК РФ** — нарушение правил эксплуатации средств хранения/обработки.
>
> В РФ **нет чистого «карве-аута» для пентеста**: даже легальное тестирование требует договора, согласованного scope и rules of engagement (ROE). «Просто попробовать на чужом сайте» — это не серая зона, это состав преступления.

**Легально и безопасно учиться:** свои стенды и ВМ, публичные **CTF/полигоны** (Cybench, HackTheBox, локальные XBEN-ranges), bug-bounty **строго в рамках программы**, или заказная работа по договору с письменной авторизацией.

## 💻 По платформам

Node.js-проект + «мозг» в виде LLM. Ключевое преимущество для приватности — **keyless-режим на локальной модели** (ничего не уходит в облако).

| Система | Запуск |
| :--- | :--- |
| **Gentoo** (основная) | `emerge net-libs/nodejs`, `git clone` + `npm install`; «мозг» — локальный [Ollama](../AI/Local-LLM/Ollama%20%E2%80%94%20%D0%BC%D0%B5%D0%BD%D0%B5%D0%B4%D0%B6%D0%B5%D1%80%20%D0%B8%20%D1%81%D0%B5%D1%80%D0%B2%D0%B5%D1%80%20%D0%BB%D0%BE%D0%BA%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D1%85%20LLM.md)/vLLM или подключённый Claude Code |
| **Debian / Ubuntu** | `apt install nodejs npm` (или nodesource), далее `npm install`; локальная модель через Ollama/LM Studio |
| **Arch** (план с июня 2026) | `pacman -S nodejs npm`; арсенал-адаптеры (nmap, metasploit) — из репозиториев по мере надобности |
| **Entware / RT-AX56U** | **N/A** — нужен Node + достаточно мощная LLM; роутер (armv7, 512 МБ) не потянет ни агент-модель, ни тяжёлый арсенал. Максимум — быть **целью** учебного стенда, а не запускать T3MP3ST |

## 💡 Итог

- **Что реально:** сильный **движок разведки** и одиночный агент, прозрачные воспроизводимые бенчмарки, ответственные дефолты (scope-containment, гейт на опасные драйверы). Отличная **учебная** и research-платформа для авторизованных целей.
- **Что хайп:** «пробьёт любую защиту» и «сам ударит по любой цели» — координированный автономный эксплойт-рой **ещё не готов** (сам автор это подчёркивает), а 90% — это конкретный web-бенчмарк одним агентом, не «любой сервис».
- **Красная черта:** без письменной авторизации — не запускать по чужому. Инструмент мощный, ответственность — уголовная.
