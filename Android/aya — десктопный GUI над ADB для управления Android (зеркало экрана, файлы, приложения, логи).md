# 📱 aya — десктопный GUI над ADB для управления Android

[**aya**](https://github.com/liriliri/aya) ([liriliri](https://github.com/liriliri), **Electron/TypeScript**, лицензия **AGPL-3.0**, ~5.3k★, v1.14.2 от 28.11.2025) — настольное приложение с графическим интерфейсом, которое оборачивает **ADB (Android Debug Bridge)** в удобную «панель управления» телефоном с компьютера. По сути — комбайн «scrcpy + файловый менеджер + диспетчер приложений/процессов + logcat + shell» в одном окне. Автор — **liriliri**, известный по инструментам разработчика (eruda, licia, chii).

## ✨ Что умеет

- **Зеркалирование экрана** с низкой задержкой + управление **клавиатурой/мышью**, запись видео и скриншоты (подход как у scrcpy).
- **Файловый менеджер** устройства — просмотр ФС, загрузка/скачивание файлов.
- **Менеджер приложений** — установка/удаление APK, управление пакетами.
- **Процессы и производительность** — мониторинг CPU/памяти/FPS в реальном времени.
- **Layout Inspector** — просмотр дерева UI (полезно для разработки/реверса интерфейсов).
- **Logcat-вьюер** — логи с **фильтрами и экспортом**.
- **Терминал** = ADB shell с поддержкой **нескольких сессий**.

## 🧩 Как работает и что нужно

> [!info] Только USB-debugging, root не нужен
> aya работает поверх **ADB**, поэтому на телефоне нужно включить **«Отладку по USB»** (Настройки → Для разработчиков) и **авторизовать** компьютер (диалог с отпечатком ключа). **Root не требуется** для основных функций. Подключение — по **USB** или по сети (**ADB over Wi-Fi**, `adb tcpip`).

> [!note] Кроссплатформенный десктоп
> Сборки под **Windows x64, macOS (arm64/x64), Linux x86_64**. Это **Electron**-приложение — тяжеловат по памяти, но зато единый GUI на всех ОС. Требуется рабочий **adb** (обычно из пакета `android-tools`/platform-tools).

## ⚠️ Безопасность: USB-debugging — это мощный доступ

> [!caution] Авторизуй только доверенные компьютеры
> Включённая отладка по USB + авторизация ПК дают этому компьютеру **глубокий доступ** к устройству: файлы, установка/удаление приложений, shell, чтение логов. Практика:
> - **Авторизуй только свои** машины; на чужом/публичном ПК отладку не подтверждай (это вектор атаки — «juice jacking» и слив данных).
> - **ADB over Wi-Fi** удобно, но открывает порт по сети — включай только в доверенной сети и **выключай после**.
> - После работы можно **отозвать авторизации**: Настройки → Для разработчиков → «Отозвать доступ для отладки по USB».
> - Тот же ADB-доступ используется и для **форензики/извлечения данных** — см. [AndroidForensics](../Security/Forensics/AndroidForensics%20%E2%80%94%20%D0%B8%D0%B7%D0%B2%D0%BB%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5%20%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85%20%D1%81%20Android%20%D1%87%D0%B5%D1%80%D0%B5%D0%B7%20ADB%20%28DFIR-%D1%82%D1%80%D0%B8%D0%B0%D0%B6%2C%20%D1%80%D0%B0%D0%B7%D0%B1%D0%BE%D1%80%20%D0%B8%20%D0%BF%D1%80%D0%B0%D0%B2%D0%BE%29.md): что реально можно вытащить через ADB и правовые рамки.

## 🆚 Чем отличается от scrcpy

**scrcpy** — про **зеркалирование и управление** экраном (и всё, делает это отлично, легковесно, CLI). **aya** идёт шире: то же зеркало **плюс** файлы, приложения, процессы, логи, shell, layout-инспектор — **единый GUI-центр** управления. Платишь за это весом Electron. Если нужно только «показать экран телефона на ПК» — хватит scrcpy; если хочется всё в одном окне — aya.

## 🖥️ По системам

| Система | Как поставить |
| :--- | :--- |
| **Gentoo** (основная) | AppImage/бинарь из релизов; нужен adb — `emerge dev-util/android-tools` |
| **Debian / Ubuntu** | `.deb`/AppImage из релизов + `apt install android-tools-adb` |
| **Arch** (план с июня 2026) | AUR (`aya-bin`) или AppImage; `pacman -S android-tools` |
| **macOS / Windows** | Готовые сборки (arm64/x64 для mac, x64 для Windows) |
| **Entware / RT-AX56U** | **N/A** — это **десктопный GUI**, управляющий телефоном; на роутере не запускается и незачем. Управляемое устройство — **Android-телефон**, а не роутер |
