# ✉️ SMS2Email — пересылка входящих SMS на почту через свой SMTP

**SMS2Email** ([github.com/SMS2Email/SMS2Email](https://github.com/SMS2Email/SMS2Email)) — простое приложение для Android, которое **автоматически пересылает входящие SMS на указанный e-mail** через **твой SMTP-сервер**. Полезно, когда нужно **удалённо получать SMS** (например, с номера, который не носишь с собой). Apache-2.0, Kotlin, F-Droid/IzzyOnDroid, с **воспроизводимыми сборками** (reproducible builds — хороший знак доверия). ~162★, молодой (с декабря 2025), активный (v1.1.31).

> [!info] Что это по сути
> - **Однонаправленно**: SMS → e-mail. Ответить с почты нельзя — только читать пересланное.
> - Отправка через **любой SMTP** (Gmail, свой сервер, и т.п.) — письма летят на твой ящик.
> - Ставится из **F-Droid / IzzyOnDroid** (в Google Play нет). Открытый код + reproducible builds → можно проверить, что APK собран из этого исходника.

> [!caution] Безопасность — тут есть о чём подумать (пост молчит)
> - **SMS часто = коды 2FA/OTP/банк.** Пересылая их на почту, ты фактически **переносишь свой второй фактор в e-mail**: если почту взломают — вместе с ней утекут и коды. SMS-2FA и так слабая; такой мост её ещё ослабляет. Для критичного лучше держать 2FA в приложении-аутентификаторе, а не гнать в почту.
> - **SMTP-пароль лежит на телефоне.** Заводи **отдельный app-password / отдельный ящик** под это, а не пароль основного аккаунта. Обязательно **SMTP по TLS/SSL** (порт 465/STARTTLS), иначе SMS уйдут открытым текстом.
> - **Разрешение на чтение SMS** — чувствительное. Тут спасает открытость и reproducible builds, но помни, что приложение видит все входящие сообщения.

> [!info] Где реально пригодится
> - **Второй номер как SMS-шлюз**: старый телефон с SIM дома пересылает SMS на почту — читаешь откуда угодно.
> - **Архив/бэкап SMS** в почтовый ящик.
> - **Приём OTP с второстепенного номера** удалённо (с оглядкой на риск выше).
> - Мониторинг SIM в IoT/сигнализации/роутере.
>
> Документация скудная (README — по сути бейджи и скриншоты), настройка — прямо в приложении. Проект молодой (25 открытых issue) — ждать шероховатостей.

## 🖥️ Применимость на системах владельца

| Система | Как |
| :--- | :--- |
| **Android-телефон** (лучше отдельный/второй) | ✅ поставить из F-Droid, указать SMTP (TLS!) и адрес получателя. Держать телефон онлайн (Wi-Fi/данные) для отправки |
| **Приёмная сторона** | 📥 любой e-mail; можно завести под это отдельный ящик — вплоть до своего opentrashmail для автоматизации/парсинга |
| **Твои десктопы / RT-AX56U** | ➖ само приложение живёт на телефоне; роутер/ПК ни при чём (разве что как SMTP-приёмник) |

> [!tip] Тебе по профилю (Android + self-host + автоматизация)
> Хорошо стыкуется с идеей «свой почтовый сток»: пересылай SMS на **свой** ящик (или opentrashmail) и парси/автоматизируй через JSON API/вебхуки. Только не гоняй туда критичные банковские 2FA — для них аутентификатор. И SMTP строго по TLS с отдельным паролем.
