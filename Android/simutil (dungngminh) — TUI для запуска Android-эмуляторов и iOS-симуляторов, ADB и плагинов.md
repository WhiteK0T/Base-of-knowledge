# 📱 simutil (dungngminh) — TUI для запуска эмуляторов/симуляторов и управления устройствами

[**simutil**](https://github.com/dungngminh/simutil) (автор **dungngminh**, **Dart**, лицензия **MIT**, ~940★, v0.6.2 от 08.07.2026, очень свежий — репо с фев. 2026, пилится активно) — кроссплатформенный **терминальный интерфейс (TUI)**, который сводит в одно окно то, что обычно раскидано по Android Studio и Xcode: запуск **Android-эмуляторов**, **iOS-симуляторов** и работу с **физическими устройствами** через ADB. Для мобильного разработчика — «пульт» вместо возни с двумя тяжёлыми IDE ради того, чтобы просто поднять девайс.

## ✨ Что умеет

- **Запуск устройства одной клавишей** с разными режимами: обычный, **холодный старт (cold boot)**, **без звука (no-audio)** и их комбинации.
- **Просмотр logcat** с фильтрацией (Android).
- **Встроенные ADB-инструменты:** беспроводное подключение по **IP**, спаривание по **6-значному коду** (Android 11+), **QR-код** — коннект к физическим Android-устройствам без кабеля.
- **Выключение** эмуляторов/симуляторов из интерфейса.
- **Редактор настроек** по горячей клавише.
- **Плагины через YAML-конфиг** — подключение внешних инструментов (**scrcpy**, **Maestro** и др.) **без правки кода**: просто дописываешь их в конфиг. Это, пожалуй, самое интересное — simutil становится единой точкой запуска твоего мобильного тулинга.

## ⚠️ Что важно понимать (факты против хайпа)

> [!warning] iOS-симуляторы — только на macOS
> Пост подал «iOS + Android» без оговорки, но **iOS-часть работает исключительно на macOS** (нужны **Xcode + `simctl`**). На Linux/Windows доступны **только Android**-функции. Это не ограничение simutil, а требование самой Apple — iOS-симулятор в принципе не существует вне macOS.

> [!note] Это «дирижёр», а не замена SDK — сам ничего не эмулирует
> simutil **не поставляет эмуляторы** и не заменяет Android SDK/Xcode — он их **запускает и оркестрирует**. Предпосылки должны стоять заранее:
> - **Android:** Android SDK + `emulator` + `adb` (сами AVD-образы создаются штатными средствами);
> - **iOS:** Xcode + `simctl` (только macOS).
> Без установленного SDK/эмулятора запускать будет нечего.

> [!caution] Проект молодой — версия 0.x
> Репозиторий появился в **феврале 2026**, текущая версия **0.6.2** (до 1.0). Оно активно развивается и выглядит рабочим, но это **ранняя стадия**: возможны ломающие изменения, баги, нестабильность API плагинов. Для критичного workflow держи в уме, что это ещё «догоняет» зрелость.

## 🆚 Ниша и соседи

Категория — **удобный фронтенд для мобильной разработки**. simutil перекликается с [aya](aya%20%E2%80%94%20%D0%B4%D0%B5%D1%81%D0%BA%D1%82%D0%BE%D0%BF%D0%BD%D1%8B%D0%B9%20GUI%20%D0%BD%D0%B0%D0%B4%20ADB%20%D0%B4%D0%BB%D1%8F%20%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F%20Android%20%28%D0%B7%D0%B5%D1%80%D0%BA%D0%B0%D0%BB%D0%BE%20%D1%8D%D0%BA%D1%80%D0%B0%D0%BD%D0%B0%2C%20%D1%84%D0%B0%D0%B9%D0%BB%D1%8B%2C%20%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F%2C%20%D0%BB%D0%BE%D0%B3%D0%B8%29.md) (десктопный GUI над ADB), но у simutil другой упор: **эмуляторы/симуляторы + iOS**, TUI вместо GUI, и **плагины** для scrcpy/Maestro. Если aya — про зеркалирование и управление физическим телефоном, то simutil — про **быстрый запуск девайсов для разработки/тестов**.

## 🖥️ По системам

Собран на Dart → нативный бинарь.

| Система | Применимость / установка |
| :--- | :--- |
| **Gentoo** (основная) | ✅ Только **Android**-часть. Нужны Android SDK + `emulator` + `adb`. Ставить: `dart pub global activate` или бинарь/скрипт-инсталлятор; Dart из `dev-lang/dart` (или бандл Flutter) |
| **Debian / Ubuntu** | ✅ Android-часть; SDK/эмулятор + adb. Бинарь или `dart pub global activate` |
| **Arch** (план с июня 2026) | ✅ Android-часть; Dart/Flutter из репо/AUR, Android SDK; `dart pub global activate` |
| **macOS** | ✅ **Полный набор** — iOS (Xcode/`simctl`) **и** Android. Установка через **Homebrew** |
| **Windows** | ✅ Только Android; PowerShell-инсталлятор |
| **Entware / RT-AX56U** | **N/A** — нужен десктоп с Android SDK/эмулятором (тяжёлое, x86/ARM-desktop); на роутере (armv7, 512 МБ) эмулятор Android не поднять. Для физического телефона ADB гоняй с рабочей машины |
