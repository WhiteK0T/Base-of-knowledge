# 🧩 MiMo Code — терминальный AI-агент для кода с долгой памятью (Xiaomi)

**MiMo Code** ([github.com/XiaomiMiMo/MiMo-Code](https://github.com/XiaomiMiMo/MiMo-Code)) — открытый **терминальный AI-агент для программирования** от Xiaomi (релиз ~11.06.2026, версия v0.1.x). Главная фишка — **долговременная память между сессиями**: отдельный фоновый суб-агент сжимает историю работы в структурированные заметки и не даёт модели «забыть», что она делала раньше. Технически это **форк [OpenCode](Сводная%20таблица%20AI-агентов%20для%20программирования%20%28июнь%202026%29.md)** под лицензией **MIT** (с доп. ограничениями использования), работает со своей моделью **MiMo-V2.5(-Pro)** и со сторонними провайдерами (BYOK). По смыслу — прямой конкурент [Claude Code](Claude%20Code%20%E2%80%94%20гайд.md).

> [!warning] Отделяем факты от заголовков
> Пост и пресса подают агрессивно («обгоняет Claude Code»). Сверка с репозиторием/первоисточниками:
> | Заявление | Реально |
> | :--- | :--- |
> | «62% SWE-Bench Pro, 73% Terminal Bench 2, обгоняет Claude Code» | это **собственные замеры Xiaomi**, ~+5 п.п. к Claude Code; независимой проверки пока нет |
> | «Новый агент с нуля» | **форк OpenCode** + слой памяти/оркестрации поверх; ядро не своё |
> | «Долгая память между сессиями» | **правда** и главная ценность: SQLite FTS5 + файлы памяти (см. ниже) |
> | «Open source MIT» | да, но с **дополнительными ограничениями использования** — не чистый MIT |
> | «Surpasses Claude Code» | сравнение «харнесс+модель»: тянет в основном модель **MiMo-V2.5-Pro**, на ультра-длинных задачах (200+ шагов) |

## 🧬 Чем интересен — память между сессиями

Обычный CLI-агент «забывает» контекст между запусками. MiMo Code держит знание о проекте в **SQLite FTS5** (полнотекстовый поиск) + наборе файлов, которыми управляет выделенный суб-агент:

| Файл | Роль |
| :--- | :--- |
| `MEMORY.md` | память проекта: архитектурные решения, правила |
| `checkpoint.md` | автоснимки состояния сессии (ведёт отдельный суб-агент) |
| `notes.md` | черновик-«блокнот» агента |
| `tasks/<id>/progress.md` | пошаговый лог по каждой задаче |

При приближении к лимиту токенов агент **пересобирает контекст** из последнего чекпоинта, памяти проекта, прогресса задач и недавних сообщений — поэтому не теряет нить на длинных (200+ шагов) задачах. Это и есть преимущество, которое Xiaomi противопоставляет «забывающему между запусками» Claude Code.

## ⚙️ Модели и конфигурация

- Провайдеры: **MiMo Auto** (бесплатно ограниченное время), платформа **Xiaomi MiMo**, **Custom Provider** (свой OpenAI-совместимый endpoint), а также **импорт настроек из Claude Code**.
- Конфиг — `.mimocode/mimocode.json` или интерактивный TUI при первом запуске (туда же кладутся провайдеры и API-ключи).
- Родная модель — **MiMo-V2.5-Pro**: MoE на ~1.02 трлн параметров (42 млрд активных), гибридное внимание, контекст до 1M токенов, веса под MIT. Можно подключить и сторонние модели (как в OpenCode).

> [!tip] Это «харнесс», а не модель
> MiMo Code — **оболочка-агент** (как [Claude Code](Claude%20Code%20%E2%80%94%20гайд.md) или OpenCode), а не сама LLM. Цифры бенчмарков получены в связке с MiMo-V2.5-Pro; на другой подключённой модели результат будет иным. Ценность харнесса — именно память/оркестрация, а не «магия» модели.

## 📦 Установка

Нужен **Node.js** (агент ставится через npm/устанавливочный скрипт). Кроссплатформенно — команды одинаковы на всех десктоп-дистрибутивах владельца.

```bash
# macOS / Linux — однострочный установщик
curl -fsSL https://mimo.xiaomi.com/install | bash

# Любая ОС (в т.ч. Windows) — через npm
npm install -g @mimo-ai/cli

# запуск (первый старт — интерактивная настройка провайдера/ключа)
mimo
```

| Система | Как поставить |
| :--- | :--- |
| **Gentoo** | сначала `emerge net-libs/nodejs`, затем `npm install -g @mimo-ai/cli` |
| **Debian / Ubuntu** | `nodejs`/`npm` (apt или nodesource), затем `npm install -g @mimo-ai/cli` |
| **Arch** | `sudo pacman -S nodejs npm`, затем `npm install -g @mimo-ai/cli` |
| **Entware / RT-AX56U** | **неприменимо** — см. ниже |

> [!danger] На роутере смысла нет
> Это **десктопный dev-инструмент**: гоняет агентные циклы по кодовой базе и обращается к тяжёлой LLM. Запускать на ASUS RT-AX56U (armv7, 256 МБ flash) незачем — даже при BYOK (внешний API) это не сценарий для роутера. Используй на рабочей машине.

> [!caution] `curl … | bash` — прочитай, что запускаешь
> Установщик одной строкой исполняет скачанный скрипт от внешнего сервиса. Доверяешь источнику — ок; иначе ставь через `npm` или сперва глянь содержимое скрипта.

## 💡 Стоит ли смотреть

- **Да**, если интересна **память между сессиями** на длинных задачах и нужен бесплатный открытый CLI-агент (можно подключить свою модель, как в OpenCode).
- **Скептически** к «обгоняет Claude Code»: это маркетинг на собственных бенчмарках; жди независимых замеров.
- Уже есть рабочий процесс на [Claude Code](Claude%20Code%20%E2%80%94%20гайд.md)/OpenCode — миграция не обязательна; интересна именно подсистема памяти (её идеи переносимы и в другие агенты через [MCP](MCP%20%E2%80%94%20серверы%20Model%20Context%20Protocol.md)/файлы).
- Хочешь запустить **локально без облака** — MiMo-V2.5-Pro огромна (1T); реалистичнее подключить тяжёлую модель через API или взять модель полегче (см. [локальный запуск LLM](Local-LLM/llama.cpp%20%E2%80%94%20движок%20локального%20инференса%20GGUF.md)).
