# Claude Code — шпаргалка команд

Короткая выжимка самого нужного. Полный разбор — в заметке [Claude Code — гайд](Claude%20Code%20%E2%80%94%20гайд.md). Общий обзор агентов — в [Сводной таблице AI-агентов](Сводная%20таблица%20AI-агентов%20для%20программирования%20%28июнь%202026%29.md).

> [!tip] Подсказка
> Нажмите `/` в начале строки, чтобы увидеть все доступные команды; начните печатать — список отфильтруется. Команда распознаётся только **в начале** сообщения, остальной текст уходит ей как аргумент.

## ⚡ Самые необходимые команды

| Команда | Что делает |
| :--- | :--- |
| `/init` | Сгенерировать стартовый `CLAUDE.md` для проекта |
| `/team-onboarding` | Собрать гайд для онбординга команды из истории твоих сессий (за 30 дней) |
| `/context` | Показать, чем занят контекст (цветная сетка), и подсказки по оптимизации |
| `/compact` | Сжать (суммаризировать) диалог, освободив контекст; можно задать фокус: `/compact оставь детали по API` |
| `/usage` | Стоимость сессии, лимиты плана, статистика (алиасы `/cost`, `/stats`) |
| `/insights` | Отчёт-анализ твоих сессий: зоны проекта, паттерны, точки трения |
| `/review [PR]` | Локальное ревью пул-реквеста в текущей сессии |
| `/model [model]` | Сменить модель (стрелками ←/→ — уровень reasoning) |
| `/effort [level]` | Уровень «усилий» модели: `low`/`medium`/`high`/`xhigh`/`max`/`ultracode` (`auto` — по умолчанию) |
| `/rewind` | Откатить код и/или диалог к контрольной точке (алиасы `/checkpoint`, `/undo`) |
| `/clear` | Начать новый диалог с чистым контекстом (память проекта сохраняется) |
| `/plan [задача]` | Войти в режим планирования прямо из строки ввода |

## 🔀 Режимы работы (переключение `Shift+Tab`)

`Shift+Tab` циклически переключает режимы прав:

- **`default`** — обычный режим: Claude действует, спрашивая разрешение на правки/команды. *(Это и есть «режим действия» — отдельного «act» в Claude Code нет.)*
- **`acceptEdits`** — авто-приём правок файлов (accept edits): редактирования применяются без подтверждения.
- **`plan`** — режим планирования: только чтение и анализ, без изменений; в конце предлагает план.
- *(опционально, если включены)* **`auto`**, **`bypassPermissions`**.

## ⌨️ Откат: двойной `Esc`

- `Esc` — прервать текущий ответ/инструмент (наработанное сохраняется).
- `Esc Esc` (двойной):
  - если в строке ввода есть текст — очистить черновик (сохранив в историю, `↑` вернёт);
  - если строка пустая — открыть **меню rewind** (откат кода/диалога к прошлой точке). То же делает `/rewind`.

## 🐚 Вызов shell (режим `!`)

Префикс `!` в начале строки выполняет команду в shell **напрямую**, без участия Claude, и добавляет вывод в контекст:

```bash
! npm test
! git status
! ls -la
```

- Не требует интерпретации/одобрения от Claude.
- `Tab` — автодополнение из прошлых `!`-команд проекта.
- Выход из режима: `Esc`, `Backspace` или `Ctrl+U` на пустой строке.

> [!note] Полезные горячие клавиши
> `Ctrl+O` — просмотр транскрипта; `Ctrl+R` — поиск по истории; `Ctrl+B` — увести задачу в фон; `Ctrl+T` — список задач; `@` — упоминание файла; `Alt/Option+P` — сменить модель; `Alt/Option+T` — расширенное мышление.
