# 🧩 Gemma 4 12B Coder — локальный файнтюн на ризонинге Fable 5 (GGUF)

Энтузиаст (`yuxinlu1`) выложил на Hugging Face **дообученную Gemma 4 12B** для кодинга: модель «дистиллировали» на цепочках рассуждений (chain-of-thought) **Fable 5** и **Composer 2.5**, оставив только те трейсы, что **прошли тесты задач**. Раздаётся в формате **GGUF** — то есть запускается локально на обычном железе через `llama.cpp` / Ollama / LM Studio.

> [!warning] Это НЕ «Fable 5 себе на комп» — отделяем факты от хайпа
> Заголовок новости («скачиваем Fable 5») вводит в заблуждение. Скачивается **Gemma 4 12B** от Google, дообученная на **синтетических данных**, сгенерированных в т.ч. Fable 5. Это **не веса Fable 5** и не его качество — это маленькая (12B) открытая модель, которая *переняла стиль рассуждений* на узком домене (Python/алгоритмы). Ждать от неё уровня облачного Fable 5 не стоит.

| Автор | сообщество (`yuxinlu1`), не Google и не Anthropic |

## 🔬 Как это сделано (дистилляция с проверкой тестами)

Подход — **distillation на верифицированных цепочках рассуждений**:

1. Берутся задачи (Python/алгоритмы) с тестами.
2. Сильные модели (**Composer 2.5** + синтетические трейсы **Fable 5**) генерируют рассуждение + решение.
3. Решение **прогоняется против тестов задачи** — и в датасет попадают **только прошедшие** (test-gated CoT).
4. На этом отфильтрованном датасете дообучается Gemma 4 12B.

> [!info] Почему это разумнее «слепой» дистилляции
> Фильтр «оставляем только то, что прошло тесты» отсекает красивые, но неверные рассуждения. Поэтому на своём узком домене (алгоритмический Python) модель может работать заметно лучше базовой Gemma — но это не делает её универсально умнее.

## 📦 Квантизации (GGUF-файлы)

| Квант | Размер файла | Нужно RAM/VRAM (ориентир) | Когда брать |
| :--- | :--- | :--- | :--- |
| **Q2_K** | 4.83 ГБ | ~6 ГБ | очень слабое железо, заметная потеря качества |
| **Q4_K_M** | 7.38 ГБ | ~9–10 ГБ | **золотая середина** (рекомендую по умолчанию) |
| **Q6_K** | 9.79 ГБ | ~12 ГБ | ближе к оригиналу, если памяти хватает |
| **Q8_0** | 12.7 ГБ | ~14–16 ГБ | максимум качества GGUF, тяжелее всего |

> [!tip] Что выбрать
> Для 12B на десктопе обычно оптимален **Q4_K_M**: приемлемое качество при разумном размере. С GPU на 12+ ГБ VRAM есть смысл взять **Q6_K**. К размеру файла добавляй запас RAM на контекст (131K — это много KV-кэша, реально держи контекст меньше, если памяти мало).

## 🚀 Как запустить локально

Целевые системы владельца — десктопные дистрибутивы (Gentoo, Debian/Ubuntu, Arch). Сначала ставим движок инференса, затем берём GGUF-файл.

### llama.cpp (самый «нативный» путь)

```bash
# Gentoo (Portage)
emerge --ask llama-cpp        # пакет в ::guru при отсутствии в основном дереве

# Debian / Ubuntu — собрать из исходников (пакета обычно нет)
sudo apt install -y build-essential cmake libcurl4-openssl-dev
git clone https://github.com/ggml-org/llama.cpp && cd llama.cpp
cmake -B build -DGGML_CUDA=ON   # без GPU — убрать флаг
cmake --build build -j --config Release

# Arch (AUR)
yay -S llama.cpp               # или llama.cpp-cuda / llama.cpp-vulkan под GPU
```

Запуск (модель скачается с HF по `-hf`, либо укажи путь к локальному `.gguf`):

```bash
# интерактивный чат
llama-cli -hf yuxinlu1/gemma-4-12B-coder-fable5-composer2.5-v1-GGUF:Q4_K_M

# локальный OpenAI-совместимый сервер на :8080
llama-server -hf yuxinlu1/gemma-4-12B-coder-fable5-composer2.5-v1-GGUF:Q4_K_M \
  --ctx-size 8192 --port 8080
```

### Ollama

```bash
# Arch: pacman -S ollama   |   Gentoo: emerge app-misc/ollama (guru)
# Debian/Ubuntu: curl -fsSL https://ollama.com/install.sh | sh
ollama run hf.co/yuxinlu1/gemma-4-12B-coder-fable5-composer2.5-v1-GGUF:Q4_K_M
```

### LM Studio

GUI-вариант: в поиске моделей вбить `yuxinlu1/gemma-4-12B-coder-fable5-composer2.5`, выбрать квант (Q4_K_M), скачать и запустить — там же есть локальный сервер с OpenAI-совместимым API.

## 🔌 Подключить к своим инструментам

`llama-server` / Ollama / LM Studio дают **OpenAI-совместимый эндпоинт** (`/v1/chat/completions`). Его удобно завести в общий стек через [LiteLLM](../LiteLLM%20%E2%80%94%20единый%20шлюз%20%28proxy%29%20к%20100%2B%20LLM.md) и дальше дёргать из агентов / [Claude Code](../Claude%20Code%20%E2%80%94%20гайд.md)-совместимых клиентов — так же, как [FreeQwenApi](../FreeQwenApi%20%E2%80%94%20бесплатный%20OpenAI-совместимый%20API%20к%20Qwen%20Chat.md) / [FreeDeepseekAPI](../FreeDeepseekAPI%20%E2%80%94%20бесплатный%20OpenAI-совместимый%20API%20к%20DeepSeek.md), только полностью локально.

## ⚖️ Лицензия и правовые нюансы

> [!warning] Лицензия Gemma, а не «свободная»
> Веса под **Gemma Terms of Use** (Google), а не MIT/Apache. Есть [Prohibited Use Policy](https://ai.google.dev/gemma/prohibited_use_policy) и обязанность передавать условия дальше при редистрибуции. Для коммерческого/продакшн-использования — читай условия, это **не** безоговорочно открытая лицензия.

> [!note] Дистилляция чужих рассуждений — серая зона
> Датасет построен на трейсах **Composer 2.5 и Fable 5**. Условия использования этих моделей часто **запрещают обучать конкурирующие модели на их выводах**. Юридически и этически это спорный момент; для личных экспериментов — ок, для продукта — оценивай риски.

## 🛡️ Безопасность

> [!danger] «Refuses less than base» — меньше отказов
> В карточке модели прямо сказано: из-за узкого, заточенного под задачи обучения она **отказывает реже базовой Gemma** и авторы советуют «добавить свои guardrails для продакшна». То есть встроенных предохранителей меньше — не подключай вслепую к чувствительным/публичным сценариям без своего слоя модерации. (Похожий эффект целенаправленно делают через [abliteration](../Heretic%20%E2%80%94%20снятие%20safety-ограничений%20с%20открытых%20LLM%20%28abliteration%29.md).)

## 💡 Стоит ли смотреть

- **Да**, если нужен **локальный** кодинг-ассистент по Python/алгоритмам, который влезает на десктоп (8–16 ГБ RAM) и работает офлайн/приватно.
- **Скептически** к подаче «это Fable 5»: это 12B Gemma, а не топовая облачная модель; на широких задачах будет ощутимо слабее.
- **Учитывай** лицензию Gemma, спорность дистилляции чужих трейсов и сниженный уровень safety-отказов.
- Для сравнения с другими открытыми кодинг-LLM — см. [GLM 5.2](../GLM%205.2%20%E2%80%94%20открытая%20китайская%20LLM%20%28Z.ai%2C%20кодинг%29.md) и [сводную таблицу AI-агентов](../Сводная%20таблица%20AI-агентов%20для%20программирования%20%28июнь%202026%29.md).
