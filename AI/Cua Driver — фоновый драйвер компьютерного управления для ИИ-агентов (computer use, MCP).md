# 🖱️ Cua Driver — фоновый драйвер компьютерного управления для ИИ-агентов (computer use, MCP)

[**Cua Driver**](https://github.com/trycua/cua) (часть монорепо **trycua/cua**, ядро на **Rust** — `cua-driver-rs`, лицензия **MIT**, у всего проекта ~18.8k★, релиз драйвера v0.6.7) — открытый **фоновый драйвер «компьютерного управления» (computer use)**. Это «руки» для ИИ-агента: он умеет **видеть окна приложений (скриншот + дерево доступности accessibility tree), кликать, печатать, скроллить** — и делает это **в фоне, не перехватывая фокус и курсор**. Подключается к агенту как **MCP-сервер** (или вызывается как CLI). Работает на **macOS, Windows и Linux** (Linux — пока pre-release).

> [!warning] Главное, что путает пост: драйвер — это «руки», а не «мозг»
> Cua Driver сам **не думает и не решает, куда тыкать**. Он лишь даёт агенту инструменты: «дай скриншот окна», «нажми сюда», «впиши текст», «прочитай дерево элементов». **Решения принимает внешний LLM-агент** — Claude Code, Codex, Cursor и т.п., которому ты этот драйвер подключил по MCP. Поэтому фраза поста *«агент сам понимает, куда тыкать, без лишних расчётов»* — это про связку «LLM + драйвер», а не про магию самого Cua. Качество = качество модели-агента.

## 🧩 Что это даёт на практике

- **Grounding по accessibility tree + скриншоту:** драйвер читает нативные API доступности ОС и отдаёт агенту список доступных элементов окна (кнопки, поля) + картинку. Агент привязывает «куда кликнуть» к реальным координатам — отсюда «понимает, куда тыкать».
- **Не ворует фокус:** в отличие от «обычного» computer-use (который двигает твой настоящий курсор), Cua Driver работает с окнами **в фоне** — можно параллельно работать самому.
- **Один бинарь — три режима:** MCP-сервер по stdio, долгоживущий демон или разовая shell-команда.
- **Связка с любым агентом по MCP:** Claude Code, Codex, Cursor, OpenClaw, свои клиенты.

## 🧪 Факты против хайпа

> [!caution] «ЛЮБОЕ приложение» и «изолированная среда» — с оговорками
> - **«Управляет ЛЮБЫМ приложением»** — на деле точность зависит от того, **выставляет ли приложение accessibility-дерево**. С нативными приложениями и стандартными элементами — хорошо; с самописными канвасами/играми/нестандартным UI агенту остаётся только «смотреть на пиксели», и ошибок больше.
> - **«Создаёт изолированную среду, где выполняет промпт»** — пост смешал **два разных продукта Cua**. Сам **Cua Driver** просто водит мышкой по **твоему реальному рабочему столу** (просто без кражи фокуса). Отдельная **изоляция/песочница (Cua Sandbox / Cua Run)** — это уже другие компоненты (локальные/облачные VM, в т.ч. платная инфраструктура фермы машин). Драйвер ≠ песочница.
> - **Linux — pre-release:** в репозитории Linux-сборки (x86_64/ARM64) публикуются «для раннего тестирования и пока **не являются официально поддерживаемым тиром**». На macOS зрелее всего, Windows — следом.
> - **Это инструмент для разработчиков/энтузиастов**, а не «волшебная кнопка»: ставишь бинарь, регистрируешь как MCP, подключаешь модель-агента и платишь за её токены.

## 📦 Установка

Ставится скриптом-установщиком, потом регистрируется как MCP-сервер у агента.

```bash
# macOS / Linux — установить бинарь cua-driver
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/trycua/cua/main/libs/cua-driver/scripts/install.sh)"

# Windows (PowerShell) — автоопределение архитектуры, без прав админа
irm https://raw.githubusercontent.com/trycua/cua/main/libs/cua-driver/scripts/install.ps1 | iex
```

Подключение к Claude Code как MCP-сервер (из README драйвера):

```bash
# обычная регистрация
claude mcp add --transport stdio cua-driver -- cua-driver mcp

# режим совместимости с computer-use/vision Claude Code
# (screenshot требует pid и window_id и снимает только это окно)
claude mcp add --transport stdio cua-computer-use -- cua-driver mcp --claude-code-computer-use-compat
```

| Система владельца | Применимость |
| :--- | :--- |
| **Gentoo** (основная) | ✅ есть GUI/X11/Wayland — ставится install-скриптом (Linux x86_64). Нужен **внешний агент** (Claude Code/Codex). Linux-тир пока pre-release — проверяй на тестовых задачах |
| **Debian / Ubuntu** | ✅ аналогично: install-скрипт, Linux-бинарь x86_64/ARM64; помни про pre-release статус Linux |
| **Arch** (план с июня 2026) | ✅ install-скриптом или собрать из исходников (Rust, `cargo`); тот же агент-клиент по MCP |
| **Entware / RT-AX56U** | **N/A** — драйвер автоматизирует **десктопные приложения**, а на роутере (armv7, 512 МБ RAM) нет ни рабочего стола, ни GUI-программ для управления. Сам управляющий LLM-агент тоже не для роутера |

> [!note] Что нужно ещё, кроме драйвера
> 1) **Агент-клиент** с поддержкой MCP (Claude Code, Codex, Cursor…); 2) **доступ к модели** (API/локальная LLM) и её токены; 3) на macOS — выдать разрешения **Accessibility/Screen Recording** в системных настройках, иначе драйвер не «увидит» окна.

## 💡 Кому полезно

- **Автоматизация рутины в GUI-приложениях**, где нет API: заполнить форму, прокликать legacy-программу, перенести данные между окнами «руками агента».
- **Разработчикам computer-use агентов:** готовый кросс-платформенный слой «глаза+руки» по MCP, не надо писать свой драйвер мыши/скриншотов.
- **Бенчмаркинг/обучение агентов** (в связке с Cua Bench / Sandbox).
- **Не** автономный ИИ сам по себе и **не** песочница — это драйвер ввода/вывода для внешнего агента.
