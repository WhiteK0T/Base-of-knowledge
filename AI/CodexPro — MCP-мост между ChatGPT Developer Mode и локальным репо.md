# 🌉 CodexPro — MCP-мост между ChatGPT Developer Mode и локальным репо

**CodexPro** ([github.com/rebel0789/codexpro](https://github.com/rebel0789/codexpro)) — **[MCP](MCP%20%E2%80%94%20серверы%20Model%20Context%20Protocol.md)-мост**, который подключает **ChatGPT в режиме Developer Mode** к твоему локальному репозиторию: запускает локальный сервер с файловыми/git/bash-инструментами и пробрасывает его в ChatGPT, после чего веб-чат может читать, редактировать, искать по коду, смотреть git и запускать команды — как локальный кодинг-агент. Автор `rebel0789`, MIT, ~320★, **очень молодой проект (~17 коммитов)**.

> [!warning] Отделяем факты от рекламы поста
> Пост заметно преувеличивает. Сверка с README репозитория:
> | Заявление поста | Реально |
> | :--- | :--- |
> | «про подписку на Codex можно забыть» | **неправда**: нужен платный **ChatGPT Plus/Pro** + включённый Developer Mode. Репо прямо пишет: это **не обход лимитов**, ChatGPT и Codex — разные продукты со своими тарифами |
> | «делает из сайта агента, скрейпит браузер» | нет: интеграция **официальная**, через MCP-фреймворк ChatGPT Developer Mode, а не автоматизацию UI |
> | «запускать команды в терминале» | да, но **только из allowlist в safe-режиме**, и для этого поднимается **публичный туннель** (ngrok/Cloudflare) — см. безопасность |
> | «локальный Codex бесплатно» | ты используешь **уже оплаченный** ChatGPT для агентного кодинга в репо; «бесплатности» нет |

## 🧬 Как работает

```text
ChatGPT (Developer Mode)  ──MCP──▶  публичный HTTPS-туннель (ngrok/Cloudflare, токен)
                                          │
                                          ▼
                              локальный сервер CodexPro
                                          │
                                          ▼
                         твой репозиторий: файлы / git / bash
```

CodexPro поднимает локальный HTTP-сервер с MCP-инструментами и публикует его через защищённый токеном публичный туннель, чтобы ChatGPT Developer Mode мог до него достучаться. Скрейпинга веб-интерфейса **нет** — используется официальный app-фреймворк Developer Mode.

## ✨ Что умеет

- **Файлы:** чтение, запись, правки с диффами.
- **Навигация по коду:** дерево, поиск через **ripgrep**.
- **Терминал:** bash из **allowlist** (по умолчанию safe-режим).
- **Git:** `status`, `diff`, `log`.
- **Планы/handoff:** пишет `.ai-bridge/current-plan.md` для локального агента ([Codex](Сводная%20таблица%20AI-агентов%20для%20программирования%20%28июнь%202026%29.md), OpenCode, Pi).
- **Экспорт контекста:** «Pro context bundles» (в стиле `AGENTS.md`) для моделей без поддержки MCP.

Три режима: **normal** (ChatGPT сам читает/правит/проверяет), **handoff** (пишет план для локального агента-исполнителя), **pro planning** (выгрузка контекст-бандла для сессий без MCP).

## 📦 Установка

Нужен **Node.js 20+** и аккаунт **ChatGPT Plus/Pro** с включённым Developer Mode.

```bash
npm install -g codexpro
codexpro setup                  # мастер настройки
codexpro start --root /путь/к/репо   # поднять MCP против репозитория
# затем подключить напечатанный endpoint в ChatGPT → Developer Mode
```

| Система | Как ставить |
| :--- | :--- |
| **Gentoo / Debian-Ubuntu / Arch** | поставить `nodejs` (≥20) + `npm`, затем `npm install -g codexpro` |
| **macOS / Windows** | тот же `npm install -g codexpro` |
| **Entware / RT-AX56U** | **неприменимо** — десктоп-dev-инструмент, завязан на ChatGPT Plus и рабочий репозиторий |

## ☠️ Безопасность (читать обязательно)

> [!danger] Ты открываешь ФС и bash удалённой LLM через публичный туннель
> CodexPro **по своему определению не OS-песочница** — это «локальный мост с безопасными дефолтами». Риски:
> - **Публичный туннель** (ngrok/Cloudflare) делает твою машину достижимой извне; защита — только токен в URL. Утёк/угадан endpoint → доступ к инструментам.
> - **Удалённая модель запускает bash** (пусть и allowlist). Расширишь список или выключишь safe-режим — даёшь больше прав.
> - **Промпт-инъекция**: вредная инструкция в файле/issue/зависимости может заставить агента выполнить нежелательное действие над твоим репо.
>
> Практика: запускай только на **доверенном, некритичном репозитории**, держи **safe-режим**, не расширяй allowlist бездумно, **выключай туннель** после работы, не подключай к рабочим/секретным проектам.

> [!caution] Молодой проект + глобальный npm-пакет
> ~17 коммитов и ~320★ — мало истории. Пакет ставится **глобально** и получает доступ к файловой системе и шеллу. Перед запуском имеет смысл **прочитать код** и понимать, какому туннель-провайдеру ты доверяешь. Дефолтные блок-листы (`.env`, `.git`, `node_modules`, приватные ключи, сборки) — хорошо, но это не гарантия.

## 💡 Когда полезно

- Платишь за **ChatGPT Plus/Pro** и хочешь использовать его для агентного кодинга прямо в репо, не покупая отдельно Codex.
- Нужен **handoff-режим**: ChatGPT планирует, а исполняет локальный агент ([Codex](Сводная%20таблица%20AI-агентов%20для%20программирования%20%28июнь%202026%29.md)/OpenCode/Pi).
- **Не** стоит, если: нет ChatGPT Plus; репозиторий рабочий/секретный; не готов к публичному туннелю и удалённому bash. Тогда бери локальный CLI-агент напрямую ([MiMo Code](MiMo%20Code%20%E2%80%94%20терминальный%20AI-агент%20для%20кода%20с%20долгой%20памятью%20%28Xiaomi%29.md), Codex CLI) без проброса наружу.
