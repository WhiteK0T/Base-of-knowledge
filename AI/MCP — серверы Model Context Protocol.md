# MCP — серверы Model Context Protocol

**MCP (Model Context Protocol)** — открытый стандарт интеграции AI с инструментами. **MCP-сервер** даёт агенту (например, [Claude Code](Claude%20Code%20%E2%80%94%20гайд.md)) доступ к твоим инструментам, базам данных и API: вместо того чтобы копипастить данные из трекера или дашборда в чат, агент читает и действует в этих системах напрямую.

Сервер может предоставлять три вида возможностей:
- **tools** — действия/функции, которые агент может вызывать;
- **resources** — данные/контент (доступны через `@`-упоминания);
- **prompts** — готовые промпты, появляющиеся как команды.

> [!warning] Безопасность прежде всего
> Подключай только **доверенные** серверы. Сервер, тянущий внешний контент, может стать вектором **prompt injection**. Серверы уровня `project` (из `.mcp.json`) поэтому требуют ручного одобрения перед использованием.

## 🔌 Транспорты

| Транспорт | Когда использовать |
| :--- | :--- |
| **stdio** | Локальный процесс на твоей машине (скрипты, прямой доступ к системе) |
| **HTTP** (`streamable-http`) | **Рекомендуемый** для удалённых/облачных серверов; поддерживает OAuth |
| **SSE** | Server-Sent Events — **устарел**, используйте HTTP |
| **WebSocket** (`ws`) | Постоянное двунаправленное соединение для push-событий; аутентификация только через заголовки |

## ⚙️ Установка: `claude mcp add`

**Удалённый HTTP-сервер:**

```bash
claude mcp add --transport http notion https://mcp.notion.com/mcp

# с Bearer-токеном
claude mcp add --transport http secure-api https://api.example.com/mcp \
  --header "Authorization: Bearer your-token"
```

**Локальный stdio-сервер:**

```bash
claude mcp add --env AIRTABLE_API_KEY=YOUR_KEY --transport stdio airtable \
  -- npx -y airtable-mcp-server
```

> [!note] Зачем нужен `--`
> Для stdio двойной дефис `--` отделяет **опции Claude** (`--transport`, `--env`, `--scope`) от **команды сервера** и её аргументов. Всё после `--` уходит серверу как есть. Без `--` Claude попытается распарсить флаги сервера (например `--port`) как свои.

## 🗂️ Scopes (где хранится конфигурация)

| Scope | Загружается | Шарится с командой | Где хранится |
| :--- | :--- | :--- | :--- |
| **local** (по умолчанию) | только текущий проект | нет | `~/.claude.json` |
| **project** | только текущий проект | да, через git | `.mcp.json` в корне проекта |
| **user** | все твои проекты | нет | `~/.claude.json` |

Задаётся флагом `--scope local|project|user`. Приоритет при дублях по имени: **local → project → user → плагины → коннекторы claude.ai** (берётся запись из источника с наибольшим приоритетом целиком, поля не сливаются).

Пример `.mcp.json` (можно коммитить в репозиторий, поддерживает подстановку переменных окружения вида `${VAR}`):

```json
{
  "mcpServers": {
    "shared-server": {
      "command": "/path/to/server",
      "args": [],
      "env": {}
    }
  }
}
```

## 🛠️ Управление

```bash
claude mcp list          # список серверов
claude mcp get github    # детали сервера
claude mcp remove github # удалить
```

Внутри сессии:
- **`/mcp`** — статус серверов (число инструментов у каждого) и **OAuth-аутентификация** для удалённых серверов.
- Серверы из `.mcp.json`, ждущие одобрения, отображаются как `⏸ Pending approval`. Сбросить решения по одобрению: `claude mcp reset-project-choices`.

## 🧩 Промпты и ресурсы

- **Промпты сервера** появляются как команды в формате `/mcp__<server>__<prompt>` (динамически обнаруживаются у подключённых серверов).
- **Ресурсы** упоминаются через `@` (автодополнение пути/ресурса).

## 📦 Откуда брать и как делать свои

- Готовые коннекторы — в **[Anthropic Directory](https://claude.ai/directory)** (та же MCP-инфраструктура; любой удалённый сервер оттуда добавляется через `claude mcp add`).
- Свой сервер можно сделать по [гайду MCP](https://modelcontextprotocol.io/docs/develop/build-server) или дать Claude сгенерировать каркас плагином `mcp-server-dev`.
- **Плагины** могут поставлять MCP-серверы: они стартуют автоматически при включении плагина и работают так же, как настроенные вручную.

## 💡 Полезные переменные окружения

- `MCP_TIMEOUT` — таймаут запуска сервера (мс), напр. `MCP_TIMEOUT=10000 claude`.
- `MCP_TOOL_TIMEOUT` / поле `"timeout"` в `.mcp.json` — таймаут выполнения инструмента.
- `MAX_MCP_OUTPUT_TOKENS` — лимит на размер вывода инструмента (предупреждение от 10 000 токенов).
- В окружении stdio-сервера Claude Code задаёт `CLAUDE_PROJECT_DIR` (корень проекта).
