# 🐉 GLM 5.2 — открытая китайская LLM (Z.ai, кодинг)

**GLM 5.2** — большая языковая модель от **Zhipu AI** (бренд **Z.ai**), вышедшая **13 июня 2026**. Позиционируется как **coding-first / агентная** модель с контекстом до **1 000 000 токенов** и открытыми весами под **MIT**. Продолжение линейки GLM-4.5 / 4.6 / GLM-5.

> [!warning] Новость пришла в рекламном тоне — отделяем факты от хайпа
> Исходный пост («монстр», «сильнее Fable 5», «в 300 раз дешевле Claude», «42.8 балла») — **маркетинговый**. На момент запуска **Zhipu не опубликовала официальных бенчмарков** (нет SWE-bench Verified, LiveCodeBench, HumanEval, Aider). Независимые замеры — в процессе. Относись к громким цифрам как к **непроверенным заявлениям**.

## 🧬 Что известно по фактам

| Параметр | Значение |

| Разработчик | **Zhipu AI** (Z.ai), Китай |

| Лицензия весов | **MIT** (открытые веса — на неделе после запуска) |

| Доступ | GLM Coding Plan (тарифы), чат Z.ai, отдельный API |

## ⚖️ Заявления vs проверяемость

| Заявление из новости | Статус |
| :--- | :--- |
| «Сильнее Fable 5 в ризонинге» | ❓ не подтверждено независимо |
| «42.8 балла, с отрывом от Fable 5» | ❓ источник/методика бенчмарка не раскрыты |
| «В 300 раз дешевле Claude по токенам» | ❓ маркетинг; официальный прайс API — отдельно |
| «Целиком бесплатная» | ⚠️ частично: открытые веса = self-host бесплатно; облачный доступ — по тарифам/лимитам |
| «Почти без цензуры» | ⚠️ китайская модель: фильтры на «чувствительные для КНР» темы остаются |
| Открытый исходный код / веса (MIT) | ✅ заявлено официально |
| 1M контекст, coding-first | ✅ подтверждается материалами Z.ai |

> [!note] «Бесплатно» с оговоркой
> Открытые веса под MIT означают, что модель можно **запускать у себя бесплатно** (нужно своё железо/инференс). Облачный доступ через Z.ai — по подписке/лимитам, как у любого провайдера. «Бесплатность» — про лицензию, а не про безлимитный облачный инференс.

## 🚀 Где попробовать

- **Чат Z.ai** — [z.ai](https://z.ai).
- **Zcode** — площадка для вайбкодинга на GLM: [zcode.z.ai/en](https://zcode.z.ai/en).
- **API** — отдельный эндпоинт Z.ai (OpenAI-совместимый у GLM обычно есть → можно завести через шлюз).
- **Self-host** — открытые веса MIT: локально/на сервере (vLLM, llama.cpp по мере появления квантов).

> [!tip] Подключить к своим инструментам
> Если у GLM есть OpenAI-совместимый endpoint, модель удобно прокинуть в существующий стек через [LiteLLM](ProxyLLM/LiteLLM%20%E2%80%94%20единый%20шлюз%20%28proxy%29%20к%20100%2B%20LLM.md) (единый прокси к 100+ LLM) и дальше дёргать из агентов/[Claude Code](Claude%20Code%20%E2%80%94%20гайд.md)-совместимых клиентов. Так же, как [FreeQwenApi](ProxyLLM/FreeQwenApi%20%E2%80%94%20бесплатный%20OpenAI-совместимый%20API%20к%20Qwen%20Chat.md) / [FreeDeepseekAPI](ProxyLLM/FreeDeepseekAPI%20%E2%80%94%20бесплатный%20OpenAI-совместимый%20API%20к%20DeepSeek.md).

## 💡 Стоит ли смотреть

- **Да**, если нужен открытый (MIT) сильный кодинг-LLM с большим контекстом и возможностью self-host — это редкое сочетание.
- **Скептически** к сравнениям «сильнее Fable 5»: ждём независимые бенчмарки (Artificial Analysis, SWE-bench, LiveCodeBench) и реальный прайс API.
- **Учитывай** китайскую специфику фильтрации и то, что данные облачного доступа уходят на серверы Z.ai (для чувствительного — self-host).
