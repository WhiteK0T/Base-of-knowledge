# Claude Code — гайд

**Claude Code** — агентный инструмент для программирования от Anthropic, работающий в терминале (а также в IDE и десктоп-приложении). Он сам читает и редактирует код, запускает команды и тесты, разбивает задачи на шаги и итеративно их решает поверх моделей Claude.

> [!info] Связанные заметки
> Быстрый справочник — [Claude Code — шпаргалка команд](Claude%20Code%20%E2%80%94%20шпаргалка%20команд.md). Сравнение с другими инструментами — [Сводная таблица AI-агентов](Сводная%20таблица%20AI-агентов%20для%20программирования%20%28июнь%202026%29.md).

## 🚀 Базовый цикл работы

Команды полезны в конкретных точках сессии — от настройки проекта до отправки изменений:

- **Первая сессия в репозитории.** `/init` создаёт стартовый `CLAUDE.md`, затем `/memory` дорабатывает его. `/mcp` и `/agents` настраивают серверы и субагентов, `/permissions` — правила одобрений.
- **Во время задачи.** `/plan` — режим планирования перед крупным изменением. `/model` и `/effort` регулируют, сколько reasoning ты тратишь. Когда диалог разрастается — `/context` показывает, чем занят контекст, а `/compact` сжимает его. `/btw` — быстрый вопрос «в сторону», не засоряющий историю.
- **Параллельная работа.** `/agents` — менеджер субагентов; `/tasks` — что крутится в фоне; `/background` — увести всю сессию в фон; `/batch` — разложить большое изменение на независимые юниты, каждый в своём git-worktree.
- **Перед отправкой.** `/diff` — что изменилось; `/code-review` — ревью диффа на баги и упрощения (с `--fix` применяет правки); `/review` и `/security-review` — более глубокий read-only проход; `/code-review ultra` — мульти-агентное облачное ревью.
- **Между сессиями.** `/clear` — начать новую задачу, сохранив память проекта; `/resume` и `/branch` — вернуться к диалогу или ответвить его.
- **Когда что-то не так.** `/rewind` — откат кода и диалога к контрольной точке; `/doctor` и `/debug` — диагностика установки и runtime; `/feedback` — репорт бага с контекстом сессии.

## 🔀 Режимы прав (`Shift+Tab`)

Переключаются циклически по `Shift+Tab`:

| Режим | Поведение |
| :--- | :--- |
| `default` | Обычный режим действия: правки и команды — с запросом разрешения |
| `acceptEdits` | Авто-приём правок файлов без подтверждения |
| `plan` | Только чтение/анализ, без изменений; в конце выдаёт план |
| `auto` / `bypassPermissions` | Опциональные расширенные режимы (если включены) |

Отдельного режима «act» нет — действие происходит в `default`. Войти в plan-режим можно и командой `/plan [задача]`.

## 🧠 Контекст, модель и «усилия»

- `/context` — визуализация занятости окна контекста + подсказки по «тяжёлым» инструментам и раздутой памяти.
- `/compact [фокус]` — суммаризация диалога для освобождения контекста.
- `/clear` — полностью чистый старт (предыдущий диалог остаётся в `/resume`).
- `/model [model]` — смена модели и сохранение по умолчанию; стрелками ←/→ регулируется уровень reasoning.
- `/effort [level]` — уровень усилий: `low` / `medium` / `high` / `xhigh` / `max` / `ultracode` (`max` и `ultracode` — только на сессию; `ultracode` = `xhigh` + авто-оркестрация workflow). `auto` сбрасывает к умолчанию.

## 📝 Память: `CLAUDE.md`

- `/init` — создать `CLAUDE.md` (инструкции и факты о проекте, которые подхватываются каждой сессией).
- `/memory` — редактировать файлы памяти, включать/смотреть auto-memory.
- `CLAUDE.md` хранит **факты** о проекте; повторяющиеся **процедуры** лучше выносить в skills (см. ниже), т.к. их тело грузится только при вызове.

## 🧩 Skills и кастомные команды

- Кастомные команды объединены со **skills**: файл `.claude/commands/deploy.md` и skill `.claude/skills/deploy/SKILL.md` оба дают `/deploy`.
- Skill — это `SKILL.md` с инструкциями; Claude вызывает его сам по релевантности или ты явно через `/имя`.
- `/skills` — список доступных skills; `/reload-skills` — пере-сканировать без перезапуска.
- Совместимы с открытым стандартом [Agent Skills](https://agentskills.io).

## 🔌 MCP, субагенты, чекпоинты

- **MCP** (`/mcp`) — подключение внешних серверов инструментов и OAuth. MCP-серверы могут отдавать промпты-команды вида `/mcp__<server>__<prompt>`. Подробнее — [MCP — серверы Model Context Protocol](MCP%20%E2%80%94%20серверы%20Model%20Context%20Protocol.md).
- **Субагенты** (`/agents`) — делегирование побочных задач агентам с чистым контекстом и полным набором инструментов (антипод `/btw`, который видит весь диалог, но без инструментов).
- **Checkpointing** (`/rewind`, двойной `Esc` на пустой строке) — откат кода/диалога к прошлой точке.

## 🐚 Shell-режим и горячие клавиши

- Префикс `!` — выполнить shell-команду напрямую, добавив вывод в контекст: `! npm test`. Выход — `Esc` / `Backspace` / `Ctrl+U` на пустой строке.
- Фоновые bash-команды: `Ctrl+B` уводит запуск в фон; вывод пишется в файл, Claude читает его при необходимости.
- Многострочный ввод: `\`+`Enter`, `Ctrl+J`, либо `Shift+Enter` (в части терминалов — после `/terminal-setup`).
- Прочее: `Ctrl+O` — транскрипт, `Ctrl+R` — поиск истории, `Ctrl+T` — список задач, `@` — упоминание файла, `Alt/Option+P` — модель, `Alt/Option+T` — расширенное мышление, `Alt/Option+O` — fast mode.

## 💡 Полезные мелочи

- `/btw <вопрос>` — эфемерный вопрос «в сторону» (не попадает в историю, без доступа к инструментам).
- `/recap` — однострочное резюме сессии; авто-recap появляется после паузы.
- `/usage` (`/cost`, `/stats`) — стоимость и лимиты; на платных планах — разбивка по skills/субагентам/MCP.
- `/team-onboarding` — гайд для коллеги из истории твоих сессий.
- `/insights` — аналитика твоих сессий: зоны проекта, паттерны, точки трения.
- `/doctor` — проверить установку (`f` — попросить Claude починить найденное).
