# 🧠 Obsidian Mind — постоянная память для ИИ-агентов в Obsidian-хранилище

**Obsidian Mind** ([github.com/breferrari/obsidian-mind](https://github.com/breferrari/obsidian-mind)) — **шаблон Obsidian-хранилища**, который даёт кодинг-агентам ([Claude Code](Claude%20Code%20%E2%80%94%20%D0%B3%D0%B0%D0%B9%D0%B4.md) — полная поддержка, Codex CLI, Gemini CLI) **память между сессиями**. Идея: агент по протоколу хранилища **в начале сессии подтягивает контекст** (цели, активные проекты, последние изменения, открытые задачи), а по ходу **сам ведёт структурированные заметки** — решения, профили людей, инциденты, «achievements». Всё это — обычные Markdown-файлы в твоём Obsidian-волте, которые ты можешь читать и править сам. MIT, TypeScript, **~3900★**, активно развивается.

> [!warning] «Бесконечная память / никаких ограничений» — это маркетинг
> | Заявление поста | Реально |
> | :--- | :--- |
> | «бесконечная память, никаких ограничений» | ❌ контекст LLM **всё равно конечный**. Obsidian Mind — это **внешнее хранилище заметок**, из которого агент подтягивает **выборочно** нужное. Это не «бесконечный контекст», а дисциплинированные заметки + подгрузка релевантного |
> | «закидывай огромные кодовые базы и историю» | ❌ так не работает: **целую большую кодовую базу в память не грузят**. Хранят структурированные записи (решения, задачи, люди, инциденты), а агент читает то, что нужно под задачу |
> | «долговременная память между сессиями» | ✅ да — файлы персистятся, агент читает их в начале сессии |
> | «сама ведёт заметки» | ✅ через команды/хуки (`/om-standup`, `/om-dump`, `/om-incident-capture`), но агент **должен реально следовать протоколу** и это **тратит токены** на чтение/запись каждую сессию |
> | «переносить контекст между агентами» | ✅ одно хранилище работает для Claude Code / Codex / Gemini |

> [!info] Важно про ожидания: это память про РАБОТУ, не только про код
> По примерам из README упор — на **рабочий/менеджерский контекст**: 1:1 с руководителем, профили коллег (`org/people/`), decision records, **Brag Doc** (лог достижений для ревью/резюме), разбор инцидентов. То есть это скорее **«второй мозг» твоей рабочей жизни** с агентом-секретарём, чем чистая «память по коду». Для кодовой памяти годится, но позиционирование поста («для кодинг-агентов и огромных кодовых баз») сужает картину.

## 🧩 Как это выглядит (команды)

- **`/om-standup`** («start session») — грузит North Star, активные проекты, открытые задачи, свежие git-изменения → «на чём ты застрял и что дальше».
- **`/om-dump <текст>`** — брейн-дамп после встречи → сам создаёт/обновляет заметки: профиль человека, decision record, запись 1:1, пункт в Brag Doc, задачу в активном проекте.
- **`/om-incident-capture <ссылка>`** — суб-скиллы (slack-archaeologist, people-profiler) собирают таймлайн, root-cause, записи о людях.
- **«wrap up»** — проверяет, что у заметок есть связи, обновляет индексы, ищет незафиксированные «вины» (brag-spotter).

## 🛠️ Установка

```bash
npm install -g shardmind
mkdir my-vault && cd my-vault
shardmind install github:breferrari/obsidian-mind
# затем: открыть папку как Obsidian-волт, включить Obsidian CLI (Obsidian 1.12+),
# запустить в этой папке агента: claude / codex / gemini
```

Или просто `git clone`. Нужен **Node 22+**, **Obsidian 1.12+**; для семантического поиска — опц. QMD. [ShardMind](https://github.com/breferrari/shardmind) — это «пакетный менеджер» шаблонов волтов того же автора (аддитивен: удалишь `.shardmind/` — всё работает как обычный клон).

> [!tip] Тебе это особенно близко — ты и так живёшь в Obsidian
> У тебя уже есть Obsidian-хранилище (эта база знаний). Obsidian Mind — **отдельный** волт под память агента; не мешай его с личной базой сразу, а поставь отдельно и посмотри протокол заметок — часть идей (decision records, индексы, связи) можно перенять и без установки.
