# 🎯 claude-code-prompt-improver — плагин, улучшающий промпты «нуджами»

**claude-code-prompt-improver** ([github.com/severity1/claude-code-prompt-improver](https://github.com/severity1/claude-code-prompt-improver)) — **плагин-хук для [Claude Code](Claude%20Code%20%E2%80%94%20%D0%B3%D0%B0%D0%B9%D0%B4.md)** (не Codex/Cursor): он в нужные моменты — **при отправке промпта, использовании инструмента и старте субагента** — подкидывает Claude небольшие «нуджи» (context-инъекции), чтобы **первый ответ был точнее** и ты тратил меньше кругов на исправления. Девиз: «*type vibes, ship precision*». MIT, Python, **~1800★**.

> [!info] Сверка поста с репозиторием
> | Заявление поста | Реально |
> | :--- | :--- |
> | «оптимизирует промпты для любых задач, любой объём» | ✅ работает и на «fix the bug», и на инструкции в пару страниц; для **ясных** промптов почти без накладных, для **размытых** — запускает скилл и задаёт 1–6 уточняющих вопросов |
> | «небольшие инъекции, не рушит процессы» | ✅ 9 «нуджей» с условиями («если это X… иначе игнорируй») — ложное срабатывание отбрасывается дёшево |
> | «проверяет, что Claude получает всё нужное при отправке/инструменте/субагенте» | ✅ именно так: три момента — prompt submit, tool use, subagent start |
> | «оптимизирует ресурсы и время, бесплатно» | плагин бесплатный (MIT), но см. нюанс токенов ниже — «бесплатно» относится к **самому плагину**, а гоняет он твой Claude Code |

> [!caution] «31% экономии токенов» — это не «в 1.31 раза дешевле, чем без него»
> В README цифра **31% token reduction** — это **относительно ПРЕДЫДУЩЕЙ версии самого плагина** (v0.4.0 vs старая архитектура), **а не** «на 31% дешевле, чем вообще без плагина». По факту плагин **добавляет накладной расход**: на **каждый** промпт летит оценочный запрос (~189 токенов), а на размытых промптах — целый цикл ресёрча (Glob/Grep/Web/Read) + вопросы. Экономия — **чистая, на длинной дистанции**: пара сотен токенов вперёд, чтобы не гонять модель по 3–4 круга исправлений. На уже-чётких промптах это почти ноль, на «vibe»-запросах — окупается меньшим числом итераций.

> [!note] Меняет флоу на размытых запросах
> Если промпт вправду мутный, плагин **притормозит и задаст тебе 1–6 вопросов** через `AskUserQuestion`, прежде чем делать. Это по делу (лучше спросить, чем угадать), но если ты любишь «просто делай молча» — учитывай. Нужен **Claude Code 2.0.22+** (использует инструмент AskUserQuestion).

## 🧩 Как работает (9 «нуджей»)

- **`improve`** (на каждый промпт) — проверка ясности; на размытом вызывает скилл `prompt-improver` → план ресёрча → 1–6 обоснованных вопросов.
- **`plan-mode`** (на каждый промпт) — оценивает, не сложная ли задача, чтобы уйти в plan-режим; на тривиальном сам отменяется.
- **По условию**: `approach-assessment` (нетривиальное: субагент/оркестрация/просто сделать), `workflow` (многошаговое: план + роутинг моделей), `output-readability` (крупный ответ: вывод вперёд, секции/таблицы, короче), `ask-user-question` (скрытое решение — спросить с вариантами), `plan` (терсный план при входе в plan-mode), `background-exec` (долгие команды в фон), `subagent-routing` (research-субагент: ширина > глубины, выводы, не дампы).

Только 2 из 9 срабатывают всегда; остальные — по ключевым словам/точным условиям, поэтому «мимо» дёшево гасится.

## 🛠️ Установка

```bash
claude plugin marketplace add severity1/severity1-marketplace
claude plugin install prompt-improver@severity1-marketplace
# перезапусти Claude Code; проверь через /plugin
```

Требуется **Claude Code 2.0.22+**. Есть и локальная установка из клона (`.dev-marketplace`).
