# 🎨 beautify-github-readme — скилл для редизайна README на GitHub

**beautify-github-readme** ([github.com/oil-oil/beautify-github-readme](https://github.com/oil-oil/beautify-github-readme)) — это **Skill** (пакет инструкций для ИИ-агента кода — Claude Code, Codex и т.п.), который **читает твой репозиторий и переделывает README** в наглядную «страницу проекта»: правит порядок и текст, генерит **SVG-обложку (hero)**, разделители секций, диаграммы, опц. GIF, и выстраивает project-native дизайн — свой под каждый проект. Устанавливается одной командой `npx skills add oil-oil/beautify-github-readme`. MIT, Python, **~1050★**, свежий (репозиторий от 13 июля 2026, активный).

> [!warning] «Ноль ИИ-слопа / галлюцинировать фичами не будет» — это ЦЕЛЬ, а не гарантия
> | Заявление поста | Реально |
> | :--- | :--- |
> | «скилл сделает из унылых README стильные страницы» | ✅ по назначению — редизайн README через твоего агента. Но это **Skill (промпт-инструкции), не самостоятельный ИИ**: работает поверх **твоего** Claude Code/Codex и тратит **твои** токены/API |
> | «Ноль ИИ-слопа. Галлюцинировать фичами не будет» | ⚠️ это **встроенные правила** скилла («use real project material», «never invent capabilities», «never publish without approval»), а **не техническая гарантия**. Итог всё равно зависит от того, как LLM следует инструкциям — README после генерации **проверяй сам** |
> | «скилл чешет весь код проекта» | ⚠️ он **читает репозиторий/README** для контекста (ценность, доказательства), потом решает дизайн. «Весь код целиком» не обязательно перечитывает; на большом проекте whole-README режим = **много токенов** |
> | «ИИ-агент добавит скрины» | ❌ **реальные скриншоты приложения он не наколдует**. PNG/WebP-скрины берутся из **твоего** материала; сам он генерит SVG-обложки/секции/диаграммы и опц. GIF |
> | «сгенерит SVG-обложки и уникальный дизайн» | ✅ да: project-native hero, «theme wall», секции; каждый проект получает **свой** стиль, а не один шаблон |
> | «это будет лучший вариант вашего проекта» | 🟡 маркетинг |

> [!info] Что реально хорошо задумано (это не пустой хайп)
> Философия скилла как раз **анти-слоп** и совпадает с твоими заметками про борьбу с «ИИ-слопом»:
> - **Разделение слоёв**: SVG держит визуал (hero, диаграммы, идентичность), **Markdown — текст/команды/ссылки**. То есть README **не превращается в одну большую картинку**, которую нельзя искать/копировать/поддерживать (частая беда «красивых» README).
> - **GitHub-safe SVG**: GitHub **вырезает скрипты и часть стилей** из SVG в README, поэтому фэнси-обложки часто ломаются. Скилл целенаправленно целится в «безопасные для GitHub» SVG — это реальная боль, а не выдумка.
> - **Approval-gated**: по умолчанию **не коммитит, не пушит, не публикует** без явного разрешения; чтение README ≠ право его редактировать. Это правильный дефолт.
> - **Motion opt-in**: анимация (GIF) только по запросу, статический SVG остаётся редактируемым фолбэком.

## 🧩 Два режима

| Режим | Что меняет | Что не трогает по умолчанию |
| :--- | :--- | :--- |
| **Whole README** | порядок чтения, иерархию текста, «доказательства», Markdown и весь визуал | не коммитит/пушит/публикует без approve |
| **Asset-only** | делает только SVG-hero/секции/workflow/бейдж/диаграмму или GIF (+SVG-исходник) | **не трогает** текст README, порядок, ссылки, картинки |

Есть и **read-only аудит**: «проверь README на ясность/иерархию/доверие/стоимость поддержки, файлы не редактируй».

## 🛠️ Установка и запуск

```bash
# Вариант 1 — из CLI:
npx skills add oil-oil/beautify-github-readme
# Вариант 2 — попросить агента: "Install this Skill: https://github.com/oil-oil/beautify-github-readme"
```

Затем агенту, например:
```text
Use $beautify-github-readme to redesign this repository homepage around its real
project theme. Show me a local preview first and do not push anything.
```

> [!tip] Тебе это буквально применимо — эта база знаний рендерится на GitHub
> У тебя README вручную с бейджем-счётчиком заметок. Можно прогнать **asset-only** (только hero-SVG, не трогая текст и авто-счётчик через GitHub Actions) и посмотреть превью локально — **без коммита**. Осторожно: whole-README режим переставит текст и может задеть строку бейджа-счётчика и внутренние ссылки — если пробовать, только с ручной вычиткой диффа.

> [!note] Нюансы, которые стоит знать
> - **«Used by» ≈ самопродвижение**: 6 репозиториев в списке «уже используют» — это в основном **проекты того же автора** (oil-ppt, draw-ui, oil-icon, selector, codex-dev-team) + один внешний. «Доказательство внедрения» пока в экосистеме автора.
> - **Язык примеров** — английский/китайский/японский. Для **русского** README сработает, но встроенные образцы на латинице; **кириллическую типографику в SVG проверяй** (переносы, шрифты, засечки).
> - **Свежий и вирусный**: репо 2 недели, ~1050★ — быстрый рост, но зрелость/стабильность ещё не проверена временем.
