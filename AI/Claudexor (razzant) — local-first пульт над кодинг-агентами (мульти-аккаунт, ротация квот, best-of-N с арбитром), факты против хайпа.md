# 🎛️ Claudexor — local-first «пульт» над кодинг-агентами

**Claudexor** ([github.com/razzant/claudexor](https://github.com/razzant/claudexor)) — **local-first control plane** поверх кодинг-агентов, которые ты **уже оплачиваешь**: [Claude Code](Claude%20Code%20%E2%80%94%20%D0%B3%D0%B0%D0%B9%D0%B4.md), Codex CLI, Cursor CLI, OpenCode + сырые API-адаптеры — под одним типизированным интерфейсом. Идея: у каждого харнесса свой логин и квота, а Claudexor даёт слой, которого у вендоров нет — **несколько аккаунтов с ротацией по квоте, гонки best-of-N с независимым арбитром, честный учёт бюджета и защищённые пути**. Всё крутится **на твоей машине, файлы — источник правды, телеметрии нет**. MIT, TypeScript, ~250★, автор один (razzant), активная разработка (v3.0).

> [!warning] Отделяем факты от рекламы поста
> | Заявление поста | Реально |
> | :--- | :--- |
> | «токенов в 8 раз меньше», «было 15 тыс/мес → стало 2 тыс» | это **личная история автора**, не гарантия. Экономия не от «магии», а оттого что ты **уходишь с поштучной оплаты API на несколько фикс-подписок** и гоняешь их квоту по кругу. Твоя выгода зависит от твоих тарифов/объёма |
> | «один интерфейс для Claude Code/Codex/Cursor/OpenCode» | ✅ так и есть — один control plane над вендорскими CLI |
> | «несколько аккаунтов Claude/Codex, переключение за секунду с сохранением контекста» | ✅ **credential profiles** (с v2.1): изолированные логины, живой трекинг квоты, opt-in ротация «выработанного» аккаунта на лимите |
> | «несколько агентов решают одну задачу + независимый ИИ-арбитр» | ✅ **best-of-N races** с cross-family review и арбитражем; read-ходы резюмируют нативную сессию вендора, write-ходы — инспектируемые патчи |

> [!caution] Важные оговорки, о которых пост молчит
> - **best-of-N сам по себе жрёт БОЛЬШЕ токенов** (несколько харнессов на одну задачу). Общая экономия — только если она перекрывается переходом с API на подписки; иначе гонки наоборот повышают расход.
> - **Мульти-подписки и ротация «на лимите» — серая зона ToS** вендоров (Anthropic/OpenAI). Технически удобно, но это жонглирование правами доступа — на свой риск.
> - **Не для новичка**: это оркестратор ПОВЕРХ уже залогиненных вендорских CLI, а не «поставил и само работает».
> - **Молодой, один мейнтейнер** (~250★, v3.0): контракты меняются — в README прямо сказано, что снятые команды/режимы жёстко падают с ошибкой, а не тихо алиасятся. Обновляйся осознанно.

## 🧩 Что внутри (по README)

- **Режимы**: `ask` (read-only ответ; `--deep-scan` — ограниченный мульти-разведочный ресёрч кодовой базы с синтезом), `plan` (read-only планирование; `--council --n 2..4` — план параллельно на N харнессах → мёрж в один), плюс agent-режим с патчами.
- **Credential profiles + quota**: несколько подписок Claude/Codex рядом, per-window бары квоты (5ч/7дн), авто-переключение на лимите (opt-in).
- **Races/арбитраж**: best-of-N, независимый ревьюер другой «семьи» моделей, арбитр; субагенты и глубокий скан репозитория.
- **Гейты и бюджеты**: детерминированные гейты, защищённые пути, честный учёт (неизвестная стоимость никогда не `$0`), веб-доказательства как типизированные факты.
- **Локальность**: демон + CLI + десктоп-приложение (подписанный DMG на macOS), git-worktrees под изоляцию, MCP-сервер для интеграций.

## 🛠️ Установка

```bash
npm install -g claudexor   # ставит бины claudexor и claudexord
claudexor doctor           # проверка окружения
```

**Нужно**: Node.js ≥ 20.19, `pnpm` (через corepack), `git`, и **хотя бы один залогиненный вендорский CLI** (`claude`/`codex`/`cursor-agent`/`opencode`) **или** API-ключ (`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`…). Логиниться нужно **через Claudexor**, а не через голый вендорский CLI. На macOS проще поставить приложение из [Releases](https://github.com/razzant/claudexor/releases) (подписанный DMG).
