# 📊 abtop — «htop для AI-агентов кода» (мониторинг сессий Claude Code / Codex / OpenCode)

[**abtop**](https://github.com/graykode/abtop) — терминальный (TUI) монитор сессий **AI-агентов для кода**, как `htop`, только не для процессов, а для агентов. В реальном времени показывает по каждой активной сессии: **расход токенов**, **заполнение контекстного окна** (% с предупреждениями), **rate-limits**, дочерние процессы и открытые порты. Написан на **Rust**, лицензия **MIT**, ~3k★, текущая версия **v0.4.8** (8 июня 2026).

> [!info] Что поддерживает (по факту с GitHub)
> Мониторит **Claude Code**, **Codex CLI** и **OpenCode**. В Telegram-посте упомянуты только первые два — на деле агентов три. Работает **в режиме только для чтения**: не требует API-ключей и аутентификации, находит сессии по локальным процессам и файлам состояния (`~/.claude` и т.п.), ничего никуда не отправляет.

> [!warning] Чего ждать / ограничения
> - Это **наблюдатель**, а не управление: abtop ничего не меняет в сессиях, только читает и показывает.
> - У **OpenCode** пока **не отображается** процент контекстного окна; для OpenCode нужен `sqlite3` в `PATH`.
> - Цифры токенов/лимитов берутся из локальных логов агентов — это оценка по тому, что агент записал, а не «официальный» биллинг провайдера.
> - Версия `0.x` — проект молодой, API/формат может меняться.

## ✨ Возможности

- **Мульти-сессия**: одновременный обзор всех активных сессий по разным проектам.
- **Контекст-бары** с процентом заполнения окна и предупреждением, когда близко к пределу.
- **Rate-limits** в реальном времени.
- **Поиск «осиротевших» портов** (orphan ports) и дочерних процессов активных сессий.
- **12 встроенных тем**, включая дружественные к дальтоникам.
- **Интеграция с tmux** — переключение между сессиями.
- **Экспорт JSON-снапшота** для скриптов (через библиотечный API, без повторного сканирования и подпроцессов).

## 📦 Установка

Кроссплатформенный Rust-бинарь: **Linux / macOS / Windows** (нативно, WSL не требуется). Самый универсальный путь — `cargo` (нужен Rust toolchain) либо готовый бинарь из GitHub Releases.

| Система | Как поставить |
| :--- | :--- |
| **Gentoo** | ebuild'а нет → `cargo install abtop` (предварительно `emerge dev-lang/rust` или `rust-bin`). Либо бинарь из [Releases](https://github.com/graykode/abtop/releases) |
| **Debian / Ubuntu** | в репозиториях нет → `cargo install abtop` (пакет `rustup`/`cargo`) или скрипт `curl … | sh` / бинарь из Releases |
| **Arch** | `cargo install abtop`; проверь **AUR** на пакет `abtop`/`abtop-bin` (если появился — `yay -S abtop`) |
| **Windows** | PowerShell-инсталлятор из репозитория или бинарь из Releases |
| **Entware / RT-AX56U** | **N/A** — на роутере не крутятся Claude Code/Codex/OpenCode, мониторить нечего; плюс сборка Rust под armv7 тяжела. Инструмент для рабочей машины |

> [!tip] Рекомендации по терминалу
> Комфортный размер окна — **120×40** и больше (минимум 80×24). Запуск: просто `abtop`. JSON-снапшот для скриптов/дашбордов — встроенным экспортом.

## 💡 Кому полезно

- Гоняешь **несколько сессий Claude Code/Codex/OpenCode** параллельно и хочешь видеть на одном экране, **сколько съедено токенов и насколько забит контекст**, чтобы вовремя сжать/перезапустить.
- Нужно ловить **подвисшие порты и дочерние процессы** агентов.
- Любишь `htop`-подобный «пульт» в терминале вместо лазанья по логам.
